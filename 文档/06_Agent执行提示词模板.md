> 本文档为所有参与项目开发的AI/Agent提供执行规范和提示词模板。
> 任何Agent在开始工作前应先阅读 `01_项目总控与AI执行说明.md` 获取全局上下文。

## 一、Agent执行前置清单

在执行任何开发任务前，Agent 必须：

1. 阅读 `01_项目总控与AI执行说明.md` — 了解项目全貌
2. 阅读 `14_暗黑3式系统框架设计.md` — 了解系统设计权威参考
3. 阅读 `13_武侠命名映射表.md` — 了解命名规范
4. 根据任务类型，阅读对应的详细设计文档（02-12）

## 二、硬性执行规则

### 2.1 代码规范

| 规则 | 说明 |
|------|------|
| 语言 | GDScript，不使用C#或GDNative |
| 命名 | 英文 snake_case（代码），中文（面向玩家的显示名） |
| 数据驱动 | 可调参数放 `data/*.json`，不硬编码 |
| 显示名映射 | 参照 `13_武侠命名映射表.md` |
| class_name | 新增系统脚本需声明 `class_name` |

### 2.2 架构规范

| 规则 | 说明 |
|------|------|
| Autoload | 不可随意新增，修改需说明理由 |
| 信号 | 通过 EventBus 发送全局信号，不直接跨场景连接 |
| 存档 | 影响存档结构的变更必须递增 CURRENT_SAVE_VERSION |
| 现有功能 | 每次修改确保不引入新的 SCRIPT ERROR |

### 2.3 文档规范

| 规则 | 说明 |
|------|------|
| 权威参考 | `14_暗黑3式系统框架设计.md` 为最终依据 |
| 命名参考 | `13_武侠命名映射表.md` 为命名参照 |
| 数据Schema | `06_配置表字段说明.md` 为JSON格式参照 |
| 修改后 | 需同步更新相关设计文档 |

## 四、常用提示词模板

### 4.1 P1: 传承系统实现

```
你是一名Godot 4游戏开发者。请实现传承（套装）系统。

项目背景:
- 武侠挂机RPG，暗黑3式装备驱动
- 已有: 装备系统(9槽/7品质/38词缀/传奇橙字)、多乘区伤害公式
- 需新增: 传承(套装)系统

核心需求:
1. 创建 SetSystem (scripts/systems/set_system.gd)
2. 读取 data/sets/set_defs.json 中的4套传承定义
3. 追踪玩家当前穿戴的传承件数
4. 当件数达到2/4/6时激活对应效果
5. 将套装加成注入 DamageResolver 的乘区E

参考文档:
- 14_暗黑3式系统框架设计.md 第七节
- 04_传承·百炼坊·秘境·宗师·重入江湖.md 第一节
- 06_配置表字段说明.md 第九节(set_defs schema)

验收标准:
- 穿6件御风传承 → E乘区=5.0 → DPS×5
- 参照 07_配置示例与测试用例.md 第五节用例1
```

## 六、常见陷阱提醒

1. **不要使用修真/仙侠用语**：灵气、飞升、渡劫、法宝 等词汇禁止出现
2. **品质名称**：精良(不是灵品)、传承(不是道统)、上古(不是远古)、绝世(不是天道)
3. **流派名称**：御风刀(不是御风道)、血劫手(不是血劫道)、五雷掌(不是五雷道)
4. **百分比存储**：JSON中百分比用小数 0.05，不是整数 5
5. **Bucket互斥**：同一件装备上同bucket最多1条词缀，不可重复
6. **存档版本**：任何改变存档结构的修改必须递增版本号
7. **套装乘区**：套装效果走乘区E，和其他乘区是乘法关系，不是加法
