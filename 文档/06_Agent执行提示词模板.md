# Agent 执行提示词模板

本文件用于把本目录文档继续拆给其他 agent 型 AI 执行。每个提示词都假设对方 agent 需要在 `Godot 4` 项目里直接开始工作。

使用原则：

- 每次只给 agent 一个明确目标
- 提供它需要优先阅读的文档
- 要求它先搭最小闭环，不要过度扩写
- 要求它输出变更说明和验收结果

## 1. 总控 Agent

```text
你现在是本项目的总控开发 agent。请先阅读以下文档：
1. 01_项目总控与AI执行说明.md
2. 04_Godot落地架构与开发任务拆分.md
3. 05_配置表字段说明.md

你的任务是：
- 先建立 Godot 4 项目目录结构
- 建立主场景、Autoload 和基础数据加载框架
- 确保后续战斗、掉落、成长模块都能接入

要求：
- 优先做最小可运行结构
- 不要先做复杂 UI
- 不要把具体技能或敌人逻辑写死
- 所有配置字段尽量与 05 文档对齐

完成后请输出：
- 新增文件列表
- 场景树结构
- Autoload 列表
- 当前还缺什么才能进入战斗原型
```

## 2. 战斗系统 Agent

```text
你现在负责 Godot 4 项目的战斗系统。请先阅读：
1. 01_项目总控与AI执行说明.md
2. 02_核心战斗与系统循环设计.md
3. 04_Godot落地架构与开发任务拆分.md
4. 05_配置表字段说明.md

你的任务是：
- 搭建自动横版推进战斗主循环
- 实现角色自动前进、索敌、攻击、击杀、波次推进
- 实现普通敌人、精英敌人和 Boss 的最小行为
- 预留触发器与效果器接口

要求：
- 战斗必须默认自动进行
- 不要依赖复杂动画资源
- 所有伤害结算走统一接口
- 先做 1 个角色和 3 类敌人即可

完成后请输出：
- 战斗主循环实现说明
- 关键脚本职责
- 当前可测试的玩法闭环
- 尚未实现但已预留的扩展点
```

## 7. 测试 Agent

```text
你现在负责测试和验收。请先阅读：
1. 01_项目总控与AI执行说明.md
2. 02_核心战斗与系统循环设计.md
3. 03_养成经济与长期成长设计.md

你的任务是：
- 检查当前版本是否形成完整挂机闭环
- 检查 build 是否能改变战斗结果
- 检查离线收益是否合理
- 检查是否存在必须手动操作才能推进的问题

请重点验证：
- 5 分钟内是否能理解玩法
- 30 分钟内是否有明显成长
- 卡关时是否有多种解决方案

完成后请输出：
- 发现的问题列表
- 严重程度
- 可复现步骤
- 建议修正方向
```

## 8. 推荐协作顺序

如果多个 agent 并行工作，建议顺序如下：

1. 总控 Agent 先搭框架
2. 战斗系统 Agent 与配置表 Agent 并行
3. 装备与掉落 Agent 接入战斗
4. 成长与研究树 Agent 接入资源循环
5. UI Agent 最后补关键界面
6. 测试 Agent 做闭环验收

## 9. 统一输出要求

所有 agent 在完成任务后，都应按以下格式汇报：

- 做了什么
- 改了哪些文件
- 当前能跑通的功能
- 还缺什么
- 风险点是什么

这样便于下一位 agent 继续接手。
