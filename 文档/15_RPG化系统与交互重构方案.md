# AFK-RPG RPG 化系统与交互重构方案

日期：2026-03-28

---

## 1. 重构目标

本轮重构不是单纯换一版面板皮肤，而是把当前项目从“列表式挂机管理工具”推进到“更像 ARPG / RPG 的装备驱动体验”。

目标聚焦三件事：

1. 打开背包时，第一感受应是“角色装备页 + 格子库存”
2. 百炼坊应更像 RPG 工坊，而不是配置表操作面板
3. 主 HUD 应更像主循环驾驶舱，而不是散装信息卡堆叠

## 3. 当前结构问题

重构前主要问题如下：

- `InventoryPanel` 以 `ItemList + 详情文本 + 按钮` 为主，缺少 RPG 角色装备页的空间感
- 玩家没有“角色穿了什么、背包还剩多少格”的直观体感
- 百炼坊虽有真实功能，但交互仍偏“配方工具页”
- HUD 的目标、Build、掉落决策在语义上是分散的，主循环入口不够统一
- 物品展示逻辑分散在 `GameManager`、HUD、背包、百炼坊多个地方，难以继续演化

---

## 4. 本轮冻结方案

### 4.1 库存模型

- 不改存档中的 `inventory: Array`
- 不改存档中的 `equipped_items: Dictionary`
- 背包只做“格子化展示层”，不做真实摆放算法
- 本轮固定为：
  - `10 x 4` 展示网格
  - 每件物品固定 `1 x 1`
  - 空格只承担容量感与筛选感

### 4.2 装备页结构

`背包` 入口升级为真正的 RPG 装备页：

- 左：角色纸娃娃 9 槽
- 中：10x4 格子背包
- 右：物品详情 / 对比 / 标签
- 底：筛选、排序、自动分解阈值、装备、卸下、锁定、分解

纸娃娃槽位固定沿用现有真实存档槽：

- `weapon`
- `helmet`
- `armor`
- `gloves`
- `legs`
- `boots`
- `accessory1`
- `accessory2`
- `belt`

### 4.3 百炼坊结构

`百炼坊` 保持原面板 id 和入口，但内部重构成工坊式三栏：

- 左：工坊分支与候选目标
- 中：目标详情与必要选项
- 右：材料、结果预览、执行反馈

`武学秘录` 仍只作为 `百炼坊` 内部子页，不回流一级导航。

### 4.4 HUD 结构

主 HUD 的信息架构冻结为：

- 左：当前战斗、节点、血量、流派、掉落方向
- 中：战斗舞台和掉落高光
- 右：当前 Build、传承、武学秘录、下一件目标
- 底：最近掉落、已穿装备、快速入口

本轮不重做整套导航，只重构 `HUD 主链 + 背包 + 百炼坊`。

### 4.5 HUD 焦点路由

HUD 不再只是展示层，而要承担“看见问题 -> 立刻进入对应处理页”的主循环职责。

当前点击路由冻结如下：

- `当前 Build` 卡：进入 `推演与秘境`
- `当前目标` 卡：进入 `成长中心`
- `已穿装备` 卡：进入 `背包`
- `最近掉落` 卡：
  - 若是可萃取武学装备，直达 `百炼坊 -> 萃取武学`
  - 若是自动换装产物，直达 `背包` 并聚焦对应已穿槽位
  - 若是高价值入包装备，直达 `背包` 并聚焦该物品

为此，运行态引入轻量 `UI Focus Request` 机制，但不修改存档结构。

## 5. 领域服务拆分

为避免 UI 继续直接读取 `GameManager` 原始数组，本轮新增四个轻量 ViewModel / Presentation 服务：

### 5.1 `InventoryViewModelService`

职责：

- 把 `inventory: Array` 映射成格子背包状态
- 统一处理筛选、排序、分页
- 输出 `grid_entries / selected_item / compare_summary / action_state`

### 5.2 `EquipmentViewModelService`

职责：

- 输出 9 槽纸娃娃状态
- 汇总已穿装备数量、传承摘要、武学秘录摘要、下一件建议

### 5.3 `ItemPresentationService`

职责：

- 统一物品短名、槽位显示名、格子标签、标签组、对比摘要
- 统一“高价值 / 传承 / 武学 / 已锁定 / 已精炼”等展示口径

### 5.4 `CubeViewModelService`

职责：

- 按百炼分支过滤候选物
- 输出配方材料文本、结果预览、可选目标槽位、可淬火词条、秘录配置态

### 5.5 统一物品卡片组件

为了避免 `背包格子 / 纸娃娃槽位 / 百炼坊候选物` 三处各自拼 UI，本轮新增统一卡片组件：

- `ItemCardButton`

当前约定：

- 组件负责统一标题、副标题、标签、边框色、选中态、占位态
- 背包格子、纸娃娃槽位、百炼坊候选物都改为复用它
- 具体数据仍由各自的 ViewModel 提供

这样后续要继续做：

- 更强的品质框
- 追踪目标高亮
- 拖拽反馈
- 传承/武学专属视觉

都可以集中在卡片组件层处理，而不是散落在多个控制器里。

## 6. 当前已落地内容

### 6.1 GameManager 聚合接口

已新增：

- `get_inventory_screen_state()`
- `get_equipment_screen_state()`
- `get_cube_screen_state(page_id)`
- `unequip_slot(slot_id)`

这些接口只提供 UI 消费所需的只读聚合态，不改变现有存档结构。

### 6.2 背包 RPG 化

`InventoryPanel` 已完成第一轮 RPG 化重构：

- 旧 `ItemList` 主结构已替换为 `纸娃娃 + 格子背包 + 详情/对比 + 工具栏`
- 已支持：
  - 10x4 格子展示
  - 纸娃娃 9 槽选择
  - 筛选
  - 排序
  - 分页
  - 装备
  - 卸下
  - 锁定
  - 分解
  - 自动分解阈值切换

### 6.3 百炼坊工坊化

`CubePanel` 已完成第一轮三栏重构：

- 左侧从旧列表式入口改成工坊分支 + 候选物列表
- 中侧承接详情与选项
- 右侧承接材料、结果预览与执行反馈
- 五个配方和 `武学秘录` 子页都继续复用现有真实运行链

### 6.4 HUD 术语收口

HUD 已完成首轮语义校正：

- `TargetCard` 明确收口为 `当前 Build`
- `DailyGoalCard` 收口为 `当前目标`
- `EquipCard` 收口为 `已穿装备 · 9槽`

这一步主要是先让首页信息架构更像 RPG 主循环入口，而不是继续保留旧的“机缘追踪 / 今日机缘”工具式标题。

### 6.5 HUD 到面板的直达链路

HUD 已补入第一轮直达交互：

- 点击 `当前 Build` 直达 `推演与秘境`
- 点击 `当前目标` 直达 `成长中心`
- 点击 `已穿装备` 直达 `背包`
- 点击 `最近掉落` 时，会根据掉落类型自动分流到：
  - `背包`
  - `背包装备槽位`
  - `百炼坊 -> 萃取武学`

### 6.6 统一卡片组件已接入

`ItemCardButton` 已完成首轮接入：

- `InventoryPanel` 的纸娃娃槽位
- `InventoryPanel` 的格子库存
- `CubePanel` 的候选物列表

这意味着 RPG 化结构现在不只是“布局更像 RPG”，连物品展示组件也开始统一，后续继续做视觉强化和交互深化的成本会明显降低。
