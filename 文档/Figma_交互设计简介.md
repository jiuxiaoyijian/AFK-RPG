# AFK-RPG · Figma 交互设计简介

> 本文档用于冻结暗黑like 重构后的正式信息架构。
> 所有 Figma 设计、网页原型、Godot UI 回灌，都必须以本文为准。
> 画布统一为 **1280 × 720 横屏**。

---

## 一、产品定位

- 类型：桌面端挂机刷宝 ARPG
- 核心体验：**观看战斗 → 看掉落 → 换装备 → 调 Build → 冲更高层**
- 系统骨架：暗黑3式循环
- 包装方向：武侠江湖，不做西幻外观
- 目标感受：自动战斗必须好看，掉落必须让人想点开，Build 调整必须有明显体感

一句话概念：

`玩家不是在“修仙桃源”里慢慢养成，而是在一个武侠江湖包装下，持续刷装、换装、精炼、冲层。`

---

## 二、设计原则

### 2.1 先保证“看懂”

- 主界面只保留战斗、掉落、资源、当前 Build 摘要
- 深度信息全部进入二级面板
- 玩家在 10 秒内必须知道：
  - 现在在刷哪一章/哪一节点
  - 当前 Build 走向是什么
  - 最近掉了什么
  - 下一步该开哪个面板

### 2.2 先保证“像暗黑like”

- 主 HUD 不再堆很多静态卡片
- 战斗区优先级高于说明文字
- 掉落高光、套装件数、强化/精炼/萃取入口必须清楚
- 终局入口必须显性化：秘境、宗师、重入江湖不能继续埋在旧原型结构里

### 2.3 先保证“像正式产品”

- 一级导航固定
- 面板职责不能重叠
- 文案统一使用武侠命名映射表
- 禁止继续沿用旧“今日机缘 / 机缘追踪 / 闭关所得”作为主框架命名

## 三、术语与入口冻结

### 3.1 一级导航

| 快捷键 | 面板 | 正式显示名 | 职责 |
|--------|------|-----------|------|
| `I` | InventoryPanel | **背包** | 看装备、换装、锁定、分解、查看套装和宝石信息 |
| `B` | CubePanel | **百炼坊** | 萃取武学、精钢化真、回炉重铸、传承互转、淬火精炼 |
| `O` | CodexPanel | **江湖见闻录** | 掉落图鉴、词条收集、机缘目标追踪 |
| `U` | ProgressionHub | **成长中心** | 武学参悟、宗师修为、重入江湖 |
| `P` | AnalysisHub | **推演与秘境** | 机缘推演、掉落统计、试剑秘境 |
| `G` | GMPanel | **GM** | 调试专用，不纳入正式产品导航叙事 |

### 3.2 二级术语

| 系统 | 正式显示名 | 备注 |
|------|-----------|------|
| Set | **传承** | 套装系统 |
| Cube | **百炼坊** | 暗黑魔盒功能集合 |
| Codex Slot | **武学秘录** | 百炼坊内二级页，不再单独占一级入口 |
| Rift | **试剑秘境** | 终局冲层玩法 |
| Paragon | **宗师修为** | 满级后长线成长 |
| Season/Rebirth | **重入江湖** | 长线轮回 |
| Loot Analysis | **机缘推演** | 节点掉率和追踪分析 |

### 3.3 禁止继续使用的旧主框架词

- 今日机缘
- 机缘追踪
- 闭关所得
- 破局要闻
- 悟道（作为一级导航名）

这些概念可以保留在低频文案或旧功能兼容层里，但**不再作为主 UI 结构命名**。

## 四、主界面结构

### 4.1 整体布局

```text
┌──────────────────────────────────────────────────────────────┐
│ TopBar: 资源 / 背包容量 / 当前章节 / 当前节点 / 战斗状态         │
├──────────────────────────────────────────────────────────────┤
│ Left Battle Info  │              Battle Stage               │ Right Build Info │
│ 章节 / 节点 / 血条 │   玩家 vs 敌群 / 技能高光 / 掉落表演区       │ Build摘要 / 套装 / 秘录 │
│ 最近高光提示       │              Boss/精英重点区域             │ 下一步建议 / 当前缺口   │
├──────────────────────────────────────────────────────────────┤
│ Bottom Quick Strip: 最近掉落 / 高价值记录 / 当前可管理装备数       │
├──────────────────────────────────────────────────────────────┤
│ MainNav: I 背包 | B 百炼坊 | O 江湖见闻录 | U 成长中心 | P 推演与秘境 │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 必须常驻的信息

- 当前章节名
- 当前节点名
- 玩家 HP / 当前战斗状态
- 主要资源：金钱、碎铁、灵核、残片
- 当前流派名
- 当前传承件数摘要
- 最近高价值掉落

### 4.3 必须移出主 HUD 的旧内容

- 长段目标文案
- 两层以上的任务描述
- 离线总结长文本
- 大段数值分析
- 图鉴型说明文字

## 五、二级面板定义

### 5.1 背包

- 结构：左侧物品列表 + 右侧详情 + 底部操作
- 详情必须包含：
  - 品质
  - 槽位
  - 基础属性
  - 词条
  - 传奇特效
  - 传承归属
  - 是否已精炼 / 已锁定
  - 是否可萃取
- 操作必须包含：
  - 装备
  - 锁定
  - 分解
  - 自动处理阈值查看

### 5.2 百炼坊

- 顶部：5 个功能页签
  - 萃取武学
  - 精钢化真
  - 回炉重铸
  - 传承互转
  - 淬火精炼
- 右侧必须有：
  - 材料显示
  - 结果预览
  - 成功反馈区
- 底部可切到 **武学秘录** 子页：
  - 兵器槽
  - 护甲槽
  - 饰品槽
  - 当前激活效果摘要

### 5.3 江湖见闻录

- 继续承担图鉴职责，不再混入武学秘录
- 分类：
  - 底材
  - 词条
  - 真意
  - 传承
- 每个条目必须支持：
  - 查看来源
  - 查看适配流派
  - 设为追踪目标

### 5.4 成长中心

- 三个页签：
  - 武学参悟
  - 宗师修为
  - 重入江湖
- 武学参悟是当前已有研究树的正式承载页
- 宗师修为默认显示“未解锁”占位态
- 重入江湖默认显示“后续阶段开放”占位态

### 5.5 推演与秘境

- 左页：机缘推演 / 掉率 / 节点推荐
- 右页：试剑秘境入口 / 最高层 / 钥石 / 结算记录
- 当前阶段如果秘境尚未落地，右页要做“系统占位态”，不能再完全缺席

### 5.6 GM

- 调试专用
- 正式评审页里弱化显示
- 不参与产品宣传稿

## 六、视觉方向

### 6.1 基础气质

- 武侠，不仙气缥缈，不西幻盔甲感
- 更接近“铁器、旧木、黑漆、铜纹、暗布、纸签”
- 不是桃源感，不做粉雾叙事主轴

### 6.2 色彩建议

- 主底：`#121317`
- 面板：`#1A1D24`
- 次级底：`#232834`
- 金属描边：`#826A3A`
- 强调绿：`#5FA36A`
- 强调红：`#A84A3A`
- 强调蓝：`#5D7FA6`
- 文本主色：`#E6DCC8`
- 文本副色：`#9E9688`

### 6.3 动效重点

- 掉落高光
- 传承 2/4/6 件触发
- 百炼坊操作完成
- Boss 登场 / 击杀
- 试剑秘境通关

## 七、冲突裁决表

| 冲突项 | 旧方案 | 新方案 |
|--------|--------|--------|
| 一级成长入口 | 悟道 | 成长中心 |
| 图鉴页是否承载武学秘录 | 是 | 否，武学秘录并入百炼坊二级页 |
| 推演页职责 | 纯掉率统计 | 掉率统计 + 秘境入口 |
| 主界面右侧卡片 | 今日机缘 / 机缘追踪 | Build 摘要 / 传承摘要 / 下一步建议 |
| 结果卡主叙事 | 闭关所得 / 破局要闻 | 掉落高光 / 传承触发 / 秘境结果 |
| GM 是否参与正式产品导航 | 是 | 否，只保留调试入口 |

## 八、Figma 输出要求

- 请先画主界面，再画 5 个一级面板
- 每个一级面板至少给一个“默认态”和一个“高亮/有结果态”
- 成长中心和推演与秘境都必须体现“多页签结构”
- 百炼坊必须体现“武学秘录是它的内部页签”
- 如果设计稿里需要占位态，请明确写“未解锁 / 后续阶段开放”

---

## 九、验收标准

一版交互稿如果要进入 Godot 实装，至少要满足：

- 一眼能看出这是挂机刷宝，而不是传统修真挂机
- 一眼能找到背包、百炼坊、见闻录、成长、秘境入口
- 战斗区域是第一视觉中心，而不是说明文字
- Build 信息清楚，但不会压住战斗画面
- 主界面没有再出现旧“今日机缘 / 机缘追踪”主结构
