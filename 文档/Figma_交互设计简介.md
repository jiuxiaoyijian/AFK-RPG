## 一、产品概述

### 1.1 定位
- **类型**：桌面端挂机 ARPG（Steam / Web）
- **核心循环**：自动战斗 → 装备掉落 → 词条培养 → 道统（天赋树）成长 → 推进章节
- **参考品类**：暗黑3 + 放置类 + 修真世界观
- **目标体验**：玩家挂着就能看到角色变强，核心决策在于"刷什么"和"选什么道统"

### 1.2 世界观包装
- **题材**：反桃花源修真——"越好看越不对劲"
- **核心概念**：玩家扮演外来修士，探索看似世外桃源的秘境，揭开其中异象
- **关键词**：探境、破局、求机缘、夺异宝
- **视觉调性**：雾青山色、枯桃粉与旧白、墨青/血朱/香火金；古桥、祠堂、纸灯、符纸
- **不是**：不是明亮卡通风，不是常规仙侠换皮

### 1.3 术语映射

| 游戏概念 | 游戏内名称 | 说明 |
|---------|----------|------|
| 天赋树/研究 | **悟道** | 消耗资源解锁被动加成 |
| 图鉴/收藏 | **异闻录** | 记录发现过的底材、词条、传奇 |
| 数据统计 | **机缘推演** | 各节点掉落效率分析和推荐 |
| 每日目标 | **今日机缘** | 自然日刷新的主线+支线目标 |
| 离线收益 | **闭关所得** | 离线期间模拟战斗的收益报告 |
| 里程碑事件 | **破局要闻** | Boss首杀/新章解锁等成就弹窗 |
| 目标追踪 | **机缘追踪** | 指定一件装备追踪其掉落进度 |
| 流派（技能树分支） | **御风道 / 血劫道 / 五雷道** | 三条主要Build路线 |
| 货币 | **香火钱** | 基础货币 |
| 分解材料 | **祠灰** | 装备分解产物 |
| 高级材料 | **灵核** | 悟道解锁所需 |
| 传奇碎片 | **真意残片** | 传奇级合成材料 |

## 四、逐屏详细设计

### 4.1 主战斗界面 (HUD)

**始终可见**的信息层，叠加在战斗场景之上。

#### 区域分布

| 区域 | 位置 | 尺寸(约) | 内容 |
|------|------|---------|------|
| **TopBar** | 顶部 x12 y12 | 1256×44 | 左：四种资源数值 + 背包件数；右：击杀/清图/当前节点 |
| **战斗高光横幅** | 顶部居中 y66 | 420×auto | 仅在精英/Boss/核心技能释放时渐显渐隐，标题+副标题+详情 |
| **BattleCard** | 左侧 x16 y72 | 372×166 | 探境名、章节、状态、HP条、道统名+图标、掉落方向提示 |
| **TargetCard** | 右侧 x936 y118 | 328×120 | 机缘追踪路径、缺口百分比、下一件装备建议、卡关诊断 |
| **DailyGoalCard** | 右侧 TargetCard下方 | 328×138 | 今日机缘图标+标题、主目标、进度(0/3)、建议操作、支线 |
| **BattleSafeFrame** | 底部导航栏上方 | 动态宽度×210 | 透明底 + 青色细边框，标记"战斗可视安全区" |
| **EquipCard** | SafeFrame内左侧 | ~370×auto | 4格装备概览（武器/头盔/铠甲/手套）+ 品质边框 |
| **LootCard** | SafeFrame内右侧 | ~300×auto | 高价值掉落、最近掉落列表 |
| **DropToast** | 顶部居中 | 400×54 | 传奇/追踪目标掉落时的临时提示条 |

#### 战斗区域行为
- **角色** 始终在屏幕中央（x=640, y=520），左右移动范围 1/3 屏
- **怪物** 从左右两侧随机刷新，向中间移动
- **掉落物** 抛物线动画 → 落地 → 主动拾取(靠近)/被动拾取(3s) → 粒子飞向对应UI入口
- **背景图** 根据章节+节点动态切换

### 4.2 底部导航栏 (MainNavBar)

**固定在底部**，高度 92px，全宽。

| 按钮 | 快捷键 | 动态副文本示例 | 点击行为 |
|------|-------|-------------|---------|
| 背包 | I | "0 件可管理" | 打开背包面板 |
| 悟道 | U | "可悟道 0 项" | 打开悟道面板 |
| 异闻录 | O | "机缘: 风眼" | 打开异闻录面板 |
| 推演 | P | "节点 0 个 \| 最近 0 条" | 打开机缘推演面板 |
| GM | G | "调试入口" | 打开GM调试面板 |

**底部提示行**：`1/2/3 切道统 R 重开 T 切机缘 G GM F5/F8 档1 Esc 关闭`

**交互规则**：
- 同一时间只能打开一个面板（互斥）
- 再次点击同一按钮 = 关闭面板
- 打开面板时显示半透明遮罩(#000 50%)
- 面板打开时隐藏导航栏和掉落特效层
- Esc 关闭当前面板

### 4.8 GM调试面板 (GMPanel)

**仅开发/测试用**，居中弹出。功能分组：

- **资源操作**：添加/设置四种资源数量
- **物品操作**：向背包生成指定品质装备
- **进度操作**：跳转到指定节点、解锁全部悟道、解锁全部异闻录

- **存档操作**：3个存档槽位、存档/读档/删除

## 五、交互流程

### 5.1 核心Loop（玩家视角）

```
启动游戏
  ↓
显示闭关所得（如有离线收益）
  ↓
自动战斗开始 ← ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐
  ↓                                  │
击杀怪物 → 掉落装备/资源              │
  ↓                                  │
掉落物抛物线 → 落地 → 拾取飞向UI      │
  ↓                                  │
自动装备/分解（根据阈值）              │
  ↓                                  │
波次清完 → 下一波/下一节点             │
  ↓                                  │
Boss击杀 → 破局要闻弹窗               │
  ↓                                  │
解锁新节点/新章 ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

### 5.2 成长Loop（决策视角）

```
查看机缘追踪 → 了解当前Build缺口
  ↓
查看机缘推演 → 找到最优刷取节点
  ↓
查看今日机缘 → 确认当日目标
  ↓
挂机战斗 → 获得装备/资源
  ↓
打开背包 → 手动管理关键装备
  ↓
打开悟道 → 消耗资源提升道统
  ↓
Build变强 → 推进更深节点 → 循环
```

### 5.3 面板切换流程

```
主界面
  │
  ├── 点击"背包"/按I ──→ 背包面板（遮罩+隐藏导航）
  │                        ├── 选择装备 → 详情
  │                        ├── 装备/分解/锁定
  │                        └── Esc/再按I → 关闭 → 主界面
  │
  ├── 点击"悟道"/按U ──→ 悟道面板
  │                        ├── 筛选分类
  │                        ├── 选择项目 → 悟道升级
  │                        └── Esc → 主界面
  │
  ├── 点击"异闻录"/按O ─→ 异闻录面板
  │                        ├── 筛选 → 选择条目 → 详情
  │                        ├── 设为追踪/按T切换
  │                        └── Esc → 主界面
  │
  ├── 点击"推演"/按P ──→ 推演面板
  │                        ├── 查看节点统计
  │                        ├── 查看效率分析
  │                        └── Esc → 主界面
  │
  └── 自动触发 ──────────→ 闭关所得弹窗 (阻断其他面板)
                           └── 点击确认 → 主界面
```

## 九、设计优化方向（建议Figma重点关注）

### 9.1 当前已知问题
1. **信息密度过高**：HUD 同时展示战斗卡、追踪卡、今日机缘、装备概览、最近掉落，文字MUD感
2. **层级不清晰**：主要信息和次要信息的视觉权重区分不足
3. **缺少新手引导**：新玩家不知道先看什么、先做什么
4. **缺少动效反馈**：面板切换、按钮点击、状态变化缺少过渡动画
5. **战斗区域被挤压**：信息面板占据了过多的战斗可视区域

### 9.2 建议设计方向
1. **信息渐进披露**：初次打开只显示核心信息，详情通过hover/点击展开
2. **卡片化设计**：将各信息块设计为可折叠的卡片组件
3. **引导式首次体验**：新手前几分钟只显示战斗+基础掉落，随进度逐步解锁UI模块
4. **微交互动效**：面板滑入/滑出、数字跳动、图标呼吸灯、品质光效
5. **明确的视觉焦点**：战斗区域保持60%以上的屏幕占比
