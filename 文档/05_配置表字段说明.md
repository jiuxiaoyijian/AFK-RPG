# 配置表字段说明

## 1. 使用原则

本文件定义项目推荐的数据配置结构，目的是让其他 agent 可以直接按字段生成：

- JSON 配置
- Godot Resource
- 读取器脚本
- 校验逻辑

字段命名原则：

- 使用英文小写加下划线
- 所有主键统一使用 `id`
- 所有引用关系统一使用对方的 `id`
- 布尔字段统一使用 `is_` 或 `can_` 前缀
- 百分比优先使用小数，例如 `0.15` 表示 15%

## 2. 玩家成长配置

文件建议：`data/progression/player_progression.json`

### 2.1 `player_levels`

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 等级 id，如 `lv_1` |
| level | int | 等级数值 |
| exp_required | int | 升到该级所需经验 |
| hp_bonus | int | 生命加成 |
| attack_bonus | int | 攻击加成 |
| defense_bonus | int | 防御加成 |
| unlock_passive_slots | int | 解锁的被动槽位数 |
| unlock_module_slots | int | 解锁的模块槽位数 |
| unlock_features | array[string] | 解锁的功能列表 |

### 2.2 `breakthrough_tiers`

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 阶段 id |
| order | int | 顺序 |
| name | string | 显示名称 |
| required_level | int | 所需账号等级 |
| required_chapter_id | string | 所需章节 |
| required_items | array[object] | 所需物品与数量 |
| hp_multiplier | float | 生命倍率 |
| attack_multiplier | float | 攻击倍率 |
| defense_multiplier | float | 防御倍率 |
| unlock_drop_tiers | array[string] | 解锁掉落品质 |
| unlock_systems | array[string] | 解锁系统 |

`required_items` 子字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| item_id | string | 物品 id |
| count | int | 数量 |

## 16. 配置验收标准

当配置结构达到以下标准时，可认为“可交给程序实现”：

- 所有主要系统都有对应字段
- 所有字段有清晰类型和用途
- 配置之间的引用关系明确
- 可直接转换为 JSON 或 Godot Resource
- 新增一个技能、敌人、装备词条时，不需要重写主流程
