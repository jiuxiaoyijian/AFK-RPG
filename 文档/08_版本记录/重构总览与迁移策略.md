## 一、重构背景

| 维度 | 现状 | 目标 |

| 脚本语言 | GDScript（48 文件 / ~14,000 行） | C#（强类型、命名空间、面向对象） |

## 二、GDScript → C# 迁移策略

### 2.1 总方针：从零重写

不做逐文件翻译，原因：
1. 现有代码大量使用 `Dictionary`/`Array` 弱类型，C# 应定义强类型数据类
2. 信号机制差异大（GDScript `signal` vs C# `[Signal]` attribute 或纯 C# event）
3. `preload()` 不可用，需改为 `ResourceLoader.Load<T>()` / `GD.Load<T>()`
4. 18 份新文档定义了比现有代码更完整的需求，直接按文档实现更干净

### 2.4 信号策略

| GDScript | C# 方案 | 适用场景 |
|----------|---------|---------|
| `signal xxx` + `emit()` | `[Signal] delegate void XxxEventHandler()` + `EmitSignal()` | 需跨语言/编辑器可见的信号 |
| 自定义信号 | 纯 C# `event Action<T>` | 仅 C# 内部使用的事件 |

推荐：EventBus 使用 `[Signal]` 保持 Godot 编辑器兼容；系统间内部通信用 C# event。

### 2.5 数据加载

| GDScript | C# |
|----------|-----|
| `JSON.parse_string(text)` → `Variant` | `System.Text.Json.JsonSerializer.Deserialize<T>(text)` |
| `FileAccess.open(path, READ)` | `Godot.FileAccess.Open(path, ModeFlags.Read)` |
| 弱类型 `Dictionary` | 强类型 `record` / `class` |

### 2.6 Autoload 注册

`project.godot` 中需将所有 autoload 路径从 `.gd` 改为 `.cs`：

```ini
[autoload]
EventBus="*res://scripts/Autoload/EventBus.cs"
ConfigDB="*res://scripts/Autoload/ConfigDB.cs"
; ... 其余同理
```

## 三、像素占位资源策略

### 3.1 生成规格

| 资源类型 | 尺寸 | 风格 | 数量 |
|----------|------|------|------|
| 角色精灵（idle/walk/attack） | 32×32 px | 像素画，4 色调，可辨识姿态 | ~20 |
| 敌人精灵 | 32×32 px | 像素画，红色调区分 | ~10 |
| Boss 立绘 | 64×64 px | 像素画，体型更大 | 3 |
| 图标（道具/技能/资源） | 16×16 px | 像素画，单色+轮廓 | ~74 |
| UI 框/按钮/面板 | 64×64 px (9-patch) | 像素画，深色基调 | ~124 |
| 背景视差层 | 1280×720 px | 像素画，每层2-3色 | ~96 |
| 特效序列帧 | 32×32 px | 像素画，简单闪烁 | ~6 |
| 杂项（头像/占位等） | 32×32~64×64 px | 像素画 | ~5 |

### 3.2 生成工具

`tools/generate_pixel_placeholders.py`（Python + Pillow），功能：
- 读取现有 `assets/generated/` 目录树
- 对每张 PNG 生成同名同路径的像素占位图
- 按尺寸分类自动选择生成模板
- 在图片上标注用途文字（如 "weapon_icon"）

### 3.3 路径保持不变

所有像素占位图覆盖原路径，确保代码中的 `res://assets/generated/...` 引用无需修改。原图归档到 `_legacy/assets/`。

## 六、风险矩阵

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| C# 信号与 GDScript 不兼容 | 高 | 中 | EventBus 用 `[Signal]`，内部用 C# event |
| `preload()` 不可用 | 中 | 确定 | 统一使用 `GD.Load<T>()`，编写加载辅助类 |
| JSON 解析差异 | 中 | 中 | 定义强类型 Model，用 `System.Text.Json` 反序列化 |
| Web 导出不可用 | 高 | 确定 | 已决策放弃 Web，仅桌面端 |
| 重写工作量超预期 | 高 | 中 | 按版本拆分，每版本独立可验收 |
| 像素占位影响测试判断 | 低 | 低 | 占位图标注用途文字，确保功能可测 |
