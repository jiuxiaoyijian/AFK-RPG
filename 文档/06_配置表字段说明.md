# 配置表字段说明

> 本文档定义项目中所有JSON数据文件的字段Schema。
> 所有配置文件存放在 `data/` 目录下。

---

## 一、词缀池 `data/equipment/affixes.json`

### 顶层结构

```json
{
  "affixes": [ ...词缀对象数组... ]
}
```

### 词缀对象字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一标识，如 `"flat_atk"` |
| `name` | string | ✅ | 显示名称，如 `"攻击力"` |
| `bucket` | string | ✅ | 所属乘区bucket，如 `"mainstat"`, `"crit"`, `"speed"`, `"skill_bonus"`, `"debuff"` |
| `type` | string | ✅ | 值类型: `"flat"` (固定值) 或 `"percent"` (百分比) |
| `min_value` | float | ✅ | 值域下限 |
| `max_value` | float | ✅ | 值域上限 |
| `allowed_slots` | array[string] | ❌ | 可出现的槽位列表，空或缺省=全部槽位 |
| `weight` | int | ❌ | 抽取权重，默认100 |
| `tags` | array[string] | ❌ | 标签，如 `["offense"]`, `["defense"]`, `["whirlwind"]` |
| `description` | string | ❌ | 玩家可见描述 |

### 示例

```json
{
  "id": "crit_rate",
  "name": "暴击率",
  "bucket": "crit",
  "type": "percent",
  "min_value": 0.02,
  "max_value": 0.10,
  "allowed_slots": ["gloves", "accessory1", "accessory2"],
  "weight": 100,
  "tags": ["offense"]
}
```

## 二、装备基底 `data/equipment/equipment_bases.json`

### 顶层结构

```json
{
  "bases": [ ...装备基底对象数组... ]
}
```

### 装备基底对象字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一标识，如 `"iron_blade"` |
| `name` | string | ✅ | 显示名称，如 `"铁刀"` |
| `slot` | string | ✅ | 装备槽位: `weapon`/`helmet`/`armor`/`gloves`/`legs`/`boots`/`accessory1`/`accessory2`/`belt` |
| `base_damage` | float | ❌ | 基础伤害（仅兵器有） |
| `base_defense` | float | ❌ | 基础防御（仅防具有） |
| `required_level` | int | ❌ | 最低等级要求 |
| `weapon_type` | string | ❌ | 武器类型（仅兵器有）: `"blade"`, `"axe"`, `"spear"`, `"orb"`, `"staff"` |
| `icon` | string | ❌ | 图标资源路径 |
| `set_id` | string | ❌ | 所属传承ID（传承件专用）: `"set_wind"`, `"set_blood"`, `"set_thunder"`, `"set_iron"` |
| `description` | string | ❌ | 描述 |

## 九、传承定义 `data/sets/set_defs.json`（P1新增）

### 顶层结构

```json
{
  "sets": [ ...传承对象数组... ]
}
```

### 传承对象字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 传承ID: `"set_wind"`, `"set_blood"`, `"set_thunder"`, `"set_iron"` |
| `name` | string | ✅ | 显示名称: `"御风传承"` |
| `description` | string | ✅ | 传承描述 |
| `pieces` | array[string] | ✅ | 包含的装备基底ID列表(6个) |
| `bonuses` | array[object] | ✅ | 件数效果 |

### bonuses 对象字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `required_pieces` | int | ✅ | 需要件数: 2, 4, 6 |
| `effect_type` | string | ✅ | 效果类型 |
| `effect_value` | float | ✅ | 效果数值 |
| `description` | string | ✅ | 效果描述 |

### 示例

```json
{
  "id": "set_wind",
  "name": "御风传承",
  "description": "青云门御风刀法的正统传承",
  "pieces": ["set_wind_weapon", "set_wind_helmet", "set_wind_armor", "set_wind_gloves", "set_wind_legs", "set_wind_boots"],
  "bonuses": [
    { "required_pieces": 2, "effect_type": "skill_range", "effect_value": 0.3, "description": "御风刀范围扩大30%" },
    { "required_pieces": 4, "effect_type": "stack_atk_speed", "effect_value": 0.05, "description": "御风刀命中后获得风势层，每层+5%攻速，最多10层" },
    { "required_pieces": 6, "effect_type": "skill_damage_mult", "effect_value": 4.0, "description": "御风刀伤害+400%" }
  ]
}
```

## 十二、字段命名约定

| 规则 | 示例 |
|------|------|
| ID用英文snake_case | `"flat_atk"`, `"set_wind"` |
| 显示名用中文 | `"攻击力"`, `"御风传承"` |
| 百分比存储为小数 | 5% → `0.05` |
| 布尔值使用true/false | `"is_legendary": true` |
| 枚举值使用小写英文 | `"type": "percent"` |
