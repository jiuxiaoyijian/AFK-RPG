## 协作约定

1. **研发流程优先（强约束）**：新系统/功能必须按 [研发流程总览](00_项目总纲/研发流程总览.md) 的管线阶段推进。每个阶段有门禁，未通过不进入下一阶段。
2. **体验优先（强约束）**：新系统设计**必须**先完成 L1-L5 自检（见 [体验分析框架](00_项目总纲/体验分析框架.md)）。**未通过自检的文档不接受 PR 合入**。
3. **新建文档**：从 `_模板/` 复制对应模板，填写后放入对应目录
4. **命名规则**：使用中文语义名，不使用编号前缀（编号仅用于顶级目录）
5. **大文档拆分**：超过 500 行的文档应拆分为子目录（如 `01_系统设计/装备系统/`）
6. **变更记录**：重大设计变更须同步更新 `00_项目总纲/变更日志.md`
7. **Review 流程**：文档修改通过 PR 提交，至少一人 review 后合并。新系统设计的 review 必须包含**只看 L1-L5 的专项审查**。
