# 每日目标系统 — 开发需求案

> 供实现与联调使用：主脚本 `scripts/autoload/daily_goal_system.gd`，关联 GameManager autoload。引擎 Godot 4，语言 GDScript，项目类型：武侠挂机 RPG（2D 横版）。

---

## §1 文档目的与范围

**目的**：定义「每日目标」（每日任务 / 每日机缘）的玩法规则、数据结构、UI 线框、状态机与验收项，使其他 AI 或工程师可直接据此实现，并与现有系统（推图、秘境、百炼坊、背包、奖励弹窗）对齐。

**范围**：包含每日四条子目标 + 全勤元目标、UTC+8 04:00 重置、奖励领取与全勤判定、连续登录（7 天全勤额外奖励）、面板与弹窗线稿、边界与异常流程。**不包含**：具体美术资源路径、音效命名、营销文案定稿（可在 L1 Fantasy 层替换措辞）。

---

## §2 术语表

| 术语 | 含义 |
|------|------|
| 每日目标 | 当日四条可完成目标 + 一条「全勤」元目标 |
| 节点 | 推图流程中可计数的通关单位（与现有关卡/节点定义一致） |
| 全勤 | 当日四条子目标均已完成且均已领取（或按 §9 定义的统一领取点） |
| 连续天数 streak | 连续多个自然日（按 §3 重置日界）达成全勤的计数 |
| last_reset | 上次已执行日界重置的锚点时间（ISO8601，带 +08:00） |

---

## §3 项目背景与系统定位

- **体裁**：武侠挂机 RPG，2D 横版，Godot 4 + GDScript。
- **定位**：每日目标支撑「约 5 分钟登录也有进展」的日循环，是 L2 日维度驱动的核心模块之一。
- **集成**：`daily_goal_system.gd`（Autoload）与 `GameManager` 协同：进度事件由战斗/秘境/百炼坊等系统上报或通过 GameManager 转发；存档读写与全局暂停策略与项目现有规范一致。

---

## §4 设计理念与数值原则

### 4.1 设计理念

- 任务**简单明确**，奖励**有感但不压迫**；不登录**不产生巨大落差**（无惩罚性累积负债）。
- 覆盖**推图 / 战斗 / 秘境 / 百炼坊**，引导玩家自然触达各子系统。

### 4.2 每日目标列表（策划表）

| 目标 | 条件 | 奖励 |
|------|------|------|
| 每日修行 | 通关任意 3 个节点 | 50 精铁 + 5 秘卷残页 |
| 每日历练 | 击败 30 个敌人 | 1000 碎银 + 100 锻材 |
| 每日秘境 | 完成 1 层秘境 | 1 枚宝石升级石 |
| 每日百炼 | 使用百炼坊任意功能 1 次 | 30 精铁 |
| 全勤奖励 | 完成以上全部 | 1 枚粗铁钥石 + 随机玄品装备 1 件 |

### 4.3 重置与连续奖励规则摘要

- **日界**：每日 **UTC+8 04:00** 重置当日进度与可领状态（细则见 §10、§12）。
- **不累积**：未完成目标不跨日累积；不扣罚。
- **连续登录奖励**：**连续 7 天**达成全勤 → **额外**：真意装备保底 1 件（与全勤基础奖励叠加规则在 §6 明确）。

## §5 L1–L5 体验设计（本模块专述）

| 层级 | 设计要点 |
|------|----------|
| **L1 Fantasy** | 「每日门派修行，弟子日课」——列表即门派功课告示。 |
| **L1 Emotion** | 主情绪：**安心**（每日有保障进展）；次情绪：**完成感**（打勾、领奖）。 |
| **L1 Payoff** | 登录 → 看目标列表 → 完成各项 → 全勤打勾 → 奖励弹窗 → 满足离开。 |
| **L2 时间尺度** | **秒**：条件满足时进度条/数值实时更新；**分**：目标组合支持约 **5 分钟**内全部完成的路径；**时**：与日常游玩自然穿插；**日**：每日重置；**周**：7 日连续全勤额外仪式奖励。 |
| **L3 系统覆盖** | 推图节点、击杀、秘境层数、百炼坊交互四类来源，避免单一玩法刷满一日。 |
| **L4 学习曲线** | **1 分钟**读懂列表与奖励；**约 1 小时**内可完成当日全部（含穿插玩法）；**3 天**适应节奏；**7 天**形成连续全勤动机。 |
| **L5 仪式感** | 全勤达成：**打勾动画** + **奖励弹窗** + **7 天连续额外奖励**的简短仪式（可共用「里程碑」动效框架）。 |

## §7 UI 线框图（ASCII）

### 7.1 每日目标主面板

**方案说明**：顶部固定标题区；中部为可滚动目标列表，每条含图标位、文案、进度条与「领取」按钮态；底部固定全勤摘要条 + 连续天数徽章，避免被键盘或安全区遮挡。

```
+------------------------------------------------------------------+
|  [X]                        每日机缘                              |
+------------------------------------------------------------------+
|                                                                  |
|  +------------------------------------------------------------+  |
|  |  [icon] 每日修行    通关任意 3 个节点                         |  |
|  |         进度  [=========>        ] 2 / 3                    |  |
|  |                                    [ 进行中 ]               |  |
|  +------------------------------------------------------------+  |
|  |  [icon] 每日历练    击败 30 个敌人                            |  |
|  |         进度  [==================] 30 / 30                 |  |
|  |                                    [ 可领取 ]               |  |
|  +------------------------------------------------------------+  |
|  |  [icon] 每日秘境    完成 1 层秘境                             |  |
|  |         进度  [====>              ] 0 / 1                    |  |
|  |                                    [ 进行中 ]               |  |
|  +------------------------------------------------------------+  |
|  |  [icon] 每日百炼    百炼坊任意功能 1 次                      |  |
|  |         进度  [==================] 1 / 1                   |  |
|  |                                    [ 已领取 ]               |  |
|  +------------------------------------------------------------+  |
|                                                                  |
+------------------------------------------------------------------+
|  全勤奖励：粗铁钥石 x1 · 随机玄品装备 x1                          |
|  状态：[ 未达成 ][ 可领取 ][ 已领取 ]     连续全勤：●●○○ 3 天     |
+------------------------------------------------------------------+
```

**边界补全**：列表项在「已领取」后进度条仍可显示满条以便回顾；全勤条在四条未齐时显示灰色遮罩 + 文案「完成全部日课解锁」。

### 7.2 奖励领取弹窗（单条或合并领取）

**方案说明**：弹窗居中模态；展示奖励图标槽位与数量；全勤或 7 日额外奖励可增加第二行「额外」标签；按钮区主按钮确认、次按钮关闭。

```
+---------------------------+
|        获得奖励            |
+---------------------------+
|  [icon] 精铁        x50   |
|  [icon] 秘卷残页     x5   |
+---------------------------+
|   [额外] 真意装备保底 x1   |  <-- 仅 streak 里程碑触发时显示
+---------------------------+
|     [ 确定 ]   [ 关闭 ]    |
+---------------------------+
```

**边界补全**：若一次合并发放多种货币 + 装备，列表超过 4 行时中部区域可滚动；装备名过长用省略号，完整名在 Tooltip 或详情页展示（与项目现有物品 UI 一致）。

## §8 数据结构（存档 / 传输 JSON 形态）

以下为逻辑形态示例；实际键名可与本地化 id 对齐，但字段语义须一致。

```json
{
  "daily_goals": [
    {"id": "daily_train", "desc": "通关3节点", "target": 3, "progress": 0, "reward": {"精铁": 50, "秘卷残页": 5}},
    {"id": "daily_fight", "desc": "击败30敌人", "target": 30, "progress": 0, "reward": {"碎银": 1000, "锻材": 100}},
    {"id": "daily_rift", "desc": "完成1层秘境", "target": 1, "progress": 0, "reward": {"宝石升级石": 1}},
    {"id": "daily_forge", "desc": "百炼坊使用1次", "target": 1, "progress": 0, "reward": {"精铁": 30}},
    {"id": "daily_all", "desc": "完成全部目标", "type": "meta", "reward": {"粗铁钥石": 1, "随机玄品": 1}}
  ],
  "streak": {"current": 0, "best": 0},
  "last_reset": "2026-03-19T04:00:00+08:00"
}
```

**实现须补充的字段（建议）**：

- 每条子目标：`claimed: bool` 或 `claimed_at`（防重复领）。
- 全勤元目标：同上。
- `streak_last_full_day`：上次计入 streak 的「日界日期」（YYYY-MM-DD，Asia/Shanghai）用于防重复 +1。
- `pending_streak_bonus`：若有「可领未领」的 7 日额外奖励，单独队列，避免与 §10 过期策略冲突。

## §9 核心操作流（状态机）

### 9.1 完成目标 → 领取 → 全勤检查 → 连续天数 +1

**推荐主流程（可领取粒度为「每条子目标 + 全勤条」）**：

1. **事件入口**：推图通关节点 +1；击杀敌人 +1；秘境完成层数 +1；百炼坊任意功能调用 +1（各来源须幂等或可去重，见 §12）。
2. **进度更新**：`progress = min(progress + delta, target)`；刷新 UI；若 `progress >= target` 且未领取，按钮切「可领取」。
3. **领取单条**：玩家点击「可领取」→ 校验进度与未领状态 → 发放该条 `reward` → 标记 `claimed` → 播放轻量特效。
4. **全勤检查**：若四条子目标均为 `claimed == true`（或等价：已完成且已领），则全勤元目标置为「可领取」。
5. **领取全勤**：发放 `daily_all.reward` → 标记全勤已领。
6. **连续天数 +1**：在**全勤达成且全勤奖励领取完成**的同一事务末尾：若本「日界日期」尚未计入 streak，则 `streak.current += 1`，更新 `best = max(best, current)`，写入 `streak_last_full_day`；若 `streak.current % 7 == 0`（或第 7、14…次，按产品选定的「每满 7 天」规则），追加发放「真意装备保底」并弹出 §7.2 额外行。

**方案说明**：streak 的 +1 **绑定在全勤奖励领取点**可避免玩家「完成但未领」导致 streak 与策划预期不一致；若产品要求「完成即算全勤天数」，须在 §10 与存档字段中改为「达成即写 streak_last」，并处理未领跨日过期的一致性。

### 9.2 简状态图（文字）

`进行中 → 可领取 → 已领取`（每条独立）；全勤条依赖四条 `已领取` 才 `可领取 → 已领取`；streak 在「全勤已领取」事件沿用上文规则。

## §10 边界与异常场景

| 场景 | 处理方案 |
|------|----------|
| **跨时区 / 旅行** | 一切以 **Asia/Shanghai（UTC+8）** 计算「日界日期」：`last_reset` 与当前时刻比较，若跨过下一个 04:00 锚点则触发重置。客户端仅展示本地时间时，仍用服务器或统一时区库换算，避免设备时区篡改刷日。 |
| **离线期间进度** | 离线时照常累加「可验证」的计数（若项目离线结算推图/击杀）：在登录合并结算后一次性加进度；**不得**超过当日 target；若合并后跨日界，先按 §12 做重置再写入新日进度。 |
| **未领取奖励过期** | 日界重置时：**未领取的「当日可领奖励」作废**（不邮件补发，除非全局统一「补领邮箱」规则）；streak 若采用「领取才算」则未领全勤不算 streak +1。须在 UI 用「今日即将重置」弱提示（可选 P1）。 |
| **连续天数中断** | 任一「日界日期」未达成全勤（或未完成领取，视 §9 选定规则），则 `streak.current` 归零或降至基础值；`best` 保留历史最高。 |
| **首次使用引导** | 首次打开面板：遮罩 + 指向列表 + 单行文案；记录 `daily_goal_intro_seen`；设置里可重播。 |

**边界补全**：作弊改系统时间：以服务端时间或加密戳为准；纯单机则采用「单调时钟 + 上次在线时间」检测异常回拨，回拨不触发重复重置或重复领奖。

## §11 与其他系统的接口（事件契约）

| 事件源 | 事件名（示例） | 载荷 | 每日目标行为 |
|--------|----------------|------|----------------|
| 推图 | `node_cleared` | 节点 id、是否胜利 | `daily_train` +1（仅胜利且计数节点） |
| 战斗 | `enemy_defeated` | 敌人 id、归属击杀者 | `daily_fight` +1（过滤召唤物/环境击杀规则与项目一致） |
| 秘境 | `rift_floor_completed` | 层数 | `daily_rift` 按「完成层」+1 |
| 百炼坊 | `forge_action_used` | 功能枚举任意 | `daily_forge` +1（同日内多次仅第一次计数，若策划改为「每次」需改 target） |

**方案说明**：`daily_goal_system` 订阅上述信号或由 GameManager 统一派发；所有增量接口须 **idempotent key**（如战斗实例 id）可选，防止双发。

## §12 验收标准（测试要点）

### 12.1 重置正确性

- 在 03:59 → 04:01（+8）跨越时，`last_reset` 更新为新锚点，`progress` 与 `claimed` 清空； streak 按 §10 判断是否中断。
- 快速登录登出多次，重置仅执行一次。

### 12.2 进度统计

- 各事件在达到 target 后不再增加；重复事件不突破上限。
- 离线合并与在线实时加总结果一致。

### 12.3 全勤判定

- 仅四条子目标均达 target 且领取完毕 → 全勤可领。
- 任一条未领跨日 → 全勤不可领且当日全勤失败（与 §10 一致）。

### 12.4 连续天数

- 连续 7 天全勤（且领取）：触发「真意装备保底」且次数可重复（每满 7 天一次，或仅首次，由策划在配置表锁定）。
- 中断后 current 归零，best 不变。

## §13 配置与本地化

- 所有 `desc`、奖励展示名走本地化 key；数值来自策划表，支持热更 JSON。
