# 装备与Build系统 — 开发需求案（§1–§15）

> 武侠挂机 RPG，2D 横版，Godot 4，GDScript。本文档供实现与联调直接使用；与 L1–L5 体验目标一致。

---

## §1 文档目的与范围

**目的**：定义装备生成、九槽位、七品质、七大乘区词缀桶（Bucket）互斥、多乘区伤害结算、装备 UI、武学诊断与换装闭环，使 `equipment_generator_system.gd` 与 `data/equipment/*.json`、掉落表、战斗 DPS 一致可验证。

**范围内**：装备实例数据结构、随机生成管线、面板与弹窗信息架构、诊断推荐文案规则、与关卡 ilvl 的绑定、与三流派 Build 的接口约定。

**范围外**：具体美术资源命名流水线、音效文件清单（仅引用 L5「六步音画序列」完成态）、非装备类养成（心法等级表等若未在本项目装备域内则不在此展开）。

## §2 术语表

| 术语 | 定义 |
|------|------|
| ilvl | 装备等级，默认等于当前关卡等级。 |
| 部位 / 槽位 | 兵器、护额、护甲、护手、腿甲、轻靴、佩饰×2、腰封，共 9 槽。 |
| 品质 | 凡品→精良→玄品→真意→传承→上古真意→绝世真意（7 级）。 |
| 词缀 | 基础外的随机属性条目；每条归属唯一 Bucket。 |
| Bucket | 乘区词缀池分类；同一件装备同一 Bucket 最多 1 条。 |
| 传承 / 套装 | `bucket_set`，仅传承件定义，不可作为随机池条目出现。 |
| 传奇特效 | `bucket_legendary`，真意及以上品质可出现（见 §6）。 |
| Build 三要素 | 流派选择、传承凑件、乘区均衡（见 §13、§11）。 |

---

## §3 系统目标与非目标

**目标**

1. 装备是 Build 核心：多乘区公式引导玩家跨桶投资，避免单桶堆满最优。
2. 生成可复现、可测试：ilvl→品质→基础→词缀数→按桶无放回抽取→数值 roll。
3. 玩家可理解：武学诊断输出最弱乘区与替换方向，与 UI 线框一致。

**非目标**

1. 不在本文档内规定具体技能 ID 全表（仅约定「御风 / 血劫 / 五雷」类技能增幅词缀形态）。
2. 不强制实现交易行或玩家间装备交换。

## §5 九槽位与词缀数量

| 部位 | 可出词缀数 | 说明 |
|------|------------|------|
| 兵器 | 4–6 | 决定基础伤害 |
| 护额 | 3–5 | CDR / 资源 |
| 护甲 | 3–5 | 防御为主 |
| 护手 | 3–5 | 暴击 / 攻速 |
| 腿甲 | 3–5 | 混合 |
| 轻靴 | 3–5 | 移速 / 抗性 |
| 佩饰 ×2 | 3–4 | 宝石槽 + 词缀 |
| 腰封 | 3–4 | 特殊机制类 |

**品质对词缀数**：具体条数区间由品质档位在生成器中映射（与掉落表给出的品质一致）；佩饰若含「宝石槽」占位，需在数据层标明是否消耗一条词缀额度或独立孔位（实现二选一须在 §15 锁定）。

**Bucket 互斥（全局规则）**：同一件装备上，同一 Bucket 最多 1 条词缀；抽取时按 Bucket 无放回。

## §6 七级品质与特效规则

顺序：**凡品(灰) → 精良(蓝) → 玄品(黄) → 真意(橙) → 传承(绿) → 上古真意(暗金) → 绝世真意(红)**。

| 品质 | 规则摘要 |
|------|-----------|
| 上古真意 | 词缀数值范围 **+30%**（在 affix 的 min/max 上按比例放大，或等价查表列；需与存档兼容）。 |
| 绝世真意 | **所有词缀取该条 max**（不再 roll 区间内随机值）。 |
| 真意+ | **传奇特效**（`bucket_legendary`）：仅真意及以上可出现；橙字展示，与 `legendary_affixes.json` 对齐。 |
| 传承 | **套装**（`bucket_set`）：仅传承件配置，**不可**进入随机 affix 池。 |

颜色与本地化键名与 UI 品质标识一致（HUD / 弹窗 / 飘字）。

### §6.2 七大乘区词缀池（Bucket 结构）

**互斥规则**：同一件装备上，同一 Bucket 最多 1 条词缀（与 §5 全局规则一致）。

| 乘区 | Bucket ID | 词缀示例 |
|------|-------------|----------|
| A 主属性 | `bucket_primary` | 力道 +X，灵慧 +X，根骨 +X |
| B 武技增幅 | `bucket_skill` | 御风刀伤害 +15–30%，血劫手 +15–30%，五雷掌 +15–30% |
| C 暴击 | `bucket_crit` | 会心率 1–10%，会心伤害 10–100% |
| D 速度 | `bucket_speed` | 攻速 +3–7%，CDR +3–8% |
| E 套装 | `bucket_set` | 传承件专属（**不可随机**） |
| F 传奇特效 | `bucket_legendary` | 橙字专属特效（真意+品质） |
| G 目标增伤 | `bucket_target` | 精英增伤 +10–20%，Boss 增伤 +10–20% |

## §8 数据模型（实现约束）

**装备实例（逻辑字段，可与存档 JSON 对齐）**

- `instance_id`, `base_id`（`equipment_bases`）, `slot`, `ilvl`, `rarity`
- `rolled_base`（武器 DPS 区间 roll 结果等）
- `affixes[]`：每项含 `affix_id`, `bucket`, `rolled_value`, `is_legendary`, `is_set`
- `locked`（玩家锁定防误拆）

**affixes.json**：每条必须含 `bucket`、`slot_tags`（可装备部位）、`weights_by_ilvl` 或等价权重、`min`/`max`、`roll_type`（flat / percent）。

**legendary_affixes.json**：仅 `bucket_legendary`；绑定 `min_rarity = 真意`。

**校验**：加载时断言 `bucket_set` 不在随机 affix 列表；传承 base 自动注入套装词条而非随机。

## §9 生成管线与换装操作流

### §9.1 装备生成管线（六步）

1. **确定 ilvl** = 关卡等级（或关卡配置表中的装备等级）。
2. **确定品质**：查掉落表 / 保底规则（与 L2 节奏一致，见 §11）。
3. **确定基础属性**：查 `equipment_bases`（按 ilvl 缩放曲线）。
4. **随机词缀数量**：由品质 + 部位上限决定（§5）。
5. **逐条抽词缀**：Bucket 互斥 + 同 Bucket 无放回；按权重抽取；若候选池耗尽则降级为「通用主属性/防御」安全池（触发条件见 §10 第 1 条）。
6. **每条 roll 值**：普通区间随机；上古真意 +30%；绝世真意取 max。

### §9.2 换装操作流（从掉落到 DPS 变化）

1. **掉落**：敌人死亡 → 掉落表决定品质与部位池 → 调用生成器产出实例 → 地面光柱 / L5 序列（已完成则播放）。
2. **拾取**：玩家触碰或自动入包 → 背包格子 +1；若已满进入 §10「背包满载」分支。
3. **对比**：打开装备或详情弹窗 → 读取已穿戴同部位 → 对比栏用与战斗相同的聚合器做「单件边际贡献」估算（或完整重算差分，二选一须性能评估后固定）。
4. **穿戴**：校验部位 → 卸下旧件入包或指定空位 → 写角色装备槽 → 发送 `equipment_changed`。
5. **DPS 变化**：战斗与 HUD 订阅事件 → 重算 §11 公式 → 更新 DPS 数字与训练木桩/下次命中预览（若有）。

**验收**：同一件装备穿戴前后，日志中各乘区因子与最终伤害一致可复现。

## §10 边界与异常（8 类，每类 1–2 句方案）

1. **Bucket 冲突**：生成与重铸时维护 `used_buckets: Set`；新抽 affix 前过滤同 bucket；若过滤后池空，使用预置「安全 affix」（仅 `bucket_primary` 或部位白名单）保证条数达标。  
2. **背包满载**：拾取时若满格，弹窗二选一「丢弃新件 / 快速拆解旧件」；拆解规则引用项目统一回收表，避免静默吞装备。  
3. **首次穿装引导**：首次获得第一件可装备时，软引导高亮槽位 + 打开详情一次；完成后写 `tutorial_first_equip_done`。  
4. **部位不匹配**：穿戴按钮灰显；Tooltip 说明「需护手槽」；禁止通过 API 强行写入错误槽。  
5. **ilvl 与表缺失**：若 `equipment_bases` 无该 ilvl 行，钳制到最近可用 ilvl 并打 `push_warning` 日志，便于数据修补。  
6. **绝世真意 + 传奇**：仍 roll 传奇条目种类，但数值全 max；若传奇含多段效果，每段子字段均取配置 max。  
7. **套装件数变化**：穿脱导致套装层数变化时，立即重算 `bucket_set` 加成并广播 UI；诊断栏提示「再获 1 件传承激活 4 件套」。  
8. **存档兼容 / 热更**：旧存档缺 `bucket` 字段时，启动迁移：按 `affix_id` 反查表补 bucket；无法识别则标记为 legacy 并不参与互斥（仅展示），鼓励玩家替换。

## §11 七层多乘区伤害公式与 L2 节奏对齐

**公式（与实现一致）**

```
最终伤害 = 武器基础
         × (1 + 主属性/100)
         × (1 + 会心率 × 会心伤害)
         × (1 + 技能伤害%/100)
         × (1 + 元素伤害%/100)
         × (1 + 套装特效%/100)
         × (1 + 传奇特效%/100)
         × (1 + 精英增伤%/100)
```

**设计意义**：同乘区内加法（递减），跨乘区乘法（爆炸增长），引导均衡搭配；武学诊断按因子对总伤的弹性分解最弱项。

**L2（已完成目标，本系统需对齐）**

- Fantasy：继承前辈馈赠；Emotion：期待；Payoff：金柱→绝世真意→DPS 跳字。
- 节奏：1 min 凡品 → 5 min 蓝 → 15 min 黄 → 首章 Boss（90–150 min）保底橙 → 日循环约 20 min 期望 1 橙 → 周 3 天 2 件套 → 月 14 天满 6 件。

**对齐方式**：掉落表品质权重与保底计数器驱动「橙/传承」产出节奏；诊断在橙后强调「多乘区」而非再堆单一攻击；DPS 跳字读取公式最终结果。

## §12 验证与测试

| 编号 | 验证项 | 方法 |
|------|--------|------|
| T1 | 装备生成正确性 | 固定 RNG seed，断言 ilvl、品质、base、affix 条数与每条 bucket 唯一。 |
| T2 | Bucket 互斥 | 压力随机 1e4 次生成，统计同件同 bucket 重复率必须为 0。 |
| T3 | 品质数值规则 | 上古真意样例：roll 值 ∈ [floor(min×1.3), ceil(max×1.3)]；绝世：全等于 max。 |
| T4 | DPS 计算 | 给定面板向量，手算公式与 `DamageCalculator`（或等价单测）误差 0。 |
| T5 | 换装事件 | 穿脱前后两次聚合面板快照 diff 与 UI 一致。 |
| T6 | 套装层数 | 2/4/6 件边界切换时 `bucket_set` 因子突变正确。 |

**自动化**：优先 GDScript 单元测试 + 可选场景集成测试（拾取 mock 掉落）。

## §13 Build 三要素与武学诊断

1. **流派选择**：御风 / 血劫 / 五雷 —— 影响 `bucket_skill` 权重与诊断文案（「当前技能乘区未匹配流派」）。
2. **传承凑件**：6 件定义 Build；诊断显示件数与下一阈值收益。
3. **乘区均衡**：自动分析当前装备对各乘区贡献，输出「当前最弱乘区是 C 暴击」及「建议替换护手/佩饰补会心」类具体部位指向规则（规则表可配置）。

**诊断输出字段**：`weakest_bucket`, `second_weakest`, `recommended_slots[]`, `one_line_summary`。

## §15 开放问题、风险与附录

**开放问题**

- 佩饰「宝石槽」是否占用词缀条数（§5）须在数据层单一结论。  
- 元素伤害% 来源若不仅来自装备，需列明其它系统写入顺序与上限。

**风险**

- 对比栏「边际贡献」若近似不当会误导；默认采用全量重算差分更稳妥。  
- 绝世真意全 max 可能与 PVP 或未来压缩数值冲突，预留上限钳制。

**附录 A — Build 与体验（L1/L3/L4/L5 摘要）**

- L1：已定义 Fantasy/Emotion/Payoff。  
- L3：三流派 + 多乘区均衡。  
- L4：5 min 品质区分 → 1 h 感受 bucket 互斥 → 10 h 主动筛词缀 → 100 h 理解多乘区。  
- L5：橙装落地 6 步音画序列（与拾取/鉴定事件绑定）。

**附录 B — 公式变量与面板映射（实现核对用）**

| 因子 | 面板来源 |
|------|-----------|
| 武器基础 | 兵器 rolled_base |
| 主属性 | 全装备聚合力道/灵慧/根骨（流派主属性优先显示） |
| 会心率、会心伤害 | `bucket_crit` 聚合 |
| 技能伤害% | `bucket_skill` + 非装备加成 |
| 元素伤害% | 元素类 affix / 其它系统 |
| 套装%、传奇% | `bucket_set`, `bucket_legendary` |
| 精英增伤% | `bucket_target`（Boss 关切换 Boss 子项） |
