# 武学秘录系统 — 开发需求案（§1–§15）

> 本文档面向实现侧：可直接作为 Godot 4 / GDScript 功能开发的单一事实来源。主逻辑脚本：`scripts/systems/martial_codex_system.gd`；与掉落/图鉴追踪相关的 autoload：`LootCodexSystem`（`scripts/autoload/loot_codex_system.gd`）。百炼坊 UI 中已有「武学秘录」页签入口（`cube_panel_controller.gd` 等）。

---

## §1 项目背景与定位

- **项目类型**：武侠题材挂机 RPG，2D 横版，引擎 Godot 4，脚本语言 GDScript。
- **武学秘录定位**：对标《暗黑破坏神 3》卡奈魔盒的「传奇特效栏」——玩家通过百炼坊的**萃取**功能，将橙装上的**可萃取传奇特效**永久写入秘录，再在三个固定槽位中各选一项激活，使角色获得与「装备携带该传奇词缀」等价的战斗加成。
- **与现有代码关系**：数值合并路径上已有 `MartialCodexSystem.build_combat_bonuses` 与 `BuildQueryService` 的集成雏形；萃取、解锁、三槽激活的完整产品化（仪式动画、图鉴进度、边界文案、存档字段）以本文档为准逐步补齐。

---

## §2 设计目标（Goals）

1. **长期收集**：萃取一次永久解锁对应传奇特效，形成可展示的「武学图谱」资产。
2. **策略组合**：兵器 / 护甲 / 饰品三栏独立决策，在已解锁集合内免费切换，鼓励针对不同关卡调整 Build。
3. **规则清晰**：激活特效在规则上**等价于**装备上的传奇词缀效果；与身上装备叠加时，叠加规则可预期、可测试、可解释。
4. **节奏友好**：萃取为「分时」操作（约 1 分钟级），切换为「秒级」即时反馈；日常推图持续产出萃取所需原料与橙装来源。
5. **可扩展**：每个流派规划约 5–8 个可萃取传奇特效（数据驱动，来自装备系统的 `bucket_legendary` / 传奇词缀配置）。

---

## §3 范围（In Scope / Out of Scope）

### In Scope

- 百炼坊内：萃取入口、放入装备、校验、消耗（**仅消耗源装备**，不额外消耗货币/材料，除非产品后续单列「萃取石」类资源——见 §14）。
- 武学秘录面板：三槽展示、已解锁列表、当前激活标记、切换按钮。
- 萃取成功弹窗：特效名、描述、动画占位、跳转秘录。
- 图鉴收集进度：已解锁数 / 可萃取总数、按流派（或 Build 流派标签）分组统计。
- 存档：`unlocked_effect_ids`、`active_slots`（与 `martial_codex_system.gd` 中 `create_default_state` 一致）及必要的 UI 状态持久化约定。
- 与 `LootCodexSystem` 的协同：传奇发现、追踪推荐农场节点等**可选增强**（不阻塞秘录核心闭环）。

### Out of Scope（本需求案不强制实现）

- 秘录特效的 PVP 平衡专项（沿用 PVE 数值管线即可）。
- 跨账号云同步（若单机存档则仅本地）。
- 全量剧情演出（仅要求仪式级占位与扩展点）。

## §4 名词与定义

| 名词 | 定义 |
|------|------|
| **传奇特效 / 武学效果** | 配置在传奇词缀条目上的唯一效果，对应 `legendary_affix_id`（与 `MartialCodexSystem.get_effect_definition` 数据源一致）。 |
| **萃取** | 将指定橙装（或带可萃取传奇槽位的装备）销毁，把其上绑定的传奇特效**永久解锁**到秘录。除源装备外默认不消耗其他资源。 |
| **槽位 bucket** | `weapon` / `armor` / `accessory` 三栏；与 `SLOT_GROUPS` 中装备部位标签映射一致。 |
| **激活** | 在某个 bucket 内从已解锁列表中选定 0 或 1 个特效生效；同一 bucket 同时最多 1 个激活。 |
| **叠加** | 秘录激活提供的词缀效果与角色当前穿戴装备上的同类传奇效果按 §10 规则合并。 |

---

## §5 系统依赖与关联

- **配置**：`ConfigDB.get_all_legendary_affixes()` — 每条需具备 `id`、`slot_tags`、展示名与描述字段（及战斗加成解析所需结构）。
- **核心逻辑**：`MartialCodexSystem` — `unlock_effect`、`set_active_effect`、`sanitize_state`、`build_combat_bonuses`、`build_runtime_state`。
- **全局状态**：`GameManager`（或等价单例）持有 `martial_codex_state`，并提供 `get_martial_codex_runtime_state`、`activate_martial_codex_effect` 等 UI 调用封装。
- **事件**：`EventBus.martial_codex_changed` — HUD、背包、Cube 面板等订阅刷新。
- **百炼坊 UI**：`cube_panel_controller.gd`、`cube_view_model_service.gd` 中 `page_id == "martial_codex"` 分支。
- **Loot 图鉴**：`LootCodexSystem` — 与「发现传奇」追踪互补；秘录解锁**不等于**图鉴发现时可并行存在，需在文案上区分「已见过」与「已萃取解锁」。

## §6 数据模型与配置约定

### 6.1 存档状态（与实现对齐）

```gdscript
{
  "unlocked_effect_ids": [],       # Array[String]，去重有序均可，sanitize 时去重
  "active_slots": {
    "weapon": "",
    "armor": "",
    "accessory": "",
  }
}
```

### 6.2 可萃取标记

- 每条传奇词缀配置需可判断是否允许进入秘录萃取池（建议字段如 `martial_codex_extractable: bool`，或约定仅 `bucket_legendary` 且带特定 tag 的条目可萃取）。
- **槽位归属**：由 `slot_tags` 与 `SLOT_GROUPS` 映射决定；无法映射的条目不得出现在萃取结果中（实现应拒绝并给明确 `reason`）。

### 6.3 示例特效（配置层描述，非穷尽）

以下与装备系统 `bucket_legendary` 中条目对应，供策划填表与验收对齐：

- **风眼**：御风刀旋转时间延长 50%。
- **血诏**：流血层数上限 +3。
- **雷暴矩阵**：五雷掌连锁弹射 +2。
- **铁壁心法**：格挡时回复 5% 最大生命。
- 每个流派约 **5–8** 个可萃取传奇特效，由数据表驱动。

## §8 核心规则（产品层）

1. **萃取后永久解锁**：同一 `effect_id` 解锁一次即可，不消耗源装备以外资源（默认）。
2. **三槽各选其一**：`weapon` / `armor` / `accessory` 各最多 1 个激活；允许某槽为空（`effect_id == ""`）。
3. **效果等价**：激活特效在战斗计算中与「身上某件装备携带该传奇词缀」使用同一套解析与加成管线。
4. **免费切换**：仅允许在**已解锁**且 **bucket 匹配** 的特效间切换；无货币消耗。
5. **与装备叠加**：秘录激活效果默认**叠加**在装备传奇效果之上；冲突与上限见 §10。

## §9 萃取操作流（端到端）

**目标**：从获得橙装到秘录中激活一条闭环清晰、可断点续传。

1. **获得橙装**：战斗/掉落获得带可萃取传奇词缀的装备（背包可见传奇条目）。
2. **打开百炼坊**：主城或 HUD 进入百炼坊界面。
3. **选择萃取页**：在百炼坊 Tab 中选择「萃取」（与「武学秘录」页签区分：萃取是消耗装备的操作页）。
4. **放入装备**：将目标橙装拖入萃取槽；系统校验：是否可萃取、`effect_id` 是否有效、是否已解锁。
5. **确认萃取**：玩家确认后，**删除源装备**，调用 `unlock_effect`；失败则保留装备并弹出 `reason`。
6. **秘录解锁反馈**：播放 §7.2 弹窗（翻页动画占位 + 金字名称 + 描述）。
7. **跳转秘录面板**：玩家点「前往武学秘录」则切换 Cube 的 `martial_codex` 页或打开独立秘录面板。
8. **激活**：在对应 bucket 列表中点击「激活」；调用 `set_active_effect`；`EventBus.martial_codex_changed` 触发全局刷新。

**方案说明**：步骤 6–7 必须可跳过（留在百炼坊），避免打断连续萃取节奏；步骤 8 可在首次萃取后高亮推荐槽位一次（新手引导可选）。

## §10 边界情况与冲突处理

### 10.1 已解锁条目再次萃取

- **行为**：若玩家对另一件带相同 `legendary_affix_id` 的装备再次萃取，`unlock_effect` 返回 `ok: false`，`reason: "该武学已解锁"`（与现有逻辑一致）。
- **UX**：百炼坊确认前即灰显禁止按钮，或在确认时 Toast 提示，**不消耗装备**。

### 10.2 三栏已满时的「切换」

- **语义**：每栏始终最多 1 个激活；「已满」指三栏均有非空激活。切换某一栏时只需在同一栏内改选另一已解锁特效，**无需先卸下**（若需空槽，提供「停用」将 `effect_id` 置空）。
- **实现**：`set_active_effect(slot_id, new_id)` 直接覆盖该槽；其他槽不变。

### 10.3 跨系统效果叠加与冲突

- **默认**：秘录提供的加成与装备上的同类加成进入同一合并函数（如 `build_combat_bonuses` 与装备 bonus merge）。
- **冲突策略（需实现一致）**：
  - **可叠加数值**：加法叠加（如 +50% 与 +30% 旋转时间 → 按策划表定义是加法还是乘法；文档建议**同类乘区统一在配置层声明**，代码只执行表驱动规则）。
  - **唯一行为类**（如「同一技能仅触发一次特殊机制」）：以 **装备优先** 或 **取高不叠加** 二选一，必须在配置 `stacking_rule` 字段声明；无声明时默认「可数值叠加、唯一机制不重复触发」。
- **校验**：`sanitize_state` 在加载存档时剔除「未解锁却激活」或「bucket 不匹配」的脏数据，静默修正并打日志。

### 10.4 收集进度持久化

- **解锁列表**、**三槽激活**随角色存档读写；热更配置新增特效 ID 时，总分子应取「当前版本配置中可萃取全集」，避免旧存档显示百分比异常（建议在进度面板显示 `40 (+2 未实装)` 或仅统计已存在配置 ID）。

## §11 体验层级 L1–L5（新编）

| 层级 | 内容 |
|------|------|
| **L1 Fantasy** | 传承前辈绝学，不朽的武学图谱在手中展开。 |
| **L1 Emotion** | 主情绪：收集与成就；次情绪：策略组合与「套装外第三维度」掌控感。 |
| **L1 Payoff** | 萃取 → 秘录翻开动画 → 新特效金字写入 → 三栏重新审视 → 切换激活 → DPS / 生存指标变化可感知。 |
| **L2 时间尺度** | 秒：切换即时生效；分：单次萃取操作约 1 分钟；时：每 1–2 小时可能获得一次新萃取机会；日：日常推图保障原料与橙装；周：目标解锁当前流派秘录大部；月：跨流派全收集。 |
| **L3 策略空间** | 三栏 × 每栏多候选 = 明确有限的离散组合空间，便于平衡与攻略传播。 |
| **L4 上手曲线** | 约 5 分钟认识秘录 UI；约 1 小时完成首次萃取并激活；约 10 小时理解本流派特效协同；约 100 小时追求全图鉴与跨流派优化。 |
| **L5 仪式感与奖励** | 萃取成功仪式（翻页 + 金字）；图鉴阶段奖励（称号、头像框、秘录皮肤等择一实现，占位即可）。 |

## §12 测试与验收标准（QA）

1. **萃取正确性**：可萃取装成功解锁且装备删除；不可萃取装拒绝且不删除；重复解锁拒绝且不删除。
2. **激活 / 切换生效**：切换后 `build_runtime_state` 与战斗内属性同步；HUD / 详情面板与 `EventBus` 刷新一致。
3. **叠加计算**：双来源同类属性按 §10.3 规则与配置一致；回归用例至少覆盖：双数值叠加、唯一机制不重复、取高规则各 1 例。
4. **收集进度**：总进度与分组计数与 `unlocked_effect_ids` 及配置表一致；新档、读档、脏数据 `sanitize` 后 UI 不崩溃。
5. **存档往返**：存档 → 清内存 → 读档后解锁与激活保持一致。

## §13 音频与表现（占位）

- 萃取成功：翻页声 + 短促金石撞击；金字写入时 UI 缩放 pulse 0.2s。

## §14 待定与决策点

- 是否引入**额外消耗品**（萃取石）以调控经济；当前文档默认 **仅消耗装备**。
- 唯一机制类叠加全局策略（装备优先 vs 取高）需策划单表裁定。
- 图鉴分组「流派」与配置字段的绑定键名（`school_id` / `build_tag`）。
