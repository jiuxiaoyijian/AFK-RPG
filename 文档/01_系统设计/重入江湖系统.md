# 重入江湖系统 — 开发需求案

> 本文档为完整实现规格，供 AI 或开发者直接落地。主脚本：`scripts/systems/season_system.gd`；持久化与存档：`SaveManager` autoload。引擎：Godot 4，语言：GDScript，品类：武侠挂机 RPG（2D 横版）。

---

## §1 目标与范围

### 1.1 产品目标

- **重入江湖**对标暗黑 3 的赛季式轮回：玩家在满足条件后可主动触发「轮回」，获得跨存档的**永久被动加成（江湖阅历）**，形成「放下进度、越来越强」的长期循环。
- 与**赛季**概念对齐：赛季周期约 3 个月；赛季结束时的合服、排行榜定格、专属奖励发放与本文档的「轮回」数据模型兼容（赛季 ID、赛季起止时间写入存档）。

### 1.2 范围内（Must）

- 触发条件校验、重入确认 UI、执行重置、江湖阅历累加与展示。
- **前世宝库**：跨轮回永久保存，容量与优先级规则、存取流程。
- 与 `SaveManager` 的读写、版本字段、中断恢复。
- 轮回相关**成就**（跨轮回不重置）。

### 1.3 范围外（可后续迭代）

- 赛季合并逻辑的具体数值与运营配置界面（仅预留字段与接口说明）。
- 全服排行榜服务端实现（客户端仅「定格本地快照」需求可在 §8 说明）。

### 1.4 成功标准

- 玩家可在满级且秘境层数达标后完成一次完整重入；新档可见永久加成与前世宝库内容；旧档敏感数据已按规则清空且无脏档。

## §2 术语表

| 术语 | 含义 |
|------|------|
| 重入江湖 / 轮回 | 满足条件后触发的进度重置 + 永久加成累加流程 |
| 江湖阅历 | 每次重入后叠加的永久被动加成集合 |
| 前世宝库 | 跨轮回保存的少量装备槽位，新轮回开始后可取回 |
| 保留项 | 重入后不清空的数据（武学秘录萃取、宗师修为点、江湖见闻录等） |
| 重置项 | 重入后清空或回到初始状态的数据（等级、装备、材料、章节进度等） |
| 赛季 | 约 3 个月的活动周期；存档中记录 `season_id` 与 `season_start_time` |

## §3 核心规则摘要

### 3.1 触发条件

- 角色**达到满级**（满级数值以项目现有 `Player` / 等级表为准，实现时读取单一配置源）。
- **秘境通关层数 ≥ 20**（以当前赛季/存档记录的「已通关最高层」或等价字段为准，与战斗系统一致）。

两条件**同时满足**才可打开重入流程（可先展示灰态入口与未满足项提示）。

### 3.2 保留与重置（与设计表一致）

| 类别 | 内容 |
|------|------|
| **保留** | 武学秘录（已萃取的传奇特效）、宗师修为点、江湖见闻录 |
| **重置** | 等级、装备、材料、章节进度 |
| **额外** | 每次重入累加江湖阅历；用户可选最多 3 件装备进入前世宝库（规则见 §5） |

### 3.3 江湖阅历（永久被动）数值规则

| 重入次数 | 获得加成（当次累加后的典型形态，实现按公式递推） |
|----------|---------------------------------------------------|
| 第 1 次 | 全属性 +5%，经验获取 +10% |
| 第 2 次 | 全属性 +10%，经验获取 +20%，掉落品质 +5% |
| 第 3 次 | 全属性 +15%，经验获取 +30%，掉落品质 +10%，碎银获取 +15% |
| 第 N 次 | 每次相对上一世：**全属性 +5%**（累加）、**经验 +10%**（累加）、**掉落品质 +5%**（累加）、**碎银 +5%**（从第 4 次起与表一致递增，第 3 次碎银已为 +15%，实现时以 §5 公式为准统一） |
| **上限** | 无硬上限；需做**收益递减**（见 §5.3、§11），避免数值爆炸 |

**实现约定**：存档中存**当前累计百分比**（`legacy_bonuses`），每次重入在服务端逻辑（`season_system.gd`）原子更新 `rebirth_count` 与四项加成，UI 只读展示。

### 3.4 前世宝库

- **容量**：默认 3 件装备（`vault_capacity` 可配置）。
- **存入时机**：仅在重入确认流程中，从**当前背包/已装备**中选择（具体可选池在 §5 锁定）。
- **优先级展示**（用于排序与提示，非自动替用户选）：绝世真意 > 上古真意 > 传承 > 真意。
- **持久化**：跨轮回永久保存；新轮回开始后通过「前世宝库」面板**取回**到当前角色背包（需背包空位或走现有「邮件/暂存」机制，见 §5）。

### 3.5 赛季规划（与轮回并存）

- 每赛季约 **3 个月**。
- 赛季结束：**赛季角色合并到主角色**、排行榜定格、专属奖励发放 — 实现阶段至少预留 `season_id`、`season_end` 或运营下发配置钩子，避免硬编码在 UI。

### 3.6 轮回成就（跨轮回永久）

| 条件 | 奖励 |
|------|------|
| 首次重入 | 称号「轮回者」 |
| 3 次重入 | 称号「三世人」+ 专属外观 |
| 10 次重入 | 称号「不灭道心」 |

成就列表写入存档 `achievements`，**不因轮回清空**。

## §4 体验设计（L1–L5）

### L1 Fantasy

「放下一切重头来过，但每一世都比上一世更强。」

### L1 Emotion

- **主情绪**：重生感 — 放下的痛与新开始的期待并存。  
- **次情绪**：掌控感 — 永久加成可见、可对比上一世。

### L1 Payoff（一屏节奏）

满足条件 → 重入确认 → 角色剪影消散 → 「江湖阅历 +1」→ 永久属性条上涨 → 新角色诞生 → 打开前世宝库取出神兵 → 碾压第一章。

### L2 时间尺度

- **单轮回**：约 1–2 个月完成一轮（调数值目标，非硬编码）。  
- **跨轮回**：江湖阅历无限叠加上升，伴随递减曲线控制。

### L3 流派与 Build

每次重入可走不同流派；**前世宝库**用于引导新 Build（保留核心武器/套装部位），与武学秘录保留形成「机制 + 装备种子」双轴。

### L4 学习曲线（小时级）

- **~10h**：通过引导或面板得知重入条件。  
- **~50h**：首次达到满级 + 秘境 20+。  
- **首次重入**：完整仪式与说明。  
- **第二轮**：加速感来自经验加成与取回装备。

### L5 仪式感

- **重入仪式**：全屏剪影消散 + 本世数据统计回顾 + 永久加成弹窗。  
- **前世宝库取回**：独立取回动效/音效（可复用稀有掉落表现层级）。

## §5 详细规则与公式

### 5.1 重入条件判定（伪代码）

```
can_rebirth := (player.level >= max_level) and (rift_best_clear >= 20)
```

- `rift_best_clear` 与秘境系统使用同一数据源；若存在「当周记录」与「历史最高」，以**历史最高**为准，避免玩家掉层后无法重入。

### 5.2 江湖阅历递推公式

设 `k = rebirth_count`（完成重入后的累计次数，`k >= 1`）。存档中 `legacy_bonuses` 四项**名义百分比**与 `k` 的关系如下（与 §3.3 表一致）：

| 字段 | 公式 |
|------|------|
| `all_stats_pct` | `5 * k` |
| `exp_bonus_pct` | `10 * k` |
| `drop_quality_pct` | `5 * max(0, k - 1)`（k=1 时为 0，k=2 起每次 +5） |
| `gold_bonus_pct` | `k < 3 ? 0 : (15 + 5 * (k - 3))`（k=3 为 15，之后每次 +5） |

**实现**：在 `season_system.gd` 中提供纯函数 `compute_legacy_bonuses(rebirth_count: int) -> Dictionary`。每次重入事务内先 `rebirth_count += 1`，再按上表**全量重算**写入 `legacy_bonuses`，避免多次 PATCH 导致漂移；加载存档时若发现四项与 `rebirth_count` 不一致，以 `rebirth_count` 为准重算并写回。

### 5.3 收益递减（无硬上限）

在 §11 与战斗公式挂钩处应用**递减函数**，例如对累计属性乘区使用 `1 - exp(-all_stats_pct / D)` 或分段 cap 边际 — 具体系数放 `data/` JSON，便于调参。存档仍存**名义百分比**（便于 UI 展示），战斗内再映射为**有效加成**。

### 5.4 前世宝库选择规则

- 仅允许**装备**类 item（与 `loot_system` / `equipment_generator_system` 类型一致）。
- 选择数 ≤ `vault_capacity`（默认 3）。
- 若玩家选择少于上限，允许；重入后宝库仅存有选的条目。
- **跨版本**：存 `item_id` + `quality` + `slot` + **必要版本戳**（`schema_version` 或 `generator_revision`），加载时若装备生成规则变更，走 §10.5 兼容分支。

### 5.5 取回规则

- 取回时生成**当前版本**合法装备实例；若失败，降级为「补偿箱」或记录待处理队列，**不得静默吞装备**。

## §6 数据模型与存档

### 6.1 JSON 结构（与实现对齐）

```json
{
  "rebirth_count": 2,
  "legacy_bonuses": {
    "all_stats_pct": 10,
    "exp_bonus_pct": 20,
    "drop_quality_pct": 5,
    "gold_bonus_pct": 0
  },
  "vault": [
    {"item_id": "...", "quality": "primal", "slot": "weapon", "snapshot_version": 1},
    {"item_id": "...", "quality": "set", "slot": "chest", "snapshot_version": 1}
  ],
  "vault_capacity": 3,
  "achievements": ["first_rebirth", "three_lives"],
  "season_id": "season_2",
  "season_start_time": "2026-01-01T00:00:00Z"
}
```

说明：

- `snapshot_version` 为建议扩展字段，用于跨版本重建装备。若首版不实现，需在 §10 写明默认行为。
- `legacy_bonuses` 建议与公式重算结果一致；加载时可 `assert` 与 `rebirth_count` 一致，不一致则以 `rebirth_count` 为准重算并写回（修复脏数据）。

### 6.2 SaveManager 集成要点

- 重入执行必须是**原子事务**：先写临时备份 → 应用内存状态 → 写主存档 → 删除备份；失败则回滚。
- 与「保留项」相关的子系统在重置后仍从原键路径读取（武学秘录、宗师点、见闻录）。

### 6.3 与 `season_system.gd` 职责划分

| 职责 | 归属 |
|------|------|
| 条件检测、流程状态机 | `season_system.gd` |
| 数值重算、成就解锁 | `season_system.gd` 或委托 `AchievementSystem` |
| 存档序列化 | `SaveManager` |
| UI 绑定信号 | HUD / 独立 `RebirthUI` 场景 |

## §8 系统交互与依赖

### 8.1 依赖系统

- **等级系统**：满级判定。  
- **秘境系统**：最高通关层。  
- **装备生成 / 掉落**：宝库快照重建、品质枚举与 UI 一致。  
- **武学秘录 / 宗师 / 见闻录**：重入后保留，需确认其存档键不被重置逻辑误删。  
- **Loot / 经济**：掉落品质、碎银加成在对应结算点读取 `legacy_bonuses`（或有效值）。

### 8.2 信号建议（`season_system.gd`）

- `rebirth_available_changed(available: bool)`  
- `rebirth_started` / `rebirth_completed`  
- `vault_changed`  
- `legacy_bonuses_changed`

### 8.3 导航入口

- 主界面/HUD 增加「重入江湖」入口：条件未满足时灰显 + 点击弹出条件说明；满足时常亮。

## §9 重入完整操作流

**方案简述**：全程由 `season_system` 状态机驱动，UI 只发意图事件；执行阶段锁输入并防重复点击。

```
[条件检查]
    |
    v  (不满足) --> 提示面板（未满级 / 层数不足）--> END
    |
    v  (满足)
[打开重入面板] --> 加载本世统计、宝库预览、保留/重置列表
    |
    v
[选择前世宝库装备] --> 校验件数 <= capacity、类型合法
    |                    \
    |                     (超过/非法) --> 内联错误提示，留在本步
    v
[勾选「已知不可逆」]
    |
    v
[点击「二次确认：重入江湖」] --> 弹出模态二次确认（防手滑）
    |
    v  (取消) --> 返回重入面板
    |
    v  (确认)
[执行重置事务]
    |-- 备份存档
    |-- 应用重置：等级/装备/材料/章节等
    |-- 保留：秘录/宗师点/见闻录
    |-- 将所选装备写入 vault（JSON）
    |-- rebirth_count++ 并重算 legacy_bonuses
    |-- 解锁成就（首次/3次/10次）
    |-- 写主存档，清理备份标记
    v
[播放重入仪式] --> 剪影消散 + 统计回顾 + 永久加成弹窗
    |
    v
[新角色诞生] --> 进入新手/第一章场景（与项目新开档流程对齐）
    |
    v
[引导打开前世宝库] --> 玩家取回装备 --> END（正常游戏循环）
```

## §10 边界情况与风险

### 10.1 重入条件不满足

- 入口与面板内双重展示；任何时刻条件失效（如热修复改满级）时禁用确认按钮并刷新文案。

### 10.2 前世宝库满

- 选择第四件时：拒绝加入并提示「已达上限」；或需先移除槽内（若允许替换，首版建议**不允许替换**，简化逻辑）。

### 10.3 重置数据完整性

- 使用**白名单保留 + 黑名单重置**清单表（`data/` 配置），避免遗漏新加货币字段。  
- 单元测试：重入前后对关键字典键做 diff。

### 10.4 中断恢复

- 写档中断：下次启动检测「备份标记」，若主存档损坏则从备份恢复；若双坏，进入安全模式并上报日志。

### 10.5 跨版本兼容

- 旧档无 `vault`：补默认空数组与 `vault_capacity`。  
- `item_id` 无法重建：生成占位道具 + 发补偿（邮件）+ 记日志。

## §11 数值与节奏

### 11.1 永久加成与 DPS 公式关系

**方案简述**：全属性百分比进入属性乘区，与装备、Buff 加算或乘算遵循**现有战斗公式**统一入口；禁止在 UI 层私自叠乘。若当前公式为 `DPS = Base * Π(1+attr_i)`，则江湖阅历贡献单独函数 `f(all_stats_pct_effective)`，便于调递减。

### 11.2 经验加速与重复游玩节奏

- 经验加成作用于**击杀/任务结算经验**的最终乘区，不与「离线收益」冲突则需统一在 `OfflineRewards`（若有）同一系数读取。  
- 节奏目标：第二轮起章节推进时间较第一世缩短（由策划表驱动，不靠硬编码写死倍数）。
