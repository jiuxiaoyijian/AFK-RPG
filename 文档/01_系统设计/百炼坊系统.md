# 百炼坊系统 — 开发需求案（§1–§15）

> 面向：策划验收、程序实现、其他 AI 直接落地。  
> 引擎：Godot 4 + GDScript。主逻辑：`scripts/systems/cube_system.gd`；入口与上下文：`GameManager` autoload。  
> 配方数据：以本文 §5 为准；实现时需将 `data/equipment/cube_recipes.json`（或统一后的 `data/crafting/cube_recipes.json`）与资源 ID 对齐。

---

## §1 概述与产品定位

百炼坊是武侠挂机 RPG 中的**材料消耗主出口**与**终局 Build 调节器**，体验对标《暗黑破坏神 3》卡奈魔盒：玩家用积累的材料对单件装备执行高风险、高回报的确定性操作。

- **核心价值**：把「刷到的随机」转化为「我想要的构筑」——升品质、洗全件、洗单条、萃取传奇特效、传承套内换槽。  
- **边界**：百炼坊不负责「批量分解」「商店购买」；分解与掉落由 Loot / 背包系统承担。  
- **Fantasy（L1）**：在秘炉中锤炼出真正适合自己的神兵。  
- **主导情绪**：创造 / 改良；次要情绪：期待、紧张（押注材料与结果）。

---

## §2 目标与非目标

### §2.1 目标（必须达成）

1. 五大配方与校验规则与本文一致，且可从 UI 完整走通「选功能 → 放装备 → 验材料 → 执行 → 结果反馈」。  
2. 材料扣除、装备移除/替换、武学秘录解锁与**激活槽位规则**（兵器/护甲/饰品各 1）与服务器/存档一致。  
3. **淬火精炼**：单件装备仅允许锁定并反复洗练**一条**词缀；消耗按次数递增；词缀仅在同 bucket 内替换。  
4. 与 `EquipmentGeneratorSystem`（或等价造物节点）的 `upgrade_rare_item` / `reforge_item` / `convert_set_item` / `refine_affix_item` 等方法契约清晰、可单测。

### §2.2 非目标（本阶段不做）

- 不做玩家间交易、拍卖行。  
- 不做「一键最优洗练」或自动推荐材料投资（可做后续助手）。  
- 不修改主线关卡掉落表（仅消费侧定义材料出口）。

## §3 术语、品质与槽位

| 术语 | 说明 |
|------|------|
| 百炼坊 | 魔盒式单件加工界面与规则集合。 |
| 配方 / Recipe | `cube_recipes.json` 中一条记录，对应一个 `recipe_id`。 |
| 真意+ | 品质 ≥「真意」的装备（实现上与 `input_rarity_min: epic` 等等价，具体以 `GameManager.get_rarity_rank` 为准）。 |
| 玄品 | 稀有度 `rare`，用于精钢化真。 |
| 传承 | 套装品质 `set`，用于传承互转。 |
| 武学秘录 | `MartialCodexSystem` 维护的已解锁传奇特效集合；**激活**最多 3 个（兵器/护甲/饰品各 1）。 |
| bucket | 词缀生成用的分组 ID；淬火精炼仅在**同一 bucket** 内重掷。 |
| 精炼锁定槽 | 装备上持久字段（如 `refine_slot_index`）：首次选定词缀后写入，之后不可改选其他槽洗练。 |

**槽位大类（用于秘录激活上限）**：兵器、护甲、饰品 —— 需与装备 `slot` 或 `slot_category` 的枚举映射表一致（实现时在 §8 给出映射函数）。

## §4 系统架构与文件

### §4.1 运行时结构

```
UI(百炼坊面板)
    → 读取背包/装备实例
    → 调用 CubeSystem.can_execute_recipe / execute_recipe
    → GameManager 注入 resource_reader、equipment_generator、martial_codex_state
    → 扣费、删/换物品、写存档、播动画与飘字
```

- **静态入口**：`CubeSystem`（`scripts/systems/cube_system.gd`）负责配方存在性、材料是否足够、输入合法性、分支执行与结构化返回。  
- **上下文 `context` 建议字段**：`resource_reader: Callable`、`equipment_generator: Node`、`martial_codex_state: Dictionary`（或与 `GameManager` 一致的 accessor）。  
- **`options` 建议字段**：`convert_set` 若产品改为「随机槽位」，则服务端随机后写入 `replacement_item`，UI 可传空或忽略 `target_slot`（与现有 `target_slot` 实现需统一规格，见 §10）。

### §4.2 数据文件

- 配方列表：**单一真相源**为 JSON 数组（字段名可与现有 `cube_recipes` 键名一致）。  
- 推荐路径：`data/crafting/cube_recipes.json`；若短期仍用 `data/equipment/cube_recipes.json`，需在 `ConfigDB` 中明确加载路径，避免双份分叉。

### §4.3 配方 ID 与程序枚举（与现有代码对齐）

| 功能名 | `recipe_id`（代码） |
|--------|---------------------|
| 萃取武学 | `extract` |
| 精钢化真 | `upgrade_rare` |
| 回炉重铸 | `reforge` |
| 传承互转 | `convert_set` |
| 淬火精炼 | `refine_affix`（产品文案可称「淬火精炼」） |

## §5 数据模型要点

### §5.1 配方条目（每条）

- `id`：上表 `recipe_id`。  
- `name`：界面显示名。  
- 输入约束：`input_rarity` 精确匹配 或 `input_rarity_min` 最低品质。  
- `costs`：`[{ "resource_id", "amount" }, ...]`；淬火精炼的基础消耗在配方中给**基准值**，实际消耗 = 基准 × (1 + 0.1 × (n-1))，n 为**该装备**上已成功淬火次数（见 §6.5）。  
- `result`：逻辑标签（可与现有一致：`unlock_legendary_effect` / `upgrade_to_epic` / `reroll_full_item` / `swap_same_set_slot` / `reroll_single_affix`）。

### §5.2 资源 ID 映射（需求案 → 实现）

| 文案材料 | 建议 `resource_id` | 备注 |
|----------|-------------------|------|
| 碎银 | `gold` 或 `silver` | 基础费用；若项目已用 `gold` 表示碎银则沿用。 |
| 锻材 | `scrap` | 与现有分解产出的「锻材类」一致则不改名。 |
| 精铁 | `core` | 与每日修行、玄品分解一致。 |
| 秘卷残页 | `legend_shard` | Boss、真意+ 分解等来源。 |

实现以 `MetaProgressionSystem.get_resource_display_name` 或本地化表为准。

### §5.3 装备实例上与百炼坊强相关字段

- `rarity`、`slot` / `slot_category`、`affixes[]`、`legendary_affix`（含 `legendary_affix_id`、展示名）。  
- 套装：`set_id`（或等价）用于传承互转限定同一套。  
- **淬火**：`refine_slot_index`（锁定槽，-1 表示未锁定）；`refine_success_count`（可选，用于指数计费）。  
- 重铸保留：`is_set` / `set_id`、品质不变；词缀数组整体重随。

## §6 五大功能规格

### §6.1 萃取武学（`extract`）

| 项 | 规格 |
|----|------|
| 输入 | 1 件**真意及以上**且带可萃取传奇特效的装备。 |
| 消耗 | 精铁 ×50、秘卷残页 ×10。 |
| 输出 | **永久解锁**该 `legendary_affix_id` 至武学秘录；输入装备**销毁**。 |
| 秘录激活 | 玩家最多**同时激活** 3 个传奇特效：**兵器 / 护甲 / 饰品 各 1**；解锁数可多于 3，但激活受槽位限制。 |
| 失败/拒绝 | 已解锁同 ID、无传奇特效、品质不足、材料不足。 |

### §6.2 精钢化真（`upgrade_rare`）

| 项 | 规格 |
|----|------|
| 输入 | 1 件**玄品**（`rare`）。 |
| 消耗 | 锻材 ×80、精铁 ×30。 |
| 输出 | 1 件**真意**品质装备：**保留原有 3 条词缀**，**新增 1 条词缀 + 1 条传奇特效**（具体数值曲线由 `upgrade_rare_item` 实现）。 |
| 限制 | 无额外限制。 |

### §6.3 回炉重铸（`reforge`）

| 项 | 规格 |
|----|------|
| 输入 | 1 件**真意及以上**。 |
| 消耗 | 锻材 ×100、精铁 ×20。 |
| 输出 | **同品质、同槽位**新实例；**词缀全部重随**；**品质不变**；**套装标记与 set_id 不变**（非套装则仍为非套装）。 |

### §6.4 传承互转（`convert_set`）

| 项 | 规格 |
|----|------|
| 输入 | 1 件**传承**（`set`）。 |
| 消耗 | 精铁 ×30、秘卷残页 ×5。 |
| 输出 | **同一 `set_id`** 下**随机另一槽位**的传承装备（**不得与输入槽位相同**）；词缀全重随。 |
| 限制 | **仅允许套内**互转；若该套仅有单槽位装备（极端数据），应拒绝并提示。 |

### §6.5 淬火精炼（`refine_affix`）

| 项 | 规格 |
|----|------|
| 输入 | 1 件装备 + **选定 1 条词缀**（索引）。 |
| 消耗 | **第 1 次**锻材 ×50；每次成功后，下一次在同装备上消耗 = 上次消耗 × **1.1**（向上取整至整数）；或等价：基准 50 × (1 + 0.1×(成功次数))。 |
| 输出 | 该词缀被替换为**同 bucket** 内另一条词缀，或**同词缀新数值**（实现二选一或按权重表，须在 `equipment_generator` 文档化）。 |
| 限制 | **每件装备仅允许精炼一条词缀**：首次确认淬火时写入 `refine_slot_index`，之后 UI 仅展示该槽为可洗，其他槽禁用；未选槽前可切换，**一旦执行过第一次淬火则锁定**。 |

**品质门槛**：需求默认「任意品质可淬火」；若程序当前限制「仅真意+」，则以**本文为准放宽**或在此处显式保留门槛（团队二选一时，在 §14 记录决议）。推荐：**允许从精良起**以拉长 L2，但若数值爆炸则维持真意+。

## §8 与其他系统集成

1. **GameManager**：组装 `context`、在 `execute_recipe` 成功后扣资源、替换背包物品、调用存档。  
2. **EquipmentGeneratorSystem**：承接升品、重铸、套转、单条洗练；所有随机必须可走种子单测（若项目有战斗种子则注明是否独立 `craft_seed`）。  
3. **MartialCodexSystem**：萃取写入 `unlocked_effect_ids`；激活槽位变更不应在 `CubeSystem` 内静默完成，应由秘录 UI 或专用 API 触发，避免双写。  
4. **Loot / 分解**：材料入口不在本文展开，但 §11 的表需与经济配置一致。  
5. **HUD / 主导航**：百炼坊入口与 `main_nav_bar_controller` 等统一路由，进入时预加载配方 JSON。

## §9 完整操作流（端到端）

以下以「精钢化真」为例；其他功能将「右侧校验」替换为对应规则即可。

1. **打开百炼坊**：玩家从主城/菜单进入 → 加载 `cube_recipes` → 默认 Tab（建议上次停留 Tab 本地记忆）。  
2. **选功能**：点击「精钢化真」Tab → 左侧列表过滤为仅 `rare`。  
3. **放入装备**：点击背包中玄品 → 填入中央/右侧「输入格」→ UI 调用只读预览（可选）展示三条保留词缀。  
4. **确认材料**：底部读取玩家资源 → 若锻材/精铁不足，主按钮灰显，红字标不足项。  
5. **执行**：点击「化真」→ 二次确认（可选）→ `CubeSystem.can_execute_recipe` → 通过则 `execute_recipe("upgrade_rare", item, {}, context)`。  
6. **结果展示**：  
   - 成功：移除旧物、加入新真意 → **锤击 / 火花动画**（L5）→ 品质条跳为真意 → **传奇特效揭示** → 关闭面板或高亮「穿戴」→ 战斗 HUD **DPS 跳字**刷新。  
   - 失败：Toast 显示 `reason`，不扣材料、不删装备。

**萃取流差异**：成功后在武学秘录播放「光芒飞入」动效，并弹出解锁文案；装备移除。  
**淬火流差异**：首次需选词缀索引；成功后递增费用预览立即更新。

## §10 边界条件与校验

| 场景 | 预期行为 |
|------|----------|
| 材料不足 | `can_execute_recipe` 失败；按钮不可点或点击后 Toast；**不扣费、不改物品**。 |
| 输入品质不符 | 各配方在 `execute_recipe` 开头返回明确 `reason`（与现有一句式风格一致）。 |
| 萃取：无 `legendary_affix_id` | 拒绝执行。 |
| 萃取：已解锁同特效 | 拒绝；引导玩家去秘录查看。 |
| 重铸：非套装 | 允许；输出仍为非套装；**品质与槽位不变**。 |
| 传承互转：`set_id` 缺失或仅单槽套装 | 拒绝；`reason` 说明数据不完整或无法转换。 |
| 淬火：未选词缀 | 拒绝。 |
| 淬火：**精炼锁定** — 已存在 `refine_slot_index` 与当前选择不一致 | 拒绝或 UI 禁止切换。 |
| 淬火：该 bucket 无候选 | 程序应兜底同词缀重掷数值或记录错误日志并拒绝（避免死循环）。 |
| 并发 / 双击提交 | UI 防抖；服务端以事务方式「先锁背包格再扣费」。 |

## §11 材料经济平衡与 L2 节奏对齐

### §11.1 材料出口与来源（设计对齐表）

| 材料 | 主要来源 | 消耗出口（百炼坊） |
|------|----------|-------------------|
| 碎银 | 全品质分解等 | 基础操作费（若配方中需要，与分解循环形成软 sink） |
| 锻材 | 凡品/精良分解 | 精钢化真、回炉重铸、淬火精炼（主 sink） |
| 精铁 | 玄品分解、每日修行 | 萃取、精钢化真、回炉、传承互转 |
| 秘卷残页 | 真意+ 分解、Boss | 萃取、传承互转 |

### §11.2 L2 时间粒度（体验节奏）

- **秒级**：材料数字即时变化；按钮状态即时反馈。  
- **分级**：单次操作（含确认与动画）**≤ 30 秒**。  
- **时级**：平均 **1–2 小时** 活跃可积累 **1 次精钢化真** 量级材料（可通过调掉落表微调，以 §11.1 为锚）。  
- **日级**：日常 + 分解保证至少 **1–2 次** 百炼坊操作的材料（不一定是同一配方）。  
- **周级**：淬火精炼持续打磨 **1–2 件核心装备**。  
- **月级**：将单条词缀分布洗到满意区间（受递增成本自然封顶）。

### §11.3 L3 / L4 摘要

- **L3**：五功能覆盖升品质、全件洗、单条洗、萃取、套内换槽 —— 职责不重叠。  
- **L4**：约 **5 分钟** 理解入口与 Tab；**约 1 小时** 首次完成萃取；**约 10 小时** 理解各功能适用时机；**约 100 小时** 形成个人材料分配策略。

## §12 测试用例（每条配方至少 1 条）

以下均为** Given / When / Then** 风格，可映射为 GUT / 单元测试。

1. **萃取（extract）**  
   - Given：真意剑，带 `legendary_affix_id = "L01"`，材料足够且未解锁；When：`execute_recipe`；Then：装备移除、`unlocked_effect_id == "L01"`、材料按配方扣除。

2. **精钢化真（upgrade_rare）**  
   - Given：玄品重甲，三条固定词缀 A/B/C，材料足够；When：执行；Then：新装备为真意，仍含 A/B/C（值可相同但 ID 一致策略需在生成器定义），词缀数 +1 且存在传奇特效字段。

3. **回炉重铸（reforge）**  
   - Given：真意非套装护手，词缀列表 D；When：执行两次并固定随机种子；Then：两次输出品质均为真意、槽位均为护手，第二次词缀集合与第一次不完全相同（或统计分布在大量样本下变化）；套装标记始终 false。

4. **传承互转（convert_set）**  
   - Given：套装 S 的手套，同套存在头/鞋；When：执行；Then：输出为套装 S，槽位 ∈ {头, 鞋}，且不等于手套；`set_id` 不变。

5. **淬火精炼（refine_affix）**  
   - Given：装备已锁定 `refine_slot_index = 2`，第 n 次成功淬火；When：再执行；Then：仅索引 2 的词缀变化且新词缀 bucket 与旧的一致；消耗等于 50×1.1^(n-1) 向上取整（与 §6.5 公式一致）。

附加：**材料不足** — Given 锻材 0；When `can_execute_recipe`；Then `ok == false` 且 reason 含资源显示名。

## §13 体验与动画（L1 / L5）

### §13.1 L1 Payoff 链（精钢化真）

锤击动画 → 真意品质条弹出 → 传奇特效条揭示 → 引导「立刻装备」→ 主界面 **DPS 跳字**同步刷新。

### §13.2 L5 关键动画

- **精钢化真成功**：炉心强光、火花溅射、品质色阶从蓝→紫（或项目配色）过渡 0.6–1.2s。  
- **萃取成功**：装备剪影碎裂 → 光带飞入「武学秘录」图标 → 解锁列表新增一行。  
- **音效**：锤击重低音 + 成功铃音；失败用短促钝音避免挫败过长。
