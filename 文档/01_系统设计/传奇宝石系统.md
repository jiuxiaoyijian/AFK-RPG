# 传奇宝石系统 — 开发需求案（§1–§15）

> 本文档面向实现侧（Godot 4 / GDScript），供 AI 或程序直接落地「传奇宝石」玩法。  
> 主脚本：`scripts/systems/gem_system.gd`；配置：`ConfigDB` autoload；数据：`data/equipment/gems.json`。

---

## §1 目标与范围

**1.1 目标**  
在武侠挂机 RPG 中实现类暗黑 3 的「传奇宝石」：镶嵌于佩饰，提供可成长的独特数值效果；与秘境进度、材料消耗、成功率形成闭环。

**1.2 范围（In）**  
- 6 种传奇宝石定义、等级成长、最高等级。  
- 佩饰 2 槽镶嵌、同名互斥、免费装卸。  
- 宝石升级石消耗、按秘境层数判定的升级成功率、失败不降阶。  
- 掉落与保底规则、图鉴/背包与 UI 线框所涉数据字段（与现有装备/物品体系对齐）。

**1.3 非目标（Out）**  
- 非佩饰部位开宝石槽。  
- 付费直购宝石或升级石（若经济系统另有设计，不在本文档展开）。  
- 多颗同名宝石同时装备（明确禁止）。

**1.4 成功标准**  
玩家可在秘境玩法中获得首颗宝石与升级石；在宝石面板完成镶嵌/更换；升级时可见成功率与结果反馈；最终属性正确进入战斗/DPS 计算且与乘区文档一致。

---

## §2 术语与约定

| 术语 | 含义 |
|------|------|
| 传奇宝石 | 可升级的特殊物品，仅可镶嵌在佩饰宝石槽。 |
| 佩饰 | 装备子类，共 2 件，每件 1 个宝石槽，合计 2 槽。 |
| 宝石升级石 | 秘境通关产出的消耗品，用于宝石等级提升。 |
| 秘境层数 | 玩家本次或最近通关的秘境难度层，用于成功率判定（与 §5、§11 一致）。 |
| `gem_id` | 宝石唯一标识，用于存档与互斥校验。 |

**脚本与数据**  
- 系统逻辑：`gem_system.gd`（读取 `gems.json`，对外提供查询、镶嵌校验、升级请求、属性聚合接口）。  
- 数值与文案：`gems.json`；物品/掉落表与现有 `drop_tables.json`、装备生成器衔接处需有 `gem_upgrade_stone` 与各 `gem_id` 条目。

## §3 玩家体验目标（L1–L5）

### L1 Fantasy  
在秘境深处淬炼神石，为神兵增添灵性——佩饰上的宝石随层数与养成逐渐觉醒，成为角色 Build 的「第二签名」。

### L1 Emotion  
主情绪：**成长 / 变强**（每次升级与镶嵌都能感知数值与反馈）；次情绪：**收集 / 养成**（六种宝石对应不同流派取向）。

### L1 Payoff  
宝石升级 → **发光/粒子短演出** → 等级 +1 → 面板属性增长 → **HUD 或角色面板 DPS/战力** 同步上升（失败时播放「安全失败」弱反馈，无惩罚焦虑）。

### L2（时间尺度对齐，与 §11 概率表一致）  
- **秒**：镶嵌/取下即时生效，关闭面板前已写入存档关键字段。  
- **分**：单次「打开升级面板 → 确认材料 → 播放结果」操作约 **30 秒** 量级（含可选跳过动画）。  
- **时**：单次秘境通关稳定获得 **1–2 个** 宝石升级石（必掉，见 §6）。  
- **日**：日常节奏下玩家可对目标宝石进行 **1–2 次** 升级尝试（受石储备与成功率约束）。  
- **周**：周目标可为指定宝石 **+5 级**（需更高层秘境或承受失败重试）。  
- **月**：单颗宝石 **满级 50** 作为中长期目标；双槽两颗满级对应更长周期。

### L3 Build 分化  
6 种宝石分别强化：**普攻/技能伤害面**（猛虎）、**生存**（铁壁、噬血）、**手感与频率**（迅雷）、**爆发上限**（破军）、**精英/Boss 特化**（困兽）——与装备词条形成互补而非完全重叠。

### L4 进度锚点  
- **约 5 分钟**：完成引导，理解「佩饰两槽 + 库存列表 + 升级入口」。  
- **约 1 小时**：通过秘境 **10 层+** 首次保底获得 **1 颗** 传奇宝石。  
- **约 10 小时**：根据 Build 选定 **两颗不同宝石** 的组合并稳定镶嵌。  
- **约 100 小时**：在合理秘境层数推进下，**双宝石均达满级**（可作为硬核玩家参考，非强制 KPI）。

### L5 高光时刻  
- 升级成功：**金光环绕**佩饰图标或宝石槽位，短音效+光晕 0.5–1.5s。  
- 单颗满级：**该槽位专属常驻弱光环**（与其他 UI 特效分层，可开关「简化特效」）。

## §4 宝石列表与数值

| 宝石名 | `gem_id` 建议 | 效果（Lv1 基准） | 每级提升 | 最高等级 |
|--------|----------------|------------------|----------|----------|
| 猛虎之石 | `gem_tiger` | 攻击力 **+X%** | +1% | 50 |
| 铁壁之石 | `gem_ironwall` | 防御力 **+X%** | +1% | 50 |
| 迅雷之石 | `gem_swift` | 攻速 **+X%** | +0.5% | 50 |
| 噬血之石 | `gem_bloodthirst` | 击杀回复 **X%** 生命 | +0.2% | 50 |
| 破军之石 | `gem_critdmg` | 暴击伤害 **+X%** | +2% | 50 |
| 困兽之石 | `gem_elite` | 精英伤害 **+X%** | +1.5% | 50 |

**数值公式（实现约定）**  
- 当前等级 `L`（1–50）：`总加成 = base_effect.value + per_level.value * (L - 1)`，与 `gems.json` 中 `base_effect` / `per_level` 一致。  
- 显示层：百分比类在 UI 保留 **1 位小数**（迅雷等 0.5% 步进友好）。

## §5 镶嵌规则

1. **槽位**：佩饰 ×2，**每件 1 槽**，共 **2** 个传奇宝石槽。  
2. **容量**：每槽最多 **1** 颗传奇宝石。  
3. **互斥**：**同名宝石不可重复镶嵌**——两槽装备的 `gem_id` 必须不同；第二次镶嵌同名时拒绝并提示原因。  
4. **费用**：镶嵌与取下 **免费**，无冷却，随时可换（需非战斗锁定若项目全局有「战斗中不可换装」则遵守全局规则，本文档默认允许）。  
5. **解锁**：若佩饰槽位存在「未解锁」设计，见 **§10.4**；默认两佩饰槽从获得第一件佩饰起可用，**宝石槽随佩饰装备存在**（佩饰卸下则该槽宝石效果暂停，数据可保留或自动卸下由产品二选一，**推荐**：卸下佩饰时宝石仍挂在该佩饰上，再装备即生效，避免玩家误以为丢失）。

## §6 升级机制与获取途径

### 6.1 升级消耗  
- 每次升级消耗 **宝石升级石**，数量随当前等级递增。  
- 默认公式与数据示例一致：`count = level + 1`（升级前等级为 `level` 时，升到 `level+1` 消耗 `level+1` 个石头，可在 `upgrade_cost` 中用 `count_formula` 字符串或解析表实现）。

### 6.2 成功率（相对「用于判定的秘境层数」`F`）  
设宝石当前等级为 `G`（即将尝试升到 `G+1`，判定用 **目标等级** 或 **当前等级** 须全项目统一，**推荐：以升级前等级 `G` 与层数 `F` 比较**，与下表一致）：

| 条件 | 成功率 |
|------|--------|
| `G ≤ F` | **100%** |
| `F+1 ≤ G ≤ F+5` | **60%** |
| `G > F+5` | **30%** |

- **失败**：**不降级**（安全失败），材料仍消耗（与暗黑-like 常见体验一致；若策划要求失败不扣石，需单独立项，**本文档默认失败扣石**）。  
- **层数来源**：使用玩家**本周/本次**用于判定的层数，推荐 **「最近一次成功通关的秘境最高层」** 或 **「当前选择的挑战层」**，需在 `gem_system` 与 UI 明确展示（§10.3）。

### 6.3 掉落与保底  
- **传奇宝石**：秘境 **10 层及以上** 通关 **首次** 保底掉落 **1 颗**（可指定随机池或固定新手池）。  
- **10 层后重复**：秘境 **15 层及以上** 每次通关约 **10%** 概率额外掉落 1 颗（与保底不冲突时的随机表由掉落系统合并）。  
- **宝石升级石**：每次秘境通关 **必掉**；单次 **1–2 个**（与 L2「时」尺度对齐，具体权重表放 `drop_tables.json`）。

## §8 数据结构（`data/equipment/gems.json`）

单条宝石定义示例（数组根节点 `gems` 由项目约定）：

```json
{
  "gem_id": "gem_tiger",
  "name": "猛虎之石",
  "base_effect": {"stat": "attack_percent", "value": 1},
  "per_level": {"stat": "attack_percent", "value": 1},
  "max_level": 50,
  "upgrade_cost": [{"item": "gem_upgrade_stone", "count_formula": "level + 1"}]
}
```

**字段要求**  
- `stat` 与战斗属性枚举一致（见 §11 乘区）。  
- `count_formula` 支持安全子集表达式或离散表；**禁止**任意代码执行。  
- 可选：`drop_weight`、`tags`、`description`、`icon` 路径。

**存档建议（实现参考）**  
- `player.gems_inventory`: `Dictionary gem_id -> { level, count }` 或分离「实例」与「堆叠」按项目装备惯例。  
- `player.trinket_gem_slots`: `[{ trinket_slot_index, gem_id|null }, ...]` 与装备实例 ID 绑定，便于卸下佩饰后还原。

## §9 完整操作流（获取 → 镶嵌 → 升级 → 换宝石）

1. **获取**  
   - 玩家通关秘境 ≥10 层：触发首次保底，背包/邮件获得一颗传奇宝石；≥15 层后续有概率掉落。  
   - 每次通关获得宝石升级石（必掉，1–2）。  
   - **首次获得**：弹出 §10.5 引导，并高亮主菜单「宝石」入口（若已有导航栏位）。

2. **镶嵌**  
   - 打开宝石主面板 → 在库存选中宝石 → 选择目标佩饰槽 → 校验 **非同名**、槽空或覆盖确认（覆盖时先卸下旧宝石至库存）→ 写入存档 → **立即重算属性**。

3. **升级**  
   - 选中已镶嵌或库存中的宝石 → 进入升级面板 → 显示消耗、成功率 → 确认 → 扣材料 → RNG → 成功则 `level++` 并播放 L5 成功特效；失败播放弱特效 → UI 刷新。

4. **换宝石**  
   - 在槽位上选「更换」→ 选另一颗 **不同名** 宝石替换；或先「卸下」再镶嵌。全程免费。

## §10 边界条件与错误处理

| 场景 | 行为 |
|------|------|
| **10.1 同名重复镶嵌** | 第二颗同名宝石不可上槽；弹窗/气泡：「同名传奇宝石不可同时镶嵌」。 |
| **10.2 升级石不足** | 「确认升级」不可点；材料行红色提示缺少数量。 |
| **10.3 成功率展示** | 必须显示 **数值%** 与 **F、G 来源**；若层数数据缺失，回退为 F=0 并醒目标注「未通关秘境层数，成功率按最低档计算」或禁止升级（二选一，**推荐显示可升级但概率低**以免死锁）。 |
| **10.4 宝石槽解锁** | 若佩饰无孔：宝石槽「锁定」图标 + 条件文案（如「装备任意佩饰解锁」或随主线解锁）；与 §5 对齐。 |
| **10.5 首次获得引导** | 全服/角色标记 `has_seen_first_gem`；首次获得后 **模态轻引导** 三步：说明佩饰双槽 → 演示打开面板 → 提示升级石用途。可跳过。 |

## §11 数值与战斗集成

### 11.1 升级概率与 L2 对齐说明  
- L2 要求「单次秘境约 1–2 升级石」「日均可尝试 1–2 升级」——依赖 **§6.3 必掉 1–2** 与 **§6.2 高失败率时的石头周转**；QA 可用蒙特卡洛模拟：在 F 固定比 G 低 3–5 时，日均尝试次数是否落在目标区间，必要时仅调 `count_formula` 或掉落个数，**不改成功率档位**。  

### 11.2 属性乘区位置（与全局战斗公式一致）  
实现时必须在统一属性管线登记，建议归类如下（若项目已有 `StatAggregator`，在此注册 modifier source=`legendary_gem`）：  

| stat 键 | 建议乘区 |
|-----------|----------|
| `attack_percent` | **攻击端百分比乘区**（与其他「攻击力 +%」加法合并后再乘算） |
| `defense_percent` | **防御端百分比乘区** |
| `attack_speed_percent` | **攻速乘区**（加法叠乘或项目既定攻速公式内层） |
| `life_on_kill_percent` | **击杀回复**：基于最大生命或当前生命由现有 `life_steal` 管线决定，**独立加法项** |
| `crit_damage_percent` | **暴伤乘区**（与装备暴伤加法合并） |
| `elite_damage_percent` | **对精英/Boss 独立乘区**（仅对标签精英生效） |

**要求**：`gem_system` 输出 **扁平修正列表**（stat, value, source），由 `equipment_generator_system` 或战斗前的 `BuildResolver` 合并；**禁止**宝石内重复应用全局 Buff。

## §12 测试验收标准（QA）

1. **镶嵌正确性**  
   - 两槽镶嵌不同 `gem_id` 时，战斗内属性面板与预期公式一致；卸下一颗后加成立即移除。  

2. **升级成功率验证**  
   - 固定 RNG seed 下，100% / 60% / 30% 三档各跑 1000 次蒙特卡洛，误差在置信区间内；失败从未出现 `level` 下降。  

3. **属性计算**  
   - 取 L1、L25、L50 三组快照，手算 `base + per*(L-1)` 与 UI、战斗日志一致。  

4. **同名互斥**  
   - 库存有两颗「猛虎之石」时，仅能镶嵌一颗；第二颗尝试第二槽时拒绝。  

5. **掉落与保底**  
   - 新号首次 10 层通关必得一颗宝石；升级石每次 ≥1。

## §15 开放问题（需策划/程序联定）

1. 升级**失败是否扣除升级石**（本文默认 **扣**）。  
2. 佩饰卸下后，宝石是「挂在装备实例上」还是「弹回库存」（**推荐挂装备**）。  
3. 用于成功率判定的 F 取「历史最高」还是「最近通关」——影响肝度与公平性。  
4. 多颗同 ID 宝石在库存中的堆叠规则与「仅一颗在槽」的 UX 展示。
