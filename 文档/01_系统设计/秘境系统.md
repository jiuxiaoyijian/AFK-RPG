# 秘境系统 — 开发需求案（§1–§15）

> 面向实现：Godot 4 + GDScript，供其他 AI 或工程师直接落地秘境（大秘境式）终局玩法。  
> 主脚本：`scripts/systems/rift_system.gd`；与 `GameManager` autoload 关联。  
> 数据：`data/rift/rift_scaling.json`、`data/rift/rift_keys.json`。

---

## §1 文档目的与读者

**目的**：定义秘境从入场、战斗、结算到排行与掉落的完整规则、UI 信息架构、数据契约、边界行为与验收测试，使实现与策划表一致且可回归验证。

**读者**：Gameplay 程序、UI、数值、存档与 QA。

**非目标**：本章不写具体 Godot 节点命名以外的引擎细节；不规定美术资源文件名（仅功能占位）。

## §2 术语与定义

| 术语 | 定义 |
|------|------|
| 秘境 | 类暗黑 3 大秘境：消耗钥石入场、限时清场、层数无限、掉落与缩放随层数增长。 |
| 层数 `n` | 整数，从 1 起，无理论上限；玩家选择可进入的层数受所持钥石档位约束。 |
| 钥石 | 入场消耗品，分档位；不同档位解锁不同层数区间。 |
| 通关 | 限时内击杀秘境场景内全部敌人；失败含超时或放弃。 |
| 个人最高层 | 玩家历史成功通关的最高 `n`，用于展示与部分解锁。 |
| 真意掉率 | 表列区间为策划目标，实现以配置或公式导出为准，与 `rift_scaling.json` 对齐。 |

## §3 设计目标与范围

**目标**

- 秘境作为衡量 Build 强度的标尺与终局循环核心。
- 入场门槛明确（钥石），单局节奏短（约 3 分钟限时 + 读条/结算）。
- 层数越高：怪物越强、掉落品质加成越高；高段有指数尾巴以支撑长期推层。

**范围（Must）**

- 钥石库存、消耗、与层数区间校验。
- 缩放应用到秘境敌人生成/战斗属性管线（与项目现有敌人属性接口对接）。
- 限时 3 分钟、全清胜利条件、失败与结算 UI。
- 个人最高层持久化；里程碑首通标记与奖励/解锁钩子。
- 专属掉落表按层数门槛过滤。

**范围外（可后续迭代）**

- 全服排行榜、组队秘境、赛季重置规则（若需，在 §14 扩展）。

## §4 系统集成与文件契约

**Autoload / 管理器**

- `GameManager`：提供全局游戏状态、场景切换、玩家资源（钥石、货币等）读写接口；`rift_system.gd` 通过信号或显式 API 请求开始/结束秘境、应用奖励。

**系统脚本：`rift_system.gd`（职责划分建议）**

1. 解析 `rift_keys.json`：钥石 id、名称、对应 `min_floor`、`max_floor`、消耗数量。  
2. 解析 `rift_scaling.json`：按层或层段读取血量倍率、攻击倍率、掉落品质加成、真意掉率等；支持版本字段（见 §10）。  
3. 会话状态：`IDLE` → `ENTRY_PENDING` → `IN_RIFT` → `RESOLVING` → `IDLE`；中途异常回滚策略见 §10。  
4. 向战斗层下发：当前层 `n`、缩放系数、限时剩余秒数；从战斗层收集：剩余敌人数、击杀事件、用时。

**数据文件**

- `data/rift/rift_scaling.json`：层段或逐层表 + 51+ 公式参数（或预计算缓存表），含 `schema_version`。  
- `data/rift/rift_keys.json`：钥石档位与可挑战层数区间、来源说明（展示用）、堆叠规则。

**与掉落/装备系统**

- 秘境结算调用现有 `loot_system` / `equipment_generator` 管线时，传入「秘境上下文」：`floor=n`、`quality_bonus`、是否首通里程碑、`key_tier`；专属表项（华戒、传奇宝石、升级石）在掉落管线中按层数门槛过滤。

## §5 用户故事与验收要点（摘要）

1. **作为玩家**，我消耗一枚符合层数区间的钥石进入秘境，使高层推进有成本与节奏。  
   - *验收*：层数不在钥石区间或库存不足时无法开始，并给出明确提示（§10）。

2. **作为玩家**，我在 3 分钟内清光敌人则通关，否则失败。  
   - *验收*：倒计时到 0 仍有敌人 → 失败；最后一只死在 `t>0` → 胜利。

3. **作为玩家**，通关更高层提升掉落品质加成并可能解锁更高档钥石掉落。  
   - *验收*：加成与 `rift_scaling` 一致；钥石掉落档位与当前/通关层匹配设计表。

4. **作为玩家**，首次达到 10/20/30/50 层触发里程碑（解锁/称号/钥石）。  
   - *验收*：首通标记只触发一次；存档可迁移且不重复发奖。

## §6 核心玩法流程（状态与事件）

**流程**

1. 玩家在城镇/主界面打开秘境入口面板 → 选择层数 `n` → 系统校验钥石 → 扣钥石 → 加载秘境战斗场景。  
2. 进入战斗：`rift_system` 通知刷怪与属性缩放；启动 180s 倒计时。  
3. 每击杀同步剩余怪物数；全清且时间未超时 → 胜利分支。  
4. 胜利：生成掉落、更新个人最高层、检查里程碑、展示结算面板。  
5. 失败或放弃：不发放通关掉落（或仅保底规则若策划另定——**默认**：失败无通关奖；钥石已消耗不返还）。  
6. 「下一层」：`n+1` 若仍在所持钥石允许区间内且仍有钥石，则一键再开；否则引导换钥石或降低层数。

**关键事件（建议信号名，实现可改名但语义保留）**

- `rift_run_started(floor, key_id, time_limit_sec)`  
- `rift_enemy_count_changed(remaining)`  
- `rift_run_succeeded(elapsed_sec, floor)`  
- `rift_run_failed(reason)` — `timeout` | `player_dead` | `abandon`  
- `rift_run_settled(rewards_summary)`

## §7 UI 信息架构与线框图（ASCII）

**设计原则（每类 1–2 句）**

- **入口面板**：突出「当前能打的最高层」与钥石库存，减少误选层数导致的无效尝试；层数步进控件与钥石档位颜色编码一致。  
- **战斗 HUD**：倒计时始终可见（大字号）；剩余怪物数与进度条双通道防止玩家误判「是否还有怪」。  
- **结算面板**：先反馈胜负情绪（大字/色块），再给可扫读的掉落列表；「下一层」仅在条件满足时出现，避免误触浪费钥石。

### 7.1 秘境入口面板

**方案**：顶部展示个人历史最高层；中部为层数选择器与当前选中层对应的钥石需求预览；底部主按钮「开始秘境」；钥石不足时主按钮禁用并显示获取途径链接/文本。

```
+------------------------------------------------------------------+
|  秘境                                    [?帮助] [关闭]          |
+------------------------------------------------------------------+
|  个人最高层:  23                                                  |
|  推荐钥石:    精钢钥石 (11-30层)                                  |
+------------------------------------------------------------------+
|  选择层数:   [ - ]    <<  15  >>    [ + ]     (步进: 1)          |
|  钥石需求:    精钢钥石 x1                                         |
|  库存:        精钢 4  |  粗铁 2  |  玄铁 0  |  陨铁 0           |
+------------------------------------------------------------------+
|  [开始秘境]            (禁用态: 灰色 + 提示「钥石不足」)          |
|  小字: 限时3分钟清场全部敌人。失败不返还钥石。                    |
+------------------------------------------------------------------+
```

### 7.2 秘境战斗 HUD

**方案**：倒计时置顶居中；层数与剩余怪数分列两侧避免叠字；进度条用「已击杀/总怪物」或「剩余占比」二选一，须与数字同源数据。

```
+------------------------------------------------------------------+
| 层数: 15                    02 : 47                    剩余: 7  |
+------------------------------------------------------------------+
| 清剿进度 [=====================>          ]  18 / 25           |
+------------------------------------------------------------------+
|                         (战斗区)                                 |
+------------------------------------------------------------------+
| [暂停]                                    (可选: Buff图标行)    |
+------------------------------------------------------------------+
```

### 7.3 秘境结算面板

**方案**：通关用暖色高对比横幅，失败用冷色；掉落列表支持滚动；个人最高层仅在打破记录时额外高亮一行；「下一层」旁显示将消耗的钥石名与数量。

```
+------------------------------------------------------------------+
|                     【 秘境通关 】                               |
|  用时:  2:16      层数: 15      新纪录! (+1层)                  |
+------------------------------------------------------------------+
|  掉落:                                                           |
|    - 传奇装备: xxx                                               |
|    - 材料: xxx x3                                                |
|    - (里程碑首通奖励若有则插入一行高亮)                          |
+------------------------------------------------------------------+
|  [返回城镇]     [下一层 16层] (需: 精钢钥石 x1)                   |
+------------------------------------------------------------------+
```

**失败示例（同布局区替换文案）**

|                     【 挑战失败 】                               |
|  原因: 时间耗尽      已清: 18/25                                 |
+------------------------------------------------------------------+
|  安慰掉落(若有): ...           (默认可无)                        |

## §8 核心机制汇总表（实现对照）

| 要素 | 设计 |
|------|------|
| 入场 | 消耗秘境钥石（Boss / 秘境掉落） |
| 层数 | 1 起，无上限 |
| 目标 | 限时 **3 分钟**内击杀全部敌人 |
| 奖励 | 层数越高，掉落品质加成越高；专属掉落见表 |
| 排行 | 个人最高层记录（持久化） |

### 8.1 数值缩放表（策划目标；实现以 JSON 为准）

| 层数 `n` | 怪物血量倍率 | 怪物攻击倍率 | 掉落品质加成 | 真意掉率 |
|----------|-------------|-------------|-------------|---------|
| 1–5 | 1.0 ~ 1.5x | 1.0 ~ 1.2x | +0% | 7% |
| 6–10 | 1.5 ~ 3.0x | 1.2 ~ 1.8x | +10% | 8% |
| 11–20 | 3.0 ~ 8.0x | 1.8 ~ 3.0x | +25% | 10% |
| 21–30 | 8.0 ~ 20x | 3.0 ~ 5.0x | +50% | 13% |
| 31–50 | 20 ~ 80x | 5.0 ~ 10x | +100% | 18% |
| 51+ | `80 * 1.17^(n-50)` | `10 * 1.1^(n-50)` | +150% | 22% |

**说明**：段内可采用线性插值或逐层键值；51+ 必须连续可计算。每层约 **17%** 血量增长为体验目标，与 DPS 检验一致（§11）。

### 8.2 秘境钥石

| 钥石等级 | 来源 | 对应层数范围 |
|---------|------|--------------|
| 粗铁钥石 | 章节 Boss 掉落 | 1–10 |
| 精钢钥石 | 秘境 10 层+ 掉落 | 11–30 |
| 玄铁钥石 | 秘境 30 层+ 掉落 | 31–50 |
| 陨铁钥石 | 秘境 50 层+ 掉落 | 51+ |

### 8.3 秘境专属掉落

| 掉落 | 层数要求 | 说明 |
|------|---------|------|
| 华戒 | 20+ | 传承辅助饰品 |
| 传奇宝石 | 10+ | 佩饰专属宝石 |
| 宝石升级石 | 通关奖励 | 升级已有传奇宝石 |

### 8.4 里程碑（首通一次性）

- 首次通关 **10** 层：解锁传奇宝石系统。  
- 首次通关 **20** 层：解锁华戒掉落 + 成就称号。  
- 首次通关 **30** 层：成就称号 +「秘境征服者」。  
- 首次通关 **50** 层：解锁最高级钥石（陨铁）相关掉落/合成/商店条目（与项目经济挂钩处二选一或组合）+ 成就。

## §9 L1–L5 体验设计（文案与节奏）

**L1 Fantasy**：一层更比一层强，深入秘境证道天下。

**L1 Emotion**：主情绪挑战与征服；次情绪为限时 3 分钟带来的紧张感。

**L1 Payoff**：倒计时开始 → 3 分钟极限清场 → 在计时归零前击杀最后一只 → 通关 → 层数 +1 / 新纪录 → 专属掉落反馈。

**L2 时间尺度**

- 秒：3 分钟倒计时紧迫感。  
- 分：单次秘境约 3–5 分钟（含过场）。  
- 时：每小时可完成约 2–3 次（受钥石与玩家水平影响）。  
- 日：钥石限制每日约 3–5 次有效闯关（与经济系统对齐）。  
- 周：每周推高约 3–5 层（休闲）；肝度更高时可上调钥石供给但需经济审核。

**L3 打法差异**：低层偏速通与小怪拉火车；高层偏生存、单体优先级与技能循环优化。

**L4 渐进目标（参考）**：约 5 min 首次进入秘境 → 约 1 h 推到 10 层 → 约 10 h 到 20 层 → 约 100 h 到 50+ 层（随平衡调整）。

**L5 仪式感**

- 新纪录：层数数字放大、轻震屏、横幅「新纪录」。  
- 里程碑首通：单独弹条 + 称号解锁动画占位。

## §10 边界情况、失败恢复与兼容

| 场景 | 期望行为 |
|------|----------|
| **钥石不足** | 开始按钮禁用；提示缺口与获取途径（章节 Boss / 高层秘境掉落表）。 |
| **超时失败** | 立即锁输入 0.3s 防误触 → 失败结算；钥石不返还；不发放通关掉落。 |
| **中途断线 / 强关** | 若服务端无：本地视为 **放弃** 或 **失败**（二选一，**推荐**：放弃 = 失败且钥石已消耗）；回到城镇不重复结算。若已有半局存档，需定义是否允许续关（**默认**：不允许续关，简化反作弊与状态机）。 |
| **跨版本缩放数据兼容** | `rift_scaling.json` 含 `schema_version`；新字段缺省时用默认值并写日志；旧存档层数记录保留；若公式变更，仅影响新开局，不回溯改写历史最高层。 |
| **首次引导** | 首次获得任意钥石或首次打开入口面板时，弹简短教程：钥石档位、层数区间、限时规则、失败成本；可勾选「不再提示」。 |

## §11 缩放公式、DPS 检验与 L2 推层节奏对齐

### 11.1 血量与攻击（51+）

- 血量倍率（相对 50 层基准段上限）：`HP_mult(n) = 80 * pow(1.17, n - 50)`  
- 攻击倍率：`ATK_mult(n) = 10 * pow(1.1, n - 50)`

1–50 段可使用表内插值或逐层配置，须单调不减。

### 11.2 DPS 检验（与「每层约 17% 血量」对齐）

设通关时间目标接近常数 `T`（如 120s–180s），单层有效 DPS 需求约与血量成正比。令相邻层血量比 `r_HP(n) = HP_mult(n+1) / HP_mult(n)`：

- 对 **51+**，`r_HP(n) ≈ 1.17`，即每层约需 **+17%** 总 DPS（忽略玩家熟练度提升）才能维持相近通关时间。  
- **验收**：在固定 AI 木桩输出下，将 `n` 与 `n+1` 层模拟通关时间比值应约为 `1 / r_HP`（允许 ±策划容差）。

### 11.3 与 L2「每周 +3~5 层」对齐（数值出口）

- 若每周游玩钥石次数与 §9 日限制一致，则每周装备/等级带来的 DPS 复合增长应落在 **(1.17)^3 ~ (1.17)^5** 量级的可达成区间，否则需调整：钥石供给、段内插值曲线、或怪物数量（总血量池）。  
- **实现提示**：将「单局怪物总血量池」作为独立乘子，便于在不改单怪血量手感的情况下微调通关时间。

### 11.4 掉落品质加成

- 段内加成见 §8.1；**51+** 固定 **+150%**（或叠加 caps，若有全局上限需在 `loot_system` 文档对齐）。

## §12 测试计划（可自动化 + 手工）

| ID | 验证项 | 方法 |
|----|--------|------|
| T1 | **缩放计算正确性** | 单元测试：对 `n ∈ {1,5,10,20,30,50,51,60}` 读取配置/公式输出，与表 §8.1 基准值比对（容差 0.01）。 |
| T2 | **限时机制** | 集成：刷 3 只怪，将时间缩到 5s，最后 1s 击杀最后一只 → 成功；未击杀 → `timeout`。 |
| T3 | **钥石消耗** | 开始前记录库存 → 开局扣 1 → 失败/胜利后确认不重复扣除；非法层数不扣。 |
| T4 | **专属掉落门槛** | 掉落种子固定时，`n=9` 无传奇宝石表项；`n=10` 可出；`n=19` 无华戒，`n=20` 可出。 |
| T5 | **里程碑一次性** | 首次 `n>=10` 发解锁事件且重复通关不重复发；存档读写后仍不重复。 |
| T6 | **UI 状态** | 钥石不足时开始禁用；结算「下一层」在 `n+1` 超钥石档位时隐藏或改文案。 |

## §13 遥测与日志（建议）
