# 产品化缺口与后续优先级清单

## 1. 文档目的

本文档不是开发日志，也不是新的功能脑暴清单，而是一次回到项目原始目标的阶段复盘。

它主要回答两个问题：

- 这个项目为了提升留存，还必须继续补哪些功能与体验。
- 作为一个能被包装、展示、持续迭代的完整产品，目前还缺少哪些原型之外的关键部分。

后续如果继续开发，建议优先参考本文档判断方向，再决定具体做哪些系统与资源。

## 2. 最初北极星目标

根据 `01_项目总控与AI执行说明.md`、`02_核心战斗与系统循环设计.md`、`03_养成经济与长期成长设计.md`，本项目最初的北极星目标可以归纳为：

### 2.1 产品定位

- 做一款适合 `Steam` 桌面常驻的横版自动战斗挂机游戏
- 战斗默认自动，不走动作游戏路线
- 参考《英雄没有闪》的刷装与构筑思路，但不照搬手游结构

### 2.2 目标体验

- 低打扰运行，适合边工作边挂机
- 高反馈观看，玩家瞟一眼就能理解当前状态
- 核心乐趣来自 `刷装 -> 选词条 -> 调整 build -> 推更高关`
- 玩家不需要高频操作，但需要持续做低频有效决策

### 2.3 留存相关硬标准

- 玩家 5 分钟内能看懂玩法
- 30 分钟内能体验至少 2 次明显 build 改变
- 自动挂机过程中每 1 到 3 分钟至少产生 1 次有效决策
- 玩家能理解当前 build 为什么强、为什么弱
- 卡关时可以通过构筑调整解决，而不是只能等数值

### 2.4 设计原则

- 不要做成动作游戏
- 不要做成传统放置数值点击器
- 所有系统都要服务 build 变化
- 先做闭环 MVP，再扩内容量
- 优先保证功能正确和结构清晰，再补特效与动画

## 3. 已完成且仍应坚持的方向

截至目前，项目已经完成了不少真正服务核心目标的内容，这些方向不应动摇：

### 3.1 自动战斗主闭环

- 自动推进、索敌、战斗、击杀、结算、节点推进已经跑通
- 普通怪、精英、Boss、章节推进都已接入
- 游戏已经具备基础的挂机战斗可运行性

### 3.2 刷装与构筑主循环

- 已有装备掉落、词条、传奇特效、自动穿戴、自动分解、手动背包管理
- 已有图鉴、追踪目标、掉落统计、推荐刷图等配套系统
- 已经具备“刷什么、留什么、换什么、追什么”的基本雏形

### 3.3 长期成长骨架

- 已有研究树
- 已有多章节推进
- 已有更高品质掉落池和 Boss 特殊掉落方向
- 已有离线收益与离线报告

### 3.4 基础体验修正

- 已修复角色无怪时冲出屏幕的问题
- 已补更自然的待机回位
- 自动战斗连续性和稳定观看体验明显改善

### 3.5 交互层重构

- HUD 已从调试面板改成常驻信息区
- 已有统一导航栏、统一面板管理器、输入屏蔽逻辑
- 背包、研究、图鉴、统计已具备基础层级与状态提示

这些内容说明项目并没有偏离最初产品方向，主骨架是正确的。

## 4. 当前疑似过深或可暂缓项

目前项目没有走错方向，但确实存在一些“阶段上做得偏深”的部分。

### 4.1 掉率统计与期望值系统做得过深

已经投入较深的部分包括：

- 图鉴追踪
- 推荐刷图
- 理论掉率
- 实测样本
- 偏差状态
- 统计可视化

这套系统对硬核刷装玩家有价值，但在当前阶段，它更像一个“增强解释力”的深度层，而不是最直接决定留存的前台体验。

### 4.2 UI 细修迭代次数偏多

从 `v3.0` 到 `v3.3`，连续多轮都在做：

- 分区
- 遮罩
- 二级页层级
- 状态提示
- 配色强化

这些工作并非无效，但已经进入“让现有面板更像产品”的阶段。相较之下，部分更直接影响留存的内容事件和战斗观赏性仍偏弱。

### 4.3 架构与系统拆分存在一定超前投入

例如：

- `GameManager` 拆职责
- `LootSystem` 再拆生成层与结算层
- 多个系统边界持续重构

这些工作会提升后续扩展能力，但如果继续投入过深，容易让项目在“工程更规整”与“玩家体验更强”之间出现优先级错位。

## 5. 留存导向必补清单

如果目标是提高玩家回流和持续挂机意愿，接下来最重要的不是继续深挖统计，而是补齐下面这些直接影响留存的层。

### 5.1 回流钩子

当前离线收益已具备，但仍缺少更明确的“为什么今天要再开一次游戏”的理由。

建议补充：

- 每日/周期目标
- 当前 build 成型进度
- 离线回来后的重点提示
- 即将解锁内容预告
- 当前追踪目标距离达成还差什么

理想状态是：玩家重新打开游戏后，不只是看到“收了多少资源”，还会立刻看到“接下来该做什么”。

### 5.2 更明确的 build 成型路径

虽然已经有词条、传奇和追踪，但当前仍偏系统侧。

建议强化：

- 每个核心流派的标准成型路径
- 新手 build 推荐
- 中期 build 转型点
- 当前构筑缺口提示
- “下一件目标装备 / 下一条关键词条”提示

这会直接提升玩家对成长方向的把握感，降低“系统很多但不知道该追什么”的问题。

### 5.3 阶段事件感

长期成长要想支撑留存，不能只是更高数字，必须有阶段事件。

优先补的事件包括：

- 突破事件
- 新章节开启
- Boss 首杀
- 新流派节点解锁
- 新掉落池开放
- 关键传奇首次获得

这些事件要在 UI 和反馈层面被明确放大，让玩家感到“我刚完成了一个阶段”。

### 5.4 战斗观赏性

项目是桌面挂机，不代表战斗可以只是功能正确。

仍需补强：

- 核心技高光
- 命中与暴击反馈
- 精英/Boss 出场与死亡反馈
- 稀有掉落瞬间的强提示
- 当前战斗节奏点的视觉锚点

挂机玩家很多时候并不是在操作，而是在“偶尔看一眼”。如果这一眼不够爽，长期留存会受影响。

### 5.5 内容节奏与可追求物

当前系统骨架已经不少，但“持续可追求的内容密度”仍不够高。

后续应该继续补：

- 更明确的章节主题差异
- 每章独有掉落价值
- 更强的 Boss 识别度
- 稀有度分层更清晰的奖励池
- 研究树后段的阶段追求
- 与流派强绑定的高价值目标

## 6. 作为完成产品仍缺少的部分

如果把当前项目视为“可玩原型”，那离“完整产品”还差的，不只是更多功能，还包括包装层、资源层、展示层与发行层。

### 6.1 题材与世界观包装

当前项目更像“系统原型”，缺少玩家对世界本身的记忆点。

仍需补充：

- 明确题材基调
- 世界观背景
- 玩家角色身份
- 敌人阵营和区域设定
- 各章节为什么存在、彼此有什么区别
- Boss 的来历与主题包装

没有这一层，产品容易显得只是“挂机系统合集”，而不是一个有形象的游戏。

### 6.2 角色与敌人包装

仍缺少：

- 主角形象与职业包装
- 流派和武器的视觉识别
- 敌人族群分类
- 精英与 Boss 的独特外观和战斗人格
- 核心敌人的识别标签

目前玩家记住的是系统名词，未来要逐步变成“记住内容对象”。

### 6.3 场景与章节表现

还需要：

- 每章明确的视觉主题
- 前景 / 中景 / 背景分层
- 章节切换时的场景差异
- Boss 房和普通节点的视觉区别
- 特殊节点或事件节点包装

当前如果章节只是在数据上不同，而不是视觉上不同，长期推进的成就感会偏弱。

### 6.4 战斗特效与 UI 动效

这是完成产品极缺的一层。

仍需补充：

- 普攻与核心技特效
- 命中特效、暴击特效、处决特效
- 聚怪、护盾、破甲、流血等状态表现
- 稀有掉落闪光与弹出反馈
- 面板开关动效
- 结果卡片动效
- 章节解锁与 Boss 击败的结果演出

当前 UI 已开始结构化，但“动起来的反馈”还远远不足。

### 6.5 美术资源层

当前项目明显缺少完整的美术资产体系。

缺口包括：

- 主角立绘/像素或 2D 动画资源
- 敌人与 Boss 美术资源
- 场景背景与关卡装饰资源
- 技能图标
- 装备图标
- 资源图标
- 按钮、面板、边框、标签等 UI 框体资源
- 掉落结果卡与品质边框资源

没有这一层，产品难以脱离“功能原型”观感。

### 6.6 音频层

当前仍需补：

- 主界面与战斗 BGM
- 章节/Boss 音乐差异
- 掉落与强化提示音
- UI 开关音效
- Boss 击杀与高价值掉落音效

挂机游戏尤其依赖轻量但有辨识度的音频反馈。

### 6.7 新手体验与理解层

完成产品必须补齐：

- 新手引导
- 第一局玩法教学
- build 选择解释
- 卡关提示
- 关键系统解锁提示
- 当前目标建议

否则系统越多，越可能把新玩家劝退。

### 6.8 任务与收藏驱动

为了让中长期留存更强，还需要考虑：

- 每日/周常目标
- 成就系统
- 图鉴完成奖励
- Boss 首杀记录
- build 收藏或推荐页
- 阶段性挑战目标

这些不是最先做，但对长期活跃很重要。

### 6.9 发行与展示准备

如果目标走向 Steam 产品，还缺：

- 游戏标题包装
- 宣传语与卖点提炼
- 商店页截图规划
- Trailer 思路
- Logo 与主视觉
- 教程截图与核心玩法展示点

没有这些，产品即使可玩，也难进入“可展示、可传播、可上架”的状态。

### 6.10 工程与发行基础能力

正式产品阶段还需要补：

- 分辨率与窗口模式适配
- 常驻桌面体验优化
- 性能与内存稳定性
- 存档鲁棒性
- 配置与资源加载稳定性
- 异常恢复
- 基础数据校验与调参工具

这些不会直接提升爽感，但会直接影响产品质量底线。

## 7. 优先级建议

接下来建议按 `P0 / P1 / P2` 三层推进，而不是继续平均发力。

### 7.1 P0：不补就难以验证留存

- 回流钩子与每日目标感
- build 成型路径与推荐目标提示
- 更强的阶段事件感
- 战斗观赏性与高光反馈
- 章节/Boss 的差异化价值

这部分决定玩家会不会持续打开、持续挂机、持续追目标。

### 7.2 P1：补上后更像可卖的产品

- 题材与世界观包装
- 角色与敌人形象包装
- 场景主题差异
- 战斗特效与 UI 动效
- 图标、框体、掉落卡等基础美术资源
- 音频体系
- 更完整的新手体验

这部分决定产品能不能从“原型”进入“成型产品”阶段。

### 7.3 P2：后续打磨与扩展层

- 更深的掉率统计和推荐数学
- 更精细的 UI 微交互
- 更细的系统拆分与工程整洁度提升
- 成就、收藏、挑战模式等中后期扩展层

这些很有价值，但不应该继续抢走 P0 和 P1 的优先级。

## 8. 未来 2 到 4 周建议聚焦

如果下一阶段只能集中做一轮，我建议按下面顺序推进：

### 8.1 第一优先：补“回来就有事做”的产品层

优先做：

- 回流目标
- build 推荐与目标缺口提示
- 阶段事件反馈
- 稀有掉落和解锁结果卡

目标是先把“为什么第二天还会回来”做出来。

### 8.2 第二优先：补主战斗区观赏性

优先做：

- 核心技特效
- Boss 与精英高光
- 稀有掉落高光
- 战斗区更明确的信息锚点

目标是让首页真正成为“可挂着看”的战斗舞台。

### 8.3 第三优先：开始进入产品包装层

优先做：

- 题材定位
- 世界观与章节主题
- 主角与敌人包装
- 第一批 UI 图标和框体资源方案

目标是让项目不再只是系统可玩，而开始具备产品辨识度。

## 9. 当前结论

当前项目没有偏离最初目标，但存在明显的优先级漂移：

- 核心方向仍然正确
- 系统骨架已经足够丰富
- 但中后期在 `掉率统计深挖` 和 `UI 细修` 上投入偏深

如果下一阶段继续停留在局部系统优化，项目会越来越像一个“结构完整的原型”，而不是一个“让人愿意持续回来挂机的产品”。

因此后续最重要的判断标准应该始终是：

- 它是否能提升次日回流动机
- 它是否能加快 build 变化感知
- 它是否能增强挂机旁观爽感
- 它是否能让项目更接近一个可包装、可展示、可发行的完整产品
