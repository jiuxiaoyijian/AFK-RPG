## 一、项目目标

开发一款 **Steam 桌面武侠挂机 RPG**。

- **平台**：PC (Steam)，横版2D
- **引擎**：Godot 4.6+，GDScript
- **核心玩法**：自动战斗表演 + 装备驱动的 Build 构筑 + 无限进阶
- **题材**：武侠江湖（详见 `10_世界观与题材包装.md`）
- **玩家身份**：青云门新入门的外门弟子

---

## 二、设计哲学

借鉴暗黑破坏神III的核心乐趣公式，适配挂机自动战斗：

```
暗黑3玩家:  操作战斗 → 看掉落 → 换装备 → 调Build → 冲更高层
本项目玩家:  观看战斗 → 看掉落 → 换装备 → 调Build → 冲更高层
             ↑                                        ↑
        自动化+表演感                            决策感不能丢
```

玩家乐趣 = **战斗表演观赏 + 掉落多巴胺 + Build构筑 + 无限进阶**

## 六、MVP 范围

最小可玩版本需要：
1. 2个章节（青云山外 + 落雁谷），各含普通/精英/Boss节点
2. 3个流派可选（御风刀/血劫手/五雷掌）
3. 装备掉落 + 自动/手动穿戴
4. 多乘区伤害公式可验证
5. 离线收益
6. 存档/读档

---

## 七、AI 执行硬性要求

1. **代码语言**：GDScript，不使用 C# 或 GDNative
2. **数据驱动**：所有可调参数放在 `data/` 目录的 JSON 文件中，不在脚本中硬编码
3. **命名规范**：代码用英文 snake_case，面向玩家的显示名用中文，参照 `13_武侠命名映射表.md`
4. **Autoload限制**：已注册的 Autoload 不可随意新增，修改需说明理由
5. **不破坏现有功能**：每次修改后确保项目可运行，不引入新的 SCRIPT ERROR
6. **存档兼容**：任何影响存档结构的变更必须递增 `CURRENT_SAVE_VERSION`
7. **文档引用**：系统设计以 `14_暗黑3式系统框架设计.md` 为最终依据，本文档和其他文档为实施指南

## 八、推荐目录结构

```

├── assets/                  # 美术/音频资源
│   ├── generated/           # AI/脚本生成的资源
│   ├── fonts/               # 字体
│   └── theme/               # UI主题
├── data/                    # 数据驱动的 JSON 配置
│   ├── chapters/            # 章节定义
│   ├── drops/               # 掉落表
│   ├── enemies/             # 敌人定义
│   ├── equipment/           # 装备/词缀/传奇
│   ├── skills/              # 技能定义
│   ├── sets/                # 传承(套装)定义 [P1新增]
│   └── progression/         # 研究树/成长
├── scenes/                  # Godot 场景文件
│   ├── main/                # 主场景
│   └── ui/                  # UI场景
├── scripts/                 # GDScript 脚本
│   ├── autoload/            # 全局管理器
│   ├── combat/              # 战斗计算
│   ├── effects/             # 视觉效果
│   ├── entities/            # 玩家/敌人实体
│   ├── systems/             # 系统脚本(装备生成/战斗/掉落)
│   ├── ui/                  # UI控制器
│   └── utils/               # 工具类
└── 文档/                    # 设计文档
```
