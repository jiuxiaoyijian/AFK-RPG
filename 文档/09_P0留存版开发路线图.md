# P0 留存版开发路线图

## 1. 文档定位

本文档用于回答一个非常具体的问题：

在不继续分散到深层统计、UI 细修和过早包装工作的前提下，未来 `2-4` 周应该优先做什么，才能最快验证这个项目有没有持续留住玩家的潜力。

这里的 `P0` 不代表“最终版本最重要的一切”，而是代表：

- 不补就难以验证留存
- 不补就无法建立清晰的回流理由
- 不补就会让项目继续停留在“结构完整的原型”

## 2. 当前判断

基于现有文档与进展记录，当前项目已经具备：

- 自动战斗主闭环
- 掉落、换装、词条与图鉴追踪
- 研究树与多章节推进
- 离线收益与基础交互层

但最关键的留存缺口仍在三个地方：

1. 玩家回来后缺少明确的“今天要做什么”
2. build 成型路径存在系统，但缺少前台指引
3. 战斗虽可运行，但还不够有高光，不够支撑挂机观看

因此接下来的 P0 路线图必须围绕：

- 回流钩子
- build 目标感
- 阶段事件感
- 战斗观赏性

## 3. P0 阶段目标

本轮路线图只追求四个结果：

### 3.1 玩家重新打开游戏后，立刻知道下一步做什么

需要做到：

- 离线结算后不仅显示资源，还显示当前建议目标
- 当前 build 还差什么可以一眼看到
- 最近达成了什么阶段事件可以一眼看到

### 3.2 玩家在挂机中，持续感到“我正在朝一个目标靠近”

需要做到：

- 不是只看随机掉落
- 而是看到目标装备、目标词条、目标章节、目标研究的进展

### 3.3 玩家在主界面偶尔看一眼时，会看到值得停留的高光

需要做到：

- 稀有掉落有强反馈
- 精英与 Boss 有高光反馈
- 核心技和 build 特色有视觉辨识

### 3.4 玩家在卡关时，有明确的转向建议

需要做到：

- 告诉玩家是该补伤害、补生存，还是回刷某个节点
- 告诉玩家距离下一个成型点还差什么

## 4. 本轮不做什么

为了保证优先级，本轮建议明确暂缓以下内容：

- 更深的掉率统计、理论值、期望值继续扩展
- 继续细拆系统职责或做工程洁癖式重构
- 继续深挖二级页视觉微调
- 大规模世界观文本扩写
- 大量复杂剧情或城镇系统
- 完整成就与收藏系统

这些内容不是不做，而是不应继续抢走 `P0` 的资源。

## 5. 2 到 4 周开发拆分

## 第一阶段：回流目标层

目标：

- 让玩家离线回来后立刻看到“当前最值得做的事”

建议补充：

- `今日目标 / 当前目标` 卡
- 离线收益结算中的“推荐下一步”
- 当前追踪目标的完成进展
- 即将解锁内容预告
- 当前研究、章节、目标装备中最接近完成的一项

建议落地形式：

- 主 HUD 增加轻量目标区
- 离线收益弹层增加 `回来先做什么`
- 图鉴/背包/研究中补“当前目标进展”

验收标准：

- 玩家打开游戏后 10 秒内能知道当前推荐目标
- 不打开深层面板也能理解目前主要追求

## 第二阶段：build 目标层

目标：

- 让 build 从“系统上存在”变成“玩家能明确追”

建议补充：

- 每个核心流派的推荐成长路径
- 新手 build 推荐
- 中期 build 转向建议
- 当前 build 缺口提示
- 下一件目标装备 / 下一条目标词条提示

建议落地形式：

- 在图鉴或目标页增加 `当前流派推荐`
- 在背包或 HUD 中增加 `build 诊断`
- 在统计页或追踪页增加 `推荐刷这个节点的原因`

验收标准：

- 新玩家或回流玩家能知道“当前流派下一步要追什么”
- 卡关时至少有 2 到 3 条明确建议，而不是只能盲刷

## 第三阶段：阶段事件层

目标：

- 把成长节点做成“事件”，而不是静默数值提升

建议补充：

- 章节解锁结果卡
- Boss 首杀结果卡
- 研究节点升级关键提示
- 关键传奇首次获得提示
- 突破或阶段完成提示

建议落地形式：

- 统一结果卡模板
- 用于章节、Boss、研究、传奇、追踪达成等多个场景
- 与现有离线收益和最近掉落风格保持一致

验收标准：

- 玩家完成一个阶段后，会明显感到“我过了一道坎”
- 关键时刻不再只是文本一闪而过

## 第四阶段：观赏性增强层

目标：

- 让首页真正具备挂机观看价值

建议补充：

- 核心技命中特效
- 暴击、处决、连锁、流血结算等关键反馈
- 精英与 Boss 出场/死亡强化反馈
- 稀有掉落闪光与结果高光
- 主战斗区更明确的信息锚点

建议落地形式：

- 先做轻量级伪特效和高亮文字
- 再逐步补正式特效资源和更完整的动效

验收标准：

- 玩家瞟一眼就能区分普通战斗、高光击杀、稀有掉落
- 三个核心流派至少有基础视觉辨识

## 6. 推荐功能清单

为了方便执行，可以把 P0 拆成以下具体功能块。

### 6.1 回流与目标

- 当前目标卡
- 今日建议任务
- 离线回来推荐下一步
- build 成型进度条
- 目标装备/词条缺口提示

### 6.2 成长事件

- Boss 首杀结果卡
- 新章节解锁卡
- 研究升级关键提示
- 传奇首次获得提示
- 追踪目标达成提示

### 6.3 观看体验

- 核心技高光
- 精英与 Boss 警示和高亮
- 稀有掉落高亮
- 节奏点提示
- 流派颜色与伪图标强化

## 7. 开发顺序建议

建议严格按下列顺序推进：

1. 回流目标层
2. build 目标层
3. 阶段事件层
4. 观赏性增强层

原因：

- 先解决“玩家回来做什么”
- 再解决“玩家为什么继续追”
- 然后强化“达成时为什么有成就感”
- 最后补“挂机时为什么愿意继续看”

## 8. P0 期间的判断标准

如果这一轮做完，应该能回答下面几个问题：

- 玩家第二天打开游戏时，会不会比现在更清楚接下来该做什么
- 玩家是否更容易感知 build 是否在成型
- 玩家是否更容易因为阶段事件而产生继续推进的冲动
- 首页是否更有挂机观赏性，而不只是一个系统入口面板

如果这几个问题仍然回答不出“是”，就说明项目还不该继续往统计深度或包装层分散。

## 9. P0 完成后的下一步

只有当这轮 P0 补完并基本验证后，才建议继续投入：

- 题材与世界观包装
- 场景与章节主题强化
- 正式美术资源与图标体系
- 更完整的新手体验
- 更系统的 Steam 展示物料准备

也就是说：

- `P0` 解决留存验证
- `P1` 再进入产品包装

## 10. 当前结论

未来 `2-4` 周最重要的不是再做更深的系统，而是补齐“目标感、事件感、观赏性”。

只要这三层没有立住，项目就会继续停在：

- 系统很多
- 面板很完整
- 但玩家不一定会持续回来

而一旦这三层立住，项目才真正具备进入下一阶段产品化包装的条件。
