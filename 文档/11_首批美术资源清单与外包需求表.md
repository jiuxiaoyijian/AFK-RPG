# 首批美术资源清单与外包需求表

## 1. 文档定位

本文档用于把现阶段产品需求转化成可执行的首批美术资源清单。

它主要服务三个目标：

- 给项目当前 `P0/P1` 阶段确定最先要做的资源范围
- 给外包、合作美术或本地 AI 生成确定统一的风格和交付标准
- 避免资源制作顺序失控，先做真正影响留存和产品观感的部分

本文档基于以下文档整理：

- `08_产品化缺口与后续优先级清单.md`
- `09_P0留存版开发路线图.md`
- `10_题材与世界观包装草案.md`
- `12_开发排期表.md`
- `13_反桃花源修真命名包装表.md`

## 2. 当前资源策略

现阶段不建议一开始追求“大而全”的完整美术包，而建议按三层资源策略推进：

### 2.1 第一层：验证留存与观感的关键资源

优先服务：

- 主战斗区观赏性
- 稀有掉落与阶段事件反馈
- 流派辨识
- 基础 UI 产品感

### 2.2 第二层：建立产品包装一致性

优先服务：

- 世界观统一
- 章节主题差异
- 敌人/Boss 识别
- 商店页截图质量

### 2.3 第三层：扩内容量与长线资源

优先服务：

- 新章节
- 新流派
- 收藏/成就
- 长线运营物料

## 3. 统一美术方向

所有首批资源建议统一遵守以下方向：

### 3.1 题材基调

- 反桃花源修真
- 山水伪仙境
- 香火、祠庙与洞天秩序
- 温润表象下的异化生灵与守祠役众

### 3.2 视觉关键词

- 雾青山色
- 枯桃粉与旧白
- 墨青、血朱、香火金
- 符纸、桃木、古桥、祠堂、纸灯
- 灵田、井口、雾渡、骨灯
- 越好看，越不对劲

### 3.3 风格要求

- 不要做成明亮轻卡通
- 不要做成常规仙侠手游套皮风
- 建议采用 `半写实 2D 游戏美术` 或 `风格化写实 UI + 角色立绘/精灵`
- 保持 `山水桃源表象 + 祠庙异化内核` 的统一感

### 3.4 当前执行口径

为了避免资源制作时又回到旧废土方向，首批资源默认采用以下正式口径：

- 章节：`桃溪外渡 / 祠下灵田`
- Boss：`伏祠山君 / 祭炉监院`
- 流派：`御风道 / 血劫道 / 五雷道`
- 系统：`今日机缘 / 悟道 / 异闻录 / 机缘推演 / 闭关所得`
- 资源：`香火钱 / 祠灰 / 灵核 / 真意残片`

如果外包、AI 出图或 UI 设计稿中出现旧词，应视为需要退回修正，而不是作为可接受变体。

## 4. 资源优先级总览

建议按 `A/B/C` 三层推进。

### A 级：必须最先做

- 主角基础表现资源
- 三个核心流派的图标与高光资源
- 两章场景背景
- 关键敌人和 Boss 识别资源
- 基础 UI 图标与品质边框
- 稀有掉落/阶段事件结果卡资源

### B 级：第二批做

- 章节扩展资源
- 精英/Boss 演出资源
- 背包/研究/图鉴/统计面板的正式框体
- 更多装备图标与敌人变体

### C 级：后续扩量

- 新章节整包资源
- 成就/收藏/挑战模式资源
- 商店页宣传视觉扩展资源

## 5. 首批美术资源清单

以下清单以“当前最该先做什么”为原则。

## 5.1 角色资源

### 5.1.1 主角基础战斗资源

用途：

- 主战斗区常驻表现
- 提升挂机观看价值

建议内容：

- 待机
- 前进
- 普攻
- 受击
- 死亡
- 胜利/收招

建议数量：

- 1 套主角基础动作

备注：

- 如果预算紧，优先保证 `待机 / 前进 / 普攻 / 核心技姿态`

### 5.1.2 主角流派辨识资源

用途：

- 让三种流派不只是数值差异，而有明显观感差异

建议内容：

- `御风道` 的风痕、桃叶与刃气特征
- `血劫道` 的朱砂血线与伤痕印记
- `五雷道` 的雷箓、电痕与灼白爆点

建议数量：

- 3 组流派视觉标识

## 5.2 敌人与 Boss 资源

### 5.2.1 第一章敌人

当前对象：

- 桃瘴鼠
- 纸甲巡祠
- 香灰巫祝
- 伏祠山君

需求层次：

- 普通怪基础形象
- 精英有额外轮廓和颜色识别
- Boss 有更大体型和独特 silhouette

### 5.2.2 第二章敌人

当前对象：

- 盗火佃客
- 炽骨犬
- 守田傀
- 护炉傩兵
- 引雷役修
- 祭炉监院

需求层次：

- 能明显从第一章敌人中区分出来
- 强化“祠下灵田 + 香火伪仙”风格

### 5.2.3 首批外包建议

如果预算有限，敌人与 Boss 可按以下顺序做：

1. `伏祠山君`
2. `祭炉监院`
3. `香灰巫祝`
4. `护炉傩兵`
5. 普通怪基础套

原因：

- 先做 Boss 和精英，最能直接提升截图质量和战斗观感

## 5.3 场景资源

### 5.3.1 第一章：桃溪外渡

建议制作：

- 远景背景
- 中景渡口、木桥、村舍
- 前景路牌、符桩、纸灯
- 地面贴图
- 环境点缀：山雾、桃瓣、香炉、枯井口

### 5.3.2 第二章：祠下灵田

建议制作：

- 祠庙背景
- 灵田埂道与祭坛平台
- 香炉、幡旗与灌溉水渠
- 地面符纹与井口裂痕
- 香火雾与灵气叠层

### 5.3.3 首批场景交付建议

每章至少拆成：

- 1 张远景
- 1 张中景
- 1 套地面/平台
- 1 套前景装饰

## 5.4 UI 资源

### 5.4.1 基础图标包

优先级最高的图标：

- 三个核心流派图标
- 香火钱 / 祠灰 / 灵核 / 真意残片资源图标
- 背包 / 悟道 / 异闻录 / 机缘推演导航图标
- 装备部位图标
- 稀有度边框图标

补充建议：

- 图标不要做成通用仙侠手游圆润小图标
- 优先强调 `纸符 / 木牌 / 香炉 / 旧器 / 祠印` 这类可识别物件
- 流派图标应能在小尺寸下看出 `风 / 血 / 雷` 的差异

### 5.4.2 框体与面板

优先级最高的 UI 外观资源：

- 主 HUD 卡片框体
- 二级页面板底板
- 按钮基础状态
- 结果卡片底板
- 稀有掉落高亮框

补充建议：

- 主 HUD 与结果卡应优先保证“截图时能看出题材”
- 二级页面框体可以晚于首页 HUD，但不能和首页风格割裂
- 当前布局已较稳定，首批切图应优先适配现有结构，避免引入大规模返工

### 5.4.3 结果卡片资源

用于：

- 章节解锁
- Boss 首杀
- 稀有掉落
- 研究升级
- 离线收益重点结果

建议资源：

- 通用结果卡模板
- 稀有度边框
- 香火金高亮版本
- 血朱 / 雷白强化版本

## 5.5 特效资源

### 5.5.1 战斗特效

首批建议：

- 普攻命中特效
- 暴击高亮
- 旋风斩高光
- 流血叠层/处决提示
- 雷链弹射与命中爆点

### 5.5.2 掉落与事件特效

首批建议：

- 稀有掉落闪光
- Boss 击杀高亮
- 章节解锁闪屏
- 悟道突破提示光效

### 5.5.3 UI 动效辅助资源

首批建议：

- 按钮 hover/press 变化
- 面板弹出遮罩过渡
- 结果卡展开亮边

## 6.1 外包 / AI 出图补充说明

如果使用外包或本地 AI 生成，建议额外附上以下要求：

- 先出 `风格样张`，确认题材正确后再批量生产
- 每个模块至少提供 `1 张正向样例 + 1 张反例说明`
- 反例重点写清：不要废土工业、不要通用仙侠手游、不要甜美桃源风
- 角色和 Boss 优先做 silhouette 区分，再做纹理细节
- UI 先做首页最常见资源，再扩到二级面板

## 7. 建议交付规范

为了后续接入 Godot，建议统一要求：

### 7.1 文件组织

- 角色、敌人、Boss、场景、UI、特效分目录交付
- 文件命名必须稳定，避免交付后还要大量重命名

### 7.2 分辨率建议

- UI 图标优先支持高清缩放
- 角色与敌人建议统一基准尺寸
- 背景建议支持 16:9 主分辨率

### 7.3 透明与切图要求

- 角色、图标、UI 框体、特效尽量透明背景
- 面板和结果卡尽量采用可切分结构，方便后续适配不同尺寸

### 7.4 风格统一要求

- 所有资源必须围绕同一题材关键词
- 严禁不同外包来源之间风格完全割裂
- 先确认 1 组风格样张，再批量生产
- 交付预览图中应尽量直接使用当前正式命名，避免后期重新整理标注

## 8. 推荐制作顺序

建议按下面顺序推进：

1. 主角基础动作 + 三流派高光
2. 两章场景背景
3. 两个 Boss
4. 基础 HUD/UI 图标与结果卡模板
5. 第一批战斗与掉落特效
6. 普通怪与精英补全

原因：

- 这条顺序最能快速提升首页观感、截图质量和留存关键体验

## 9. 当前建议

如果只能立刻启动第一批资源制作，建议首批就做下面这 8 项：

1. 主角基础动作套
2. 三流派图标与高光标识
3. 桃溪外渡背景组
4. 祠下灵田背景组
5. 伏祠山君 Boss
6. 祭炉监院 Boss
7. 基础 UI 图标包
8. 稀有掉落/章节解锁结果卡模板

这 8 项最能同时提升：

- 首页观感
- 流派辨识
- 章节差异
- 稀有反馈
- 产品截图质量

## 10. 当前结论

首批美术资源不应平均铺开，而应首先服务：

- `P0 留存验证`
- `主战斗区观感`
- `章节/Boss 识别`
- `结果卡与稀有反馈`

只要这批资源先立住，项目就会比现在更快从“系统原型”进入“有产品观感的原型”阶段。
