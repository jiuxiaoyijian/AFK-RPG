# 交互设计规范

> 本文档定义本项目交互设计的通用原则、边界处理通则和线框图规范。  
> 所有系统的交互原型文档（S3 阶段）和边界补全（S4 阶段）均需遵循本规范。

**管线位置**：[研发流程总览](../00_项目总纲/研发流程总览.md) S3+S4 阶段

---

## 一、交互设计原则

### 1.1 四项核心原则

| # | 原则 | 含义 | 本项目的具体约束 |
|---|------|------|----------------|
| 1 | **一致性** | 相同操作在不同系统中产生相同反馈 | 同类按钮（确认/取消/返回）的位置、尺寸、配色在全项目统一 |
| 2 | **反馈即时性** | 每个玩家操作都必须有即时可感知的反馈 | 点击 → 100ms 内必须有视觉/音效反馈；3 秒无响应必须出 loading 提示 |
| 3 | **容错优先** | 破坏性操作必须可确认/可撤销 | 分解/出售/重置等操作必须有二次确认弹窗；关键操作保留「后悔」窗口 |
| 4 | **最少操作路径** | 核心功能入口不超过 3 次点击 | 从 HUD 到任何一级系统面板 ≤ 2 次点击；从面板到核心操作 ≤ 1 次点击 |

### 1.2 挂机场景特有约束

本项目是**挂机 RPG**，因此交互设计还需满足：

- **5 分钟登录场景**：登录 → 收取离线收益 → 查看今日目标 → 退出，整条路径 ≤ 4 步操作
- **长时间观战场景**：战斗画面在无操作状态下必须保持信息可读，HUD 不遮挡战斗区域核心
- **碎片化操作场景**：关键决策（换装/技能配置/Build 调整）不需要频繁跨面板，单面板内完成

## 二、面板层级体系

### 2.1 层级定义

| 层级 | 名称 | 说明 | 示例 |
|------|------|------|------|
| L0 | **战斗画面** | 始终可见的底层，战斗在此进行 | 角色、怪物、特效、视差背景 |
| L1 | **常驻 HUD** | 覆盖在战斗画面上的轻量信息层 | HP 条、资源栏、底部导航栏、右上角关卡信息 |
| L2 | **系统面板** | 点击导航栏后展开的功能面板 | 背包、技能、百炼坊、武学参悟、秘境 |
| L3 | **详情弹窗** | 在系统面板内打开的详细信息窗口 | 装备详情、技能说明、对比栏 |
| L4 | **模态弹窗** | 需要玩家必须处理的阻塞式弹窗 | 二次确认、首次获得、引导步骤 |

### 2.2 层级规则

- L0 → L1 **始终并存**，L1 不得遮挡 L0 的战斗走廊中央区域
- L2 打开时 **L0 暂停或半透明**（挂机可继续，但视觉降级）
- L3 打开时 **L2 保持可见但不可交互**
- L4 打开时 **所有下层不可交互**（模态阻塞）
- 每次只能打开 **1 个 L2 面板**（互斥）
- L3 和 L4 可叠加，但不超过 2 层

## 三、通用交互模式

### 3.1 导航模式

| 模式 | 使用场景 | 规范 |
|------|---------|------|
| **底部 Tab 导航** | L1 → L2 系统面板切换 | 固定 4-6 个入口；当前 Tab 高亮；支持快捷键 |
| **左侧列表 + 右侧详情** | L2 面板内的主从视图 | 列表默认选中第一项；点击列表项更新右侧 |
| **弹窗式详情** | L3 详情展示 | 右上角关闭按钮；点击空白区域也可关闭 |
| **全屏接管** | 特殊仪式场景（Boss 击杀、橙装拾取） | 自动播放 → 点击任意处跳过 → 进入结算 |

### 3.2 反馈模式

| 操作类型 | 反馈方式 | 时间要求 |
|---------|---------|---------|
| 普通点击 | 按钮缩放 + 点击音效 | ≤ 50ms |
| 装备穿戴 | 装备飞入动画 + 属性刷新 | ≤ 300ms |
| 品质掉落 | 光柱 + 音效 + 慢镜（按品质分级） | 200ms-2500ms |
| 数值变化 | 数字跳字动画（起始值 → 终值） | 500ms-1000ms |
| 错误操作 | 按钮抖动 + 提示文字 | ≤ 100ms |

## 四、8 类标准边界通则

每个系统在 S4 阶段必须逐条检查以下 8 类边界。

### 4.1 首次使用

| 检查项 | 要求 |
|--------|------|
| 首次打开面板 | 有引导覆盖层或高亮提示，告知核心操作 |
| 引导触发条件 | 明确定义（如"首次获得蓝色装备后打开背包"） |
| 引导可跳过 | 必须可跳过，跳过后不再重复触发 |
| 跳过后重看 | 设置页提供"重置引导"选项（P2 优先级） |

### 4.2 空状态

| 检查项 | 要求 |
|--------|------|
| 空列表/空背包 | 显示占位插画 + 引导文案（"前往历练获取装备"） |
| 空数据面板 | 不显示空白，而是显示预期内容的描述 |
| 不可出现 | 空白屏幕、无文字的空框、只有边框的区域 |

### 4.3 满载状态

| 检查项 | 要求 |
|--------|------|
| 背包/仓库满 | 弹窗提示 + 快捷分解/出售入口 |
| 槽位满 | 显示"已满"标记 + 提示替换流程 |
| 等级到顶 | 经验条显示"已满级"，不再显示获取经验的提示 |

### 4.4 错误与异常

| 检查项 | 要求 |
|--------|------|
| 配置数据缺失 | 使用默认值 + 日志警告，不崩溃 |
| 引用 ID 不存在 | 显示占位信息（"未知物品"），不崩溃 |
| 数值溢出 | 设定上限，超出截断并记日志 |

### 4.5 中断恢复

| 检查项 | 要求 |
|--------|------|
| 操作中途退出 | 已确认的操作持久化，未确认的操作丢弃 |
| 再次进入 | 面板恢复到最后一个稳定状态 |
| 多设备/多窗口 | 以最后一次保存为准（本项目为单机，暂不考虑） |

### 4.6 跨系统跳转

| 检查项 | 要求 |
|--------|------|
| 跳转路径 | 明确"从 A 系统跳到 B 系统"的入口和返回路径 |
| 返回时状态 | 回到 A 系统时保持离开前的滚动位置和选中项 |
| 跳转提示 | 跨系统跳转时用 Toast 提示目标系统名称 |

### 4.7 红点策略

| 检查项 | 要求 |
|--------|------|
| 红点触发条件 | 明确定义（如"有未穿戴的更高 DPS 装备"） |
| 红点消除条件 | 明确定义（如"打开背包面板并查看过该装备"） |
| 红点优先级 | 功能性红点 > 系统性红点 > 活动性红点 |
| 红点层级 | 导航栏红点 = 子面板红点的聚合（任一子项有红点则父级亮红点） |
| 红点上限 | 同时亮红点的 Tab ≤ 3 个（避免"全都在叫"的疲劳） |

### 4.8 新手引导

| 检查项 | 要求 |
|--------|------|
| 引导时机 | 与 L4 学习曲线里程碑对齐（5min/1h/10h/100h） |
| 引导形式 | 高亮遮罩 + 箭头 + 简短文字（≤ 15 字） |
| 引导步骤数 | 单次引导 ≤ 5 步 |
| 强制 vs 弱引导 | 首次核心操作用强制引导；非核心用弱引导（提示但不阻塞） |

## 五、线框图规范

### 5.1 制作方式

本项目不强制要求设计工具，以下方式均可：

- **Markdown ASCII 表格**：适合简单布局
- **Mermaid 图**：适合流程和状态转换
- **Excalidraw / draw.io**：适合复杂面板（导出 PNG 放入文档）

### 5.2 标注要求

每个线框图必须包含以下标注：

| 标注项 | 说明 |
|--------|------|
| **面板名称** | 明确标注这是什么面板 |
| **可交互区域** | 用边框或高亮标出所有可点击/可拖拽的区域 |
| **信息层级** | 标注哪些信息是主要的、哪些是次要的 |
| **状态枚举** | 每个可交互元素列出所有可能状态 |

### 5.3 状态枚举清单

每个可交互元素必须覆盖以下状态：

| 状态 | 说明 |
|------|------|
| **默认态** | 正常显示 |
| **选中态** | 被选中/激活时的表现 |
| **禁用态** | 条件不满足时的表现（灰化 + 原因提示） |
| **加载态** | 数据加载中的表现 |
| **空态** | 无数据时的表现 |
| **错误态** | 操作失败时的表现 |

## 六、操作流文档规范

### 6.1 格式

```markdown
### 操作流：[操作名称]

**入口**：[从哪个面板/按钮进入]  
**前置条件**：[需要满足什么条件]

1. 玩家点击 [元素A] → 系统反馈 [反馈A]
2. 玩家看到 [信息B] → 做出选择 [选择C/D]
3. ...
N. 操作完成 → 返回 [目标面板]

**异常路径**：
- 若条件不满足 → [处理方式]
- 若中途退出 → [恢复方式]
```

### 6.2 要求

- 每个核心功能至少写 1 条完整操作流
- 操作流必须包含正常路径 + 至少 1 条异常路径
- 操作步骤中的反馈必须与反馈模式（第三节）一致
