## 前言

本报告是[体验分析框架](../00_项目总纲/体验分析框架.md)定义的「流程 B · 既有文档 Review」的正式产出。

每份文档按 **L1-L5 五维度** 给出诊断表，每个维度记录 **现状 / 缺口 / 建议**，最终给出总体判定（**推翻重做 / 大幅补强 / 小修即可 / 保留**）与修订优先级（P0 / P1 / P2）。

### 判定标准回顾

| 判定 | 条件 |
|------|------|
| **推翻重做** | L1 体验锚点与项目总愿景根本冲突；或 L2-L5 中三项以上完全缺失且无法在原文档骨架上补救 |
| **大幅补强** | L1 清晰，但 L2-L5 中两项明显缺失 |
| **小修即可** | L1-L5 基本覆盖，仅个别标准未达成 |
| **保留** | 全部达成 |

### 审计边界

- 本阶段**不修订**原文档，仅产出诊断与建议
- 判定基于**文档自身内容**，不依赖"工程里其实做了"这种隐性信息
- Payoff / 仪式 / 挫败缓冲这类内容，如果**分散在其他文档**，本次只在诊断中注明但不扣减本文档的判定
- 交互重构类文档（RPG 化）与纯系统设计文档判定口径略有差异，见各节说明

## 审计 1 · 核心战斗与系统循环

**对象**：[核心战斗与系统循环.md](../01_系统设计/核心战斗与系统循环.md)  
**文档定位**：横版 2D 自动战斗的玩法规格 + 战斗表演层 + 三层循环定义 + 敌人与章节数据  

### 诊断表

| 维度 | 现状 | 缺口 | 建议 |
|------|------|------|------|
| **L1 体验锚点** | 第一节开宗明义："玩家不直接操作战斗，但战斗必须好看、必须可读，让玩家能'看懂'自己的 Build 在起作用" —— 隐含 Payoff（Build 起作用的可视化） | **未定义 Fantasy**（战斗中玩家在脑补自己是谁？一位在门派外历练的弟子？一位运筹帷幄的指挥者？）；**未定义主 Emotion**（应是"看懂"带来的**掌控感**？还是挂机回报的**收获感**？需明确） | **补强** |
| **L2 体验循环** | 第二节已区分"分钟 / 章节 / 长线"三层循环，比其他文档稍好；第四节有战斗速度控制表 | **秒循环**：缺失——每次招式释放 / 击杀 / 拾取的视听反馈节奏，第三节"战斗可读性反馈"表是雏形但未按秒级时序排布；**日循环**：缺失——一次登录能打几章？5 分钟 / 20 分钟分别能做什么？；**周 / 月循环**：缺失 | **大幅补强** |
| **L3 决策空间** | 第五节列出三流派核心武技 + 战术 / 爆发 / 被动四层技能结构，差异化基础在 | 战术武技 / 爆发武技 / 被动心法全部**只有分类无实例**——玩家实际能做的"技能搭配选择"完全缺失；未说明战斗中**是否有玩家主动决策点**（自动战斗也可以有保留手动触发的技能） | **大幅补强** |
| **L4 学习曲线** | 第七节定义了一 / 二章的节点配置，数值递进可见 | **5 分钟 / 1 小时 / 10 小时 / 100 小时四个里程碑全部空白**；未说明"新手什么时候理解暴击 / 流派 / Build"；Boss 战表演层面的**教学价值**（如第一次看到套装效果触发）未被设计 | **大幅补强** |
| **L5 情感反馈** | 第三节"战斗可读性反馈"表已是**仪式感清单的雏形**（7 条触发事件 + 视觉反馈），优于其他三份文档；第九节"离线战斗 · 出关所得"设计了"真意及以上品质逐个播放高光"，是极好的仪式点 | 第三节每条反馈只写"闪光 / 变色"等简短描述，缺**视 + 听 + 动效的完整音画流程**（如传承触发：0.1s 光柱 → 0.3s 全屏闪光 → 0.5s 文字飞出 → 整屏震动）；**挫败缓冲**未显式设计（卡关时除"系统建议回退"外无软着陆）；**可分享锚点**未明确（截图哪个面板？录屏哪个 Boss？） | **补强** |

### 总体判定 · **大幅补强**

### 修订优先级

| 优先级 | 动作 | 依据 |
|-------|------|------|
| P0 | 补 L1 Fantasy / Emotion 段落，锁定"玩家在战斗中脑补的形象 + 主情绪词" | 影响所有后续章节的叙事语境 |
| P0 | 补 L2 秒循环：为第三节现有 7 条反馈事件加上**音画时序轴** | 战斗"好看"承诺的底层兑现 |
| P0 | 补 L4 学习曲线：5 分钟 / 1 小时 / 10 小时 / 100 小时四个里程碑，每个里程碑点出"玩家此时应理解 / 解锁 / 追求什么" | 直接影响新手留存 |
| P1 | 补 L3 战术 / 爆发 / 被动武技的**实例集**（每类 ≥ 3 个），明确主动决策点 | 给 Build 决策留出空间 |
| P1 | 补 L2 日 / 周 / 月循环：定义一次登录的节奏承诺 | 和 [里程碑与版本规划](../00_项目总纲/里程碑与版本规划.md) 对齐 |
| P2 | 补 L5 仪式感详表（完整音画时序）+ 挫败缓冲方案 + 可分享锚点 | 可在美术 / 音效落地阶段同步 |

## 审计 2 · 装备·数值·Build系统

**对象**：[装备·数值·Build系统.md](../01_系统设计/装备·数值·Build系统.md)  
**文档定位**：9 槽 + 7 品质 + 38 词缀 + 7 层多乘区伤害公式 + 三流派 Build 指南  
**说明**：Phase A 的[体验分析框架](../00_项目总纲/体验分析框架.md#审计示范--对装备数值build系统的完整诊断)已给出示范诊断，本节为正式版本（内容一致，纳入本报告统一口径）  

### 诊断表

| 维度 | 现状 | 缺口 | 建议 |
|------|------|------|------|
| **L1 体验锚点** | 开头提到"不是数字大就好，而是'替换后我的 Build 变强了'的决策快感" —— 表达了 Payoff 的功能侧 | **Fantasy 完全缺失**（玩家穿上装备时脑补的是什么形象？"一身传承的武林新秀"？）；**Emotion 未具体化**（"决策快感"应细化为"期待"/"自豪"等情绪词） | **补强** |
| **L2 体验循环** | 第五节给出三段 DPS 示例（凡品 35 → 中期 201 → 毕业 12,294），展示数值跃升的**数学节奏** | 这是数学节奏不是玩家节奏；**时 / 日 / 周 / 月循环全部空白**——"第一件紫装在第几分钟 / 第一件橙装在第几小时 / 凑齐一套传承需要多久"这些玩家节奏承诺**完全缺失** | **大幅补强** |
| **L3 决策空间** | 第七节给出**三流派 Build 方向**（御风 / 血劫 / 五雷），各自强弱项明确；多乘区设计（第五节）天然避免"单一乘区最优解" | 未量化"至少 N 种可玩 Build 变体"；未承诺"最弱流派 DPS 不低于最强的 X%"的平衡底线；**换流派代价**未说明（传奇装备能否跨流派继承？传承件可否通过百炼坊转流派？） | **小修即可** |
| **L4 学习曲线** | 规则密度极高（9 槽 + 7 品质 + 7 乘区 + 38 词缀 + bucket 互斥 + 传奇词缀规则） | **四个里程碑全部空白**；未安排规则的**引入时序**——玩家什么时候第一次接触 bucket 概念？武学诊断（第八节）在什么时机推送？从凡品到绝世真意的 7 级品质让玩家**逐步学会**的路径未设计 | **大幅补强** |
| **L5 情感反馈** | 第五节末尾一句"这种跨越式成长感正是暗黑3式系统的核心爽点"点到即止；第六节装备生成规则中未体现仪式点设计 | **仪式感清单空白**（橙装掉落应经历哪 5-7 步的音画序列？绝世真意掉落是否要额外仪式？）；**挫败缓冲未设计**（玩家 10 小时没抽到橙字怎么办？Bad luck protection 未提及）；**可分享锚点未明确** | **大幅补强** |

### 总体判定 · **大幅补强**

### 修订优先级

| 优先级 | 动作 | 依据 |
|-------|------|------|
| P0 | 补 L1 Fantasy / Emotion 段落，定调"玩家穿装备时脑补的形象" | 装备系统是 Payoff 密度最高的系统，体验锚点缺失会让 38 条词缀全部"不知为何而存在" |
| P0 | 补 L2 时 / 日循环：定义"第一件紫 / 橙 / 传承件 / 绝世真意"的节奏承诺 | 与战斗文档对齐，决定配置表掉落权重的调参方向 |
| P1 | 补 L4 学习曲线：bucket 概念引入时机 / 武学诊断推送时机 / 7 级品质逐级解锁 | 直接影响文档可读性与新手教学设计 |
| P1 | 补 L5 仪式感清单：橙装落地 5-7 步音画流程 + 绝世真意专属仪式 | 与战斗文档第三节仪式清单联动 |
| P2 | 量化 L3 多样性承诺："≥ 3 种可玩 Build 变体，最弱流派 DPS ≥ 最强 70%"；定义流派切换代价 | 可在数值终调阶段补 |
| P2 | 补 L5 Bad luck protection：保底掉落规则 | 影响掉落表配置（`drop_tables.json`） |

## 审计 3 · 传承·百炼坊·秘境·宗师

**对象**：[传承·百炼坊·秘境·宗师.md](../01_系统设计/传承·百炼坊·秘境·宗师.md)  
**文档定位**：P1-P3 所有终局系统的设计蓝图，涵盖传承（套装）/ 百炼坊（卡奈魔盒）/ 试剑秘境（大秘境）/ 传奇宝石 / 宗师修为（巅峰）/ 重入江湖（赛季）  

### 诊断表

| 维度 | 现状 | 缺口 | 建议 |
|------|------|------|------|
| **L1 体验锚点** | 每个子系统都有"设计理念"小节，但**全部是机制理念**："传承提供巨额乘区E加成"、"百炼坊是材料消耗出口"、"秘境是 Build 强度标尺" | **全程缺玩家视角的 Fantasy / Emotion / Payoff**；这份文档覆盖了项目最具情绪张力的系统（套装成型 / 秘境推层 / 重入江湖），但没有一句话说"玩家凑齐 6 件御风传承的那一刻，应该感受到什么" | **大幅补强** |
| **L2 体验循环** | 第九节 P1 实施优先级 ≈ 工期排布；第四节秘境数值缩放表（1-5 层 / 6-10 层 / ... / 51+ 层）实际上是**玩家终局循环的节奏骨架**，但未显式标记为"周 / 月尺度" | **秒 / 分 / 时 / 日尺度未触及**（终局系统本身也需要秒级反馈，如传承触发、秘境清屏）；**周 / 月尺度隐含但未明示**（凑齐 6 件传承 ≈ ? 天 / 秘境 20 层 ≈ ? 周 / 重入江湖一轮 ≈ ? 月） | **大幅补强** |
| **L3 决策空间** | 本文档是 4 份中决策空间**最丰富**的：**4 套传承 × 5 种百炼坊功能 × 3 个武学秘录槽 × 4 种华戒混搭 × 6 种传奇宝石 × 宗师 4 分类修为** —— 矩阵极大 | 组合爆炸下的**最优解风险未评估**：华戒 + 6 件传承混搭是否存在一个压倒性最优？宗师修为"辅助"分类"碎银 / 经验 +1% 无上限" vs 攻击分类"+0.5% 上限 50"是否等价？**换 Build 代价未量化**（从御风切到血劫需要几小时？百炼坊"传承互转"的实际刷件成本？） | **补强** |
| **L4 学习曲线** | P1 / P2 / P3 已是**实施时序**，天然对应玩家学习时序 —— 先传承，再秘境，最后重入江湖 | **四个玩家里程碑仍全部空白**：1 小时玩家应该**不接触**终局系统？10 小时接触到传承？100 小时开始第一次重入江湖？文档里没有承诺；**武学秘录 3 槽激活**这一高密度决策点未安排引导时序 | **大幅补强** |
| **L5 情感反馈** | 第十节验收标准第 2 条 "2/4/6 件效果正确触发并体现在伤害计算中" 暗示会有视觉反馈；赛季终止的"专属奖励 / 限定外观 / 称号"（7.4 节）是典型的可分享锚点 | **仪式感清单空白**：凑齐 6 件传承的那一刻？秘境首次破 10 层 / 20 层 / 50 层的里程碑仪式？重入江湖按下确认那一刻？**挫败缓冲空白**：秘境推层打不过有没有软着陆？升级宝石失败（文档明确 30% 失败率）的挫败感怎么消化？**可分享锚点未明确**：秘境最高层排行榜截图 / 重入江湖代数？ | **大幅补强** |

### 总体判定 · **大幅补强**

### 修订优先级

| 优先级 | 动作 | 依据 |
|-------|------|------|
| P0 | 补 L1 全部 4 个子系统（传承 / 百炼坊 / 秘境 / 重入江湖）各自的 Fantasy / Emotion / Payoff | 终局系统是情绪密度最高的区段，体验锚点缺失会让玩家"打到了但不知道为何感动" |
| P1 | 补 L2 周 / 月节奏承诺：凑齐 6 件传承约多少天？秘境 20 层约多少周？重入江湖一轮约多少月？ | 与 [里程碑与版本规划](../00_项目总纲/里程碑与版本规划.md) 协同 |
| P1 | 补 L5 关键仪式点清单：6 件传承成型 / 秘境里程碑（10 / 20 / 50 层）/ 重入江湖按下确认 / 宝石升级成功 | 终局系统的"留存黏性"核心靠仪式感 |
| P1 | 评估 L3 最优解风险：传承 × 华戒混搭矩阵 + 宗师修为分类平衡 | 需要数值模拟 |
| P2 | 补 L4 四里程碑的终局系统解锁时序 | 可在系统联调阶段定 |
| P2 | 补 L5 挫败缓冲：秘境回退保护 / 宝石升级失败安抚 / 重入江湖反悔期 | 影响玩家长期留存 |

### 本文档的特殊性说明

本文档是**多系统合集**。严格来说，四个终局系统应拆为 4 份独立设计文档，每份都走完整 L1-L5。但考虑到当前阶段（Phase D 修订）的工作量，建议**先在本文档内分节补齐 L1-L5**，如果行数超过 500 行再考虑拆分（与 README 协作约定一致）。

## 审计 4 · RPG化系统与交互重构

**对象**：[RPG化系统与交互重构.md](../01_系统设计/RPG化系统与交互重构.md)

### 诊断表

## 跨文档的共性观察

审计过程中发现 4 份文档的**共同缺口模式**，值得在 Phase D 动笔前共识：

### 观察 1 · "功能理念"被当成"体验理念"

所有文档的开篇理念都在讲**系统为什么这样设计**（"多乘区相乘 > 单乘区堆叠"、"传承是 Build 定义性的"），而不是**玩家为什么会因此感动**。

**根源**：作者视角 = 设计者视角。解法 = 强制在"设计理念"前面再插一节"玩家体验承诺"。

### 观察 2 · 时间尺度断档严重

秒 / 分循环都有（战斗招式、清怪节奏），月循环也有（赛季重入），但**时 / 日 / 周尺度普遍空白**——而这正是**留存黏性**最关键的区间。

**根源**：系统设计关注单次体验，留存设计关注多次体验。两者在同一文档里被混写，结果时 / 日 / 周被挤掉。

### 观察 3 · 学习曲线完全缺失

4 份文档**没有任何一份**显式规划了 5 分钟 / 1 小时 / 10 小时的教学节奏 —— 即便这是挂机 RPG 新手留存的决定性因素。

**根源**：策划习惯写"规则"，不习惯写"规则被学会的顺序"。解法 = L4 自检清单强制化（已在[系统设计模板](../_模板/系统设计模板.md)落地）。

### 观察 4 · 仪式感"载体有 清单无"

特效 / 光柱 / 音效在文档里零散提及（战斗文档第三节、出关所得弹窗、赛季外观），但**没有一份文档有一张"本系统全部仪式时刻清单"的总表**。

**根源**：仪式感属于"跨系统横切关注点"，容易被任何单一文档遗漏。解法 = 考虑新建一份《全局仪式感清单》作为横切文档，或强制每份文档列本系统的仪式时刻。

## 对方法论（体验分析框架）的回流建议

本次审计过程中暴露出框架本身可改进的点，归档如下，供下轮框架修订参考：

1. **交互层文档**需单独定义"轻量 L 版本"（L1 + L2 秒级 + L5，免 L3 / L4）
2. **多系统合集文档**（如传承·百炼坊·秘境·宗师）需明确"是否要按子系统分别过 L1-L5，还是整体过一次"
3. **"仪式感跨系统"场景**需在框架中提及"横切仪式感清单"的编写范式
4. **P0 / P1 / P2 分级**目前靠经验，建议下版本提供更量化的分级标尺（例如："影响新手 30 分钟内留存的为 P0"）

上述 4 条不属于本次 Phase C 的交付范围，仅作为备忘。
