# 配置表字段说明

> 本文档已按当前仓库真实实现重写。
> 以 `scripts/autoload/config_db.gd` 的读取方式为唯一准绳，不再沿用旧版原型阶段的字段命名。

---

## 1. 装备基底 `data/equipment/equipment_bases.json`

当前文件顶层是数组，不是对象包裹：

```json
[
  {
    "id": "weapon_rust_blade",
    "slot": "weapon",
    "name": "锈刃",
    "base_rarity": "common",
    "required_breakthrough": "trainee",
    "item_level": 1,
    "base_stats": [
      { "stat_key": "weapon_damage", "value": 8.0 }
    ],
    "affix_pool_tags": ["whirlwind", "bleed"],
    "legendary_pool_tags": ["whirlwind", "bleed"],
    "icon_id": ""
  }
]
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 基底唯一 ID |
| `slot` | string | 装备槽位，当前运行态主用 `weapon/helmet/armor/gloves/legs/boots/accessory/belt` |
| `name` | string | 显示名 |
| `base_rarity` | string | 基底档次 |
| `required_breakthrough` | string | 解锁阶段 |
| `item_level` | int | 基础物等 |
| `base_stats` | array | 基础词条数组，元素为 `{ stat_key, value }` |
| `affix_pool_tags` | array[string] | 普通词缀抽取偏好 |
| `legendary_pool_tags` | array[string] | 武学/传奇词缀抽取偏好 |
| `icon_id` | string | 资源占位字段 |

## 2. 普通词缀 `data/equipment/affixes.json`

当前文件顶层也是数组。

```json
[
  {
    "id": "affix_crit_rate",
    "name": "会心率",
    "bucket": "crit",
    "stat_key": "crit_rate",
    "slot_tags": ["weapon", "helmet", "gloves", "accessory", "belt"],
    "archetype_tags": [],
    "value_min": 0.01,
    "value_max": 0.06,
    "roll_format": "percent",
    "rarity_weight": 80,
    "score_weight": 1.4
  }
]
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 词缀 ID |
| `name` | string | 显示名 |
| `bucket` | string | 评分与去重 bucket |
| `stat_key` | string | 实际叠加到战斗面板的键 |
| `slot_tags` | array[string] | 可出现槽位，`accessory` 兼容 `accessory1/accessory2` |
| `archetype_tags` | array[string] | 流派倾向 |
| `value_min` / `value_max` | float | 取值区间 |
| `roll_format` | string | 展示格式，当前常见为 `flat/percent` |
| `rarity_weight` | int | 抽取权重 |
| `score_weight` | float | 评分权重 |
| `source_profile_types` | array[string] | 可选字段，限制掉落来源类型 |

## 3. 武学/传奇词缀 `data/equipment/legendary_affixes.json`

当前文件顶层是数组，不是 `legendary_affixes` 对象。

```json
[
  {
    "id": "legend_whirlwind_eye",
    "name": "风眼",
    "archetype_tags": ["whirlwind"],
    "slot_tags": ["weapon", "gloves"],
    "description": "御风半径扩大，道术伤害提高。",
    "stat_key": "whirlwind_radius_percent",
    "value_min": 0.18,
    "value_max": 0.35,
    "secondary_stat_key": "core_damage_percent",
    "secondary_value_min": 0.08,
    "secondary_value_max": 0.16,
    "drop_weight": 10
  }
]
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 武学/传奇效果 ID |
| `name` | string | 名称 |
| `description` | string | 描述 |
| `archetype_tags` | array[string] | 流派倾向 |
| `slot_tags` | array[string] | 可掉落槽位 |
| `stat_key` / `value_min` / `value_max` | 主词条定义 |
| `secondary_stat_key` / `secondary_value_min` / `secondary_value_max` | 副词条定义 |
| `drop_weight` | int | 权重 |
| `source_profile_types` | array[string] | 可选字段，常用于 `boss` 专属池 |

说明：

- `MartialCodexSystem` 不再读取旧文档中的 `codex_category`，而是运行时按 `slot_tags` 自动映射到 `weapon / armor / accessory` 三类槽位。

## 4. 传承定义 `data/sets/set_defs.json`

当前真实顶层键为 `set_defs`：

```json
{
  "set_defs": [
    {
      "id": "set_wind",
      "name": "御风传承",
      "theme": "持续旋风与范围清场",
      "piece_slots": ["weapon", "helmet", "armor", "gloves", "legs", "boots"],
      "bonuses": [
        { "pieces": 2, "summary": "御风刀范围 +30%", "stat_key": "whirlwind_radius_percent", "value": 0.30 },
        { "pieces": 4, "summary": "命中获得风势，每层攻速 +5%，最多 10 层", "stat_key": "attack_speed_percent", "value": 0.05 },
        { "pieces": 6, "summary": "御风刀伤害 +400%", "stat_key": "set_bonus_percent", "value": 4.00 }
      ]
    }
  ]
}
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 传承 ID |
| `name` | string | 名称 |
| `theme` | string | 主题描述 |
| `piece_slots` | array[string] | 允许生成的槽位 |
| `bonuses` | array | 套装效果数组 |

`bonuses[]` 元素字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `pieces` | int | 需要激活的件数 |
| `summary` | string | HUD/面板展示文案 |
| `stat_key` | string | 实际叠加到 combat bonuses 的键 |
| `value` | float | 数值 |

说明：

- 当前仓库使用 `bonuses[].pieces`，不是旧文档里的 `required_pieces`。
- 当前仓库使用 `piece_slots`，不是旧文档里“每件单独列 pieces[]”的结构。

## 5. 百炼坊配方 `data/equipment/cube_recipes.json`

当前真实顶层键为 `cube_recipes`：

```json
{
  "cube_recipes": [
    {
      "id": "extract",
      "name": "萃取武学",
      "input_rarity_min": "epic",
      "costs": [
        { "resource_id": "scrap", "amount": 50 },
        { "resource_id": "legend_shard", "amount": 10 }
      ],
      "result": "unlock_legendary_effect"
    }
  ]
}
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 配方 ID |
| `name` | string | 名称 |
| `input_rarity` | string | 可选，要求精确品质 |
| `input_rarity_min` | string | 可选，要求最低品质 |
| `costs` | array | 消耗列表 |
| `result` | string | 结果类型标记 |

`costs[]` 元素字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `resource_id` | string | 资源 ID，当前常见为 `gold/scrap/core/legend_shard` |
| `amount` | int | 消耗数量 |

当前已使用的配方 ID：

- `extract`
- `upgrade_rare`
- `reforge`
- `convert_set`
- `refine_affix`

说明：

- 当前仓库使用 `costs[]`，不是旧文档中的 `cost` 对象。
- 当前仓库使用 `result` 字段描述目标行为。

## 9. 当前实现约定

- `ConfigDB` 读取 `equipment_bases.json / affixes.json / legendary_affixes.json` 时，顶层直接按数组读取。
- `ConfigDB` 读取 `set_defs.json / cube_recipes.json / gems.json / rift_scaling.json / rift_keys.json` 时，读取它们各自的顶层包裹键。
- `SetSystem`、`CubeSystem`、`MartialCodexSystem`、`RiftSystem` 等后续逻辑，都以这些真实字段为准。
- 如果后续需要扩表，先改 `ConfigDB` 的加载方式，再更新本文件，不要继续依赖旧原型期字段名。
