#### 词缀汇总与DPS计算

```
兵器基础伤害: 85

A 主属性: atk_pct 22% + flat_atk 65+40+55 = 160 (折算约+19%)
  → A = 1 + 0.22 + 0.19 = 1.41

B 武技增幅: whirlwind_dmg 45%+35% = 80%, skill_dmg_pct 20%
  → B = 1 + 0.80 + 0.20 = 2.00

C 暴击: crit_rate 6%+4% = 10%, crit_dmg 45%
  → C = 1 + 0.10 × 0.45 = 1.045

D 速度: atk_speed 12%+8% = 20%, cdr 8%
  → D = 1 + 0.20 + 0.08 = 1.28

E 套装: 无
  → E = 1.0

F 传奇: 无
  → F = 1.0

G 增伤: elite_dmg 15%
  → G = 1.15 (对精英) / 1.0 (对普通)

最终DPS(对精英) = 85 × 1.41 × 2.00 × 1.045 × 1.28 × 1.0 × 1.0 × 1.15
                = 85 × 4.385
                ≈ 372.7

最终DPS(对普通) = 85 × 1.41 × 2.00 × 1.045 × 1.28 × 1.0 × 1.0 × 1.0
                = 85 × 3.813
                ≈ 324.1
```

## 二、伤害公式验证用例（6个）

### 用例1: 纯凡品，零词缀加成 → 验证基础伤害

```
输入:
  兵器基础伤害 = 15
  9件全部凡品，且词缀均为flat_hp（不影响伤害乘区）

计算:
  A = 1.0, B = 1.0, C = 1.0, D = 1.0, E = 1.0, F = 1.0, G = 1.0

期望输出: DPS = 15 × 1.0^7 = 15.0
验证点: 确认基础伤害不被词缀以外的因素放大
```

### 用例2: 单乘区堆叠（只堆主属性）

```
输入:
  兵器基础伤害 = 50
  所有装备词缀全部为 atk_pct，总计 +120%

计算:
  A = 1 + 1.20 = 2.20
  B~G = 1.0

期望输出: DPS = 50 × 2.20 = 110.0
验证点: 确认单乘区下纯加法叠加
```

### 用例3: 双乘区均衡（主属性+暴击）

```
输入:
  兵器基础伤害 = 50
  主属性词缀合计: atk_pct +60%
  暴击词缀: crit_rate 15%, crit_dmg 80%

计算:
  A = 1 + 0.60 = 1.60
  C = 1 + 0.15 × 0.80 = 1.12
  B,D,E,F,G = 1.0

期望输出: DPS = 50 × 1.60 × 1.12 = 50 × 1.792 = 89.6
对比单乘区: 同样总属性+120%放进A → 50 × 2.20 = 110.0
但若拆成 A:60% + C:60%(等效) → 双乘区更优

验证点: 确认多乘区相乘大于单乘区堆叠
```

### 用例4: 全乘区均衡（7乘区各有值）

```
输入:
  兵器基础伤害 = 100
  A = 1 + 0.30 = 1.30
  B = 1 + 0.25 = 1.25
  C = 1 + 0.05×0.60 = 1.03  (5%暴击, 60%暴伤)
  D = 1 + 0.10 = 1.10
  E = 1 + 0.00 = 1.00 (无套装)
  F = 1 + 0.15 = 1.15
  G = 1 + 0.20 = 1.20

期望输出: DPS = 100 × 1.30 × 1.25 × 1.03 × 1.10 × 1.00 × 1.15 × 1.20
             = 100 × 2.463
             = 246.3

验证点: 确认7个乘区全部参与乘法计算
```

### 用例5: 传承套装触发（巨额乘区E）

```
输入:
  兵器基础伤害 = 180
  A = 1.50
  B = 2.00
  C = 1.15
  D = 1.20
  E = 1 + 4.00 = 5.00 (御风传承6件: +400%)

对比无套装: 180 × 1.50 × 2.00 × 1.15 × 1.20 × 1.00 = 180 × 4.14 = 745.2
套装倍率: 3726 / 745.2 = 5.0x（正确，因为 E=5.0）

验证点: 确认套装乘区E的巨额加成正确参与乘法
```

## 四、掉落节奏验证场景（3个）

### 场景1: 普通节点10分钟

```
环境: 第一章ch1_n2节点，连续刷10分钟

假设:
  - 每波约8秒，含过渡约10秒/波
  - 10分钟 = 600秒 ÷ 10秒 = 约60波
  - 每波击杀3-5只怪，每只掉落概率30%
  - 每波预期掉落约1.2件

预期掉落分布 (72件):
  凡品: 72 × 50% = 36件  → 分解为锻材
  精良: 72 × 25% = 18件  → 大部分分解，偶有升级
  玄品: 72 × 15% = 10.8 ≈ 11件  → 需要逐个检查词缀
  真意: 72 × 7% = 5件  → 重要掉落，可能替换装备
  传承: 72 × 2% = 1.4 ≈ 1件  → 稀有，可能改变Build
  上古真意: 72 × 0.8% = 0.6 ≈ 0-1件  → 惊喜
  绝世真意: 72 × 0.2% = 0.14 ≈ 0件  → 极稀有

验证点:
  ✅ 10分钟内能获得足够多的凡品/精良用于分解
  ✅ 玄品掉落率保证每次游玩都有比较装备的机会
  ✅ 真意掉落稀有但不至于绝望

## 五、传承（套装）凑件测试用例（2个）

### 用例1: 御风传承 2/4/6件逐步触发

```
初始状态: 9件普通装备（无套装），DPS = 500

步骤1: 穿上2件御风传承（兵器+护手）
  预期: 触发2件效果"御风刀范围扩大30%"
  DPS变化: 无直接DPS变化（范围增大是功能性提升）
  验证: SetSystem.get_active_bonuses("set_wind") 返回 [2件效果]

步骤2: 再穿上2件（护甲+护额），共4件
  预期: 额外触发4件效果"风势层，每层+5%攻速，最多10层"
  DPS变化: 假设平均维持6层 → 攻速+30% → D乘区提升
  验证: SetSystem.get_active_bonuses("set_wind") 返回 [2件效果, 4件效果]

步骤3: 再穿上2件（腿甲+轻靴），共6件
  预期: 额外触发6件效果"御风刀伤害+400%"
  DPS变化: E乘区 = 1 + 4.0 = 5.0 → DPS ×5
  从500 → 约2500（仅E乘区变化）
  验证:
    ✅ SetSystem.get_active_bonuses("set_wind") 返回 [2件, 4件, 6件效果]
    ✅ DamageResolver 的 E 乘区 = 5.0
    ✅ 取下任一件 → 降回4件效果，E乘区回到1.0

步骤4: 取下1件（腿甲换为普通）
  预期: 降回5件，只触发2件+4件效果，6件效果消失
  DPS变化: E乘区回到1.0 → DPS约500
  验证: SetSystem.get_active_bonuses("set_wind") 返回 [2件, 4件效果]
```

    { "set_wind": [2件, 4件, 6件], "set_blood": [2件] }

  DamageResolver计算:
    E乘区 = (1 + 4.0) × (1 + 流血持续效果折算)
    = 5.0 × 1.XX

  ✅ 华戒的2件效果与正常穿2件传承件的效果完全一致
  ✅ 华戒不计入御风传承的件数
```
