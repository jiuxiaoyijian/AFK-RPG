# 项目进展记录 v1.9

## 1. 本轮定位

本轮承接 `v1.8` 的掉落统计面板，继续把这套系统从“能看样本”推进到“能判断偏差”：

- 加入理论值 vs 实测值对比
- 增加偏差状态提示
- 继续维护版本化项目进展记录

## 2. 本轮目标

目标：

- 让玩家能直接看出当前节点掉落是否高于期望
- 让追踪目标也能看到理论命中率与实际命中率的偏差
- 把“欧 / 非 / 接近期望”做成更直观的文本结论

## 3. 本轮完成内容

### 3.1 统计面板新增节点整体偏差对比

本轮后，掉落统计面板中的节点详情会新增一行 `偏差对比`，用于比较：

- 理论装备掉落效率
- 实测装备掉落效率
- 理论传奇特效效率
- 实测传奇特效效率

每一项都会显示：

- 理论值
- 实测值
- 百分比偏差
- 状态标签

状态标签目前包括：

- `明显高于期望`
- `略高于期望`
- `接近期望`
- `略低于期望`
- `明显低于期望`

这让玩家不需要自己心算，也能快速判断当前节点刷得是偏欧还是偏非。

### 3.2 追踪目标新增理论/实测对比

本轮不只比较节点整体掉装表现，还会额外比较当前 `追踪传奇目标` 在所选节点上的情况。

如果该节点可以掉落当前追踪目标，就会新增一行：

- `追踪目标对比`

该对比会显示：

- 该目标在本节点的理论命中率
- 该目标在本节点的实测命中率
- 当前偏差百分比
- 当前状态标签
- 若已有命中，则补充“实测约多少次见 1 次”
- 若已有样本但仍未命中，则明确提示“当前样本多少次仍未见到目标”

这样玩家不只是知道“这个节点适不适合刷”，还知道“自己现在到底黑到什么程度”。

### 3.3 对比逻辑兼容无样本和无理论值场景

为了避免误导，本轮对比逻辑也处理了几类边界情况：

- 没有样本时，显示暂无样本
- 当前节点不掉该追踪目标时，明确提示不产出
- 无法形成理论值时，显示暂无法估算
- 理论值为 0 但实测有值时，给出超出理论池提示

这让统计文本在异常数据和低样本阶段也更稳。

## 4. 本轮关键实现点

逻辑侧重点：

- `LootCodexSystem._get_node_rate_comparison_text()`：节点整体掉率偏差

- `LootCodexSystem._get_deviation_state_label()`：把偏差映射成状态标签

## 7. 当前项目状态

截至 `v1.9`，掉落观察系统已经具备：

- 图鉴中的期望值参考
- 图鉴中的实战统计文本
- 独立掉落统计面板
- 最近掉落样本记录
- 理论值 vs 实测值偏差提示
- 追踪目标的单独偏差判断

这意味着玩家现在已经能回答：

- 当前节点整体掉装是不是偏高
- 当前节点传奇效率是不是偏低
- 我的追踪目标在这个节点是不是比理论更难出
- 我现在是短期脸黑，还是样本上已经明显低于期望

## 8. 当前已知问题

- 当前偏差仍然是文本结论，缺少更直观的颜色/条形图呈现
- 偏差判断目前主要围绕当前追踪传奇目标，还没有扩展到底材/词条目标
- 低样本阶段虽然有提示，但还没有“样本不足”强提示阈值

- 或支持按 `底材 / 词条 / 传奇` 目标切换统计视角
- 或继续拆 `GameManager`，把节点推进和背包装备管理分离
