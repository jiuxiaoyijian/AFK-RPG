# 项目进展记录 v3.5

## 1. 本轮定位

本轮承接 `v3.4` 已完成的 `地面掉落 -> 主动/被动拾取 -> 飞向 UI` 闭环，继续把战斗和掉落演出往“更像正式刷宝游戏”的方向推进。

这一轮用户提出了 5 个明确要求：

- 掉落抛物线必须高过主角和怪物头顶
- 拾取飞向 UI 的粒子效果要更明显
- 掉落物层级必须高于角色和怪物
- 主角站位改为舞台中央，怪物从左右两侧刷入
- 增加便于测试的本地手动存档功能

因此本轮的核心目标不是加新系统，而是把：

- `战斗舞台感`
- `掉落观赏性`
- `测试便利性`

一起推进一档。

## 2. 本轮目标

目标：

- 让主角不再像“从左推到右”，而是站在舞台中心承接四面来敌
- 让掉落抛物线真正跨过角色头顶，形成更明显的空中轨迹
- 让掉落与拾取特效在视觉上压过角色本体，不被遮挡
- 让测试时可以随时存档 / 回档

## 3. 本轮完成内容

### 3.1 主角出生点和复位点改到场景中央

本轮把主角的：

- 初始出生点
- 归位点

统一收束到场景中央。

同时重新定义了主角的活动舞台范围：

- 左边界约为场景 `1/3`
- 右边界约为场景 `2/3`

这意味着主角不会再一路贴着左边往右清场，而是始终站在一个更接近“主舞台中心”的区域里，敌人和掉落都围绕他发生。

这种变化带来的直接观感是：

- 主角更像画面中心的英雄
- 掉落和击杀反馈更容易被聚焦到主角周围
- 战斗镜头的视觉重心明显更稳

### 3.2 怪物改为从左右两侧随机刷出

本轮调整了刷怪逻辑：

- 怪物不再固定在主角右侧生成
- 而是会从左边或右边随机刷出
- 然后向主角所在的中央区域移动

这样战斗从原来的“单向推进”切换成了更像竞技舞台的“向中间汇聚”。

对体验的好处是：

- 主角不再只是赶路
- 左右来敌让战斗更有被围攻与破阵感
- 中央区域的掉落也更集中，方便后续做刷宝观赏性

### 3.3 掉落抛物线高度大幅抬高

为了满足“抛物线至少高过主角和怪物头顶”的要求，本轮显著提高了地面掉落的弧线高度：

- 普通资源掉落：弧线明显抬高
- 装备掉落：弧线更高
- 高品质装备 / 光柱掉落：抛物线最高

现在掉落物会明显从角色头顶上方掠过，再回落到地面附近，而不是只做一个低矮的小跳。

这让掉落演出的阅读顺序更清楚：

- 先看到怪物爆开
- 再看到物件飞起
- 最后看到它们落地等待拾取

### 3.4 掉落层级提升，确保始终压过角色和怪物

本轮专门提高了掉落物相关节点的层级：

- `LootContainer` 层级抬高
- 每个掉落视觉节点也携带自己的 `z_index`

这样可以保证：

- 掉落图标
- 品质光柱
- 拾取收束粒子

都优先显示在：

- 主角
- 普通怪
- 精英
- Boss

之上，不会因为角色模型站位而被遮挡。

### 3.5 拾取飞向 UI 的粒子效果放大

上一轮的拾取效果虽然已经有：

- 粒子聚合
- 飞向 UI

但视觉存在感还不够强。

因此本轮把拾取粒子做了增强：

- 粒子尺寸明显放大
- 粒子初始扩散半径更大
- 收束速度略放慢
- 整体飞 UI 的时间略加长

这样在拾取时，玩家能更明确看到“地上的东西被抽离成一团品质色能量，然后飞回资源栏或背包入口”的过程。

### 3.6 本地测试存档升级为可手动存 / 手动读

项目原本已经有：

- 退出自动存档
- 进入自动读档

但不方便频繁测试某个节点、某一件装备、某次掉落。

因此本轮补上了手动测试入口：

- `F5`：手动快速存档
- `F8`：手动快速读档

并在：

- 主 HUD 提示
- 底部主导航提示

里都补上了热键说明。

这样现在做测试时可以：

- 先在关键节点手动存档
- 再尝试某次掉落或某个 Build
- 不满意就直接读档重测

对迭代速度帮助很大。

## 7. 当前项目状态

截至 `v3.5`，项目的战斗与掉落体验已经从：

- 单向推进
- 低矮掉落
- 掉落可能被角色遮挡
- 只能退出自动存档

推进到：

- 主角站在舞台中心承接两侧来敌
- 掉落抛物线高过角色头顶
- 掉落与光柱层级稳定压过角色
- 拾取收束粒子更明显
- 支持 `F5 / F8` 手动存读档测试

## 8. 当前已知问题

- 怪物虽然已从左右两边刷入，但还没有更强的左右进场专属动画
- 掉落飞 UI 已经更明显，但还缺少音效配合
- 当前仍没有传奇掉落时的屏幕级闪光和震屏
- 快速存读档目前仍是单存档位

## 9. 推荐下一步

按当前节奏，下一轮建议优先做：

- 传奇掉落屏幕边缘闪光 / 镜头轻震
- 金币 / 装备 / 传奇三类拾取音效
- 多存档位或“测试快照”功能
