# 项目进展记录 v3.1

## 1. 本轮定位

本轮承接 `v3.0` 的整体 UI 重构，继续把统计面板从“文本可读”推进到“视觉可扫”：

- 把理论/实测偏差做成进度条
- 压缩导航和统计摘要文案
- 继续维护版本化项目进展记录

## 2. 本轮目标

目标：

- 让玩家在统计面板里不用逐字阅读，也能一眼判断节点效率
- 让追踪目标的欧非状态更直观
- 进一步降低信息区块里的长文本负担

## 3. 本轮完成内容

### 3.1 统计面板新增三组可视化效率条

本轮在 `掉落统计面板` 中新增了三组可视化对比：

- `装备效率`
- `传奇效率`
- `追踪目标`

每组都会显示：

- 理论值
- 实测值
- 偏差百分比
- 状态标签
- 对应进度条

进度条以“理论值 = 100”作为基准，实测值越高，条越长；越低则越短。

这样玩家不需要逐字读完整句，也能先通过长度和颜色快速判断当前节点表现。

### 3.2 偏差颜色层级继续强化

本轮把统计面板中的偏差结果进一步做了颜色分层：

- 明显高于期望：绿色
- 略高于期望：浅绿色/黄绿色
- 接近期望：冷色中性
- 略低于期望：橙色
- 明显低于期望：红色

这让 `偏欧 / 正常 / 偏非` 的判断从纯文字变成了文字 + 颜色双通道表达。

### 3.3 追踪目标的特殊状态更明确

对于当前追踪目标，本轮不仅保留了理论/实测对比，还补充了更适合玩家理解的状态文本：

- 暂无实测样本
- 当前样本若干次仍未见到目标
- 实测约多少次见 1 次
- 当前节点不产出该目标

这样即使追踪目标还没有实际掉落，面板也不会只剩下一串抽象数字。

### 3.4 统计面板摘要改为分行概览

此前统计概览更像一行长句。

本轮改成了分行摘要，按更容易扫读的方式展示：

- 总样本次数
- 装备样本数
- 传奇特效次数
- 已记录节点数
- 当前追踪目标摘要

这样顶部摘要更像一个“概览卡”，而不是一整句拼接文本。

### 3.5 导航栏“统计”按钮摘要压缩

本轮同步优化了底部主导航栏中 `统计` 按钮的摘要文案。

原本该按钮会塞入完整统计概览，长度偏长。

现在压缩成更轻量的摘要形式：

- 已记录节点数
- 最近样本条数

这让底部导航栏的信息密度更合理，不会抢主界面焦点。

## 4. 本轮关键实现点

逻辑侧重点：

- `LootCodexSystem.get_drop_stat_visual_data()`：提供统计面板的结构化视觉数据

- `drop_stats_panel_controller.gd`：将结构化数据映射到进度条和颜色标签

## 7. 当前项目状态

截至 `v3.1`，统计系统已经从：

- 纯文本偏差说明

升级为：

- 文字 + 颜色 + 进度条 三层表达

现在玩家在统计面板里已经能更直观地回答：

- 这个节点整体掉装是不是偏高
- 传奇效率是不是低于预期
- 当前追踪目标到底是正常波动，还是明显脸黑

## 8. 当前已知问题

- 当前进度条仍然是单色调，没有更细的段位背景或刻度线
- 资源条、装备卡和功能按钮还没有真正的图标资产
- 统计面板的近期样本仍以文本为主，尚未做更明确的时间线/卡片化记录
