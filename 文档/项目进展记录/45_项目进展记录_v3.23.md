# 项目进展记录 v3.23

## 1. 本轮定位

本轮先不继续推进游戏内功能，而是处理两个更基础但会直接影响后续协作效率的前置问题：

- 让本地工程内容与 GitHub 最新仓库重新对齐
- 读取新增的 Figma 交互设计文档，产出一版新的网页 UI 原型稿

这样做的目的很明确：

- 先把“本地看到的工程”和“仓库里真正最新的工程”统一起来
- 再基于最新文档语义和布局要求去做新的展示稿
- 避免在旧目录、旧原型或缺文档的前提下继续往后推进

## 2. 本轮目标

目标：

- 将本地 `AFK-RPG` 工程内容与 GitHub `main` 最新树重新对齐
- 补齐缺失的两份 Figma 文档
- 清理不应留在工程仓库里的本地设计/生成目录
- 重新生成一版符合 `反桃花源修真` 方向的网页 UI 稿

### 3.2 本地导入缓存已重新生成，运行 warning 已收敛

在仓库内容补齐后，本轮还处理了正式资源的本地导入缓存问题。

此前 `afk_rpg_formal` 资源在无界面启动时会出现一组 `.ctex` 缓存缺失 warning。  
本轮通过 Godot 的导入流程重新生成了本地导入缓存后，这批 warning 已经消失。

当前再次执行无界面启动时：

- Godot 退出码为 `0`
- 不再出现 `afk_rpg_formal` 资源缺失 warning
- 当前只剩 macOS 系统证书读取 warning，不影响项目运行

### 3.3 已基于 Figma 文档重做网页 UI 原型稿

在本地工程和文档恢复一致后，本轮开始处理新的网页展示稿。

本次重新阅读了：

- `文档/Figma_交互设计简介.md`
- `文档/Figma_分阶段提示词.md`

并据此重写了：

- `文档/prototype/index.html`

新的网页原型不再沿用旧的通用面板排布，而是直接对齐当前交互文档里定义的结构：

- `1280×720` 横屏战斗主界面
- 顶栏 `TopBar`
- 左侧 `BattleCard`
- 中央战斗区与底部 `BattleSafeFrame`
- 右侧 `TargetCard + DailyGoalCard`
- 底部 `MainNavBar`
- 模态面板：背包 / 悟道 / 异闻录 / 机缘推演 / GM
- 弹窗模式：闭关所得 / 破局要闻

视觉方向也改成了当前项目已经确认的：

- `反桃花源修真`
- “越好看越不对劲”
- 雾青、枯桃粉、旧白、香火金、血朱的整体氛围

### 3.4 新原型已经补入基础交互层

这版网页稿不是静态截图式排版，而是补了最基础的页面交互：

- 可打开与关闭 5 个底部主面板
- 支持 `I / U / O / P / G / Esc` 键盘切换
- 可切换 `闭关所得 / 破局要闻` 两种弹窗状态
- 面板内补了筛选与状态展示

这让它更接近“网页交互稿”，而不只是一个单页视觉草图。

- 视觉上已经切到 `反桃花源修真`，但仍可继续加强“表面温润、内里异化”的反差
- 目前还未把这版网页稿反向映射成游戏内真实 UI 资源需求清单
- 尚未做手机端或窄屏适配，本轮目标仍以 `1280×720` 主画板为主
