# 项目进展记录 v4.3

日期：2026-03-25

---

## 本轮目标

在不改动快捷键、面板 id 与存档版本的前提下，把此前临时态的 `U / P` 正式升级为产品化入口：

1. `U` 从单页“悟道”升级为正式的 `成长中心`
2. `P` 从临时复用面板升级为正式的 `推演与秘境`
3. 补齐 `Paragon / Season / Rift` 的 UI 摘要接口
4. 继续保持现有 `P1 / P2` 运行链和存档结构兼容

---

## 已完成

### 1. `U` 已正式升级为“成长中心”

本轮继续复用现有 `research` 面板 id 与 `ui_research` 输入动作，但界面结构已经正式重组。

当前 `成长中心` 固定为三页：

- `武学参悟`
- `宗师修为`
- `重入江湖`

其中：

- `武学参悟` 继续承接原研究树、筛选、升级逻辑
- `宗师修为` 当前接入 `ParagonSystem.build_runtime_summary()`，以占位摘要展示“未解锁 / 后续阶段开放”
- `重入江湖` 当前接入 `SeasonSystem.build_runtime_summary()`，以占位摘要展示后续阶段入口

这意味着：

- `U` 键和导航按钮都已经打开正式的 `成长中心`
- 研究树玩法没有被回归破坏
- `Paragon / Season` 已开始拥有正式入口，而不再只是埋在默认存档状态里

### 2. `P` 已正式升级为“推演与秘境”

本轮继续保留 `drop_stats` 面板 id 与 `ui_drop_stats` 输入动作，但把内容组织从单页临时态改成了多页正式结构。

当前 `推演与秘境` 固定为三页：

- `机缘推演`
- `试剑秘境`
- `秘境结算`

当前页面职责如下：

- `机缘推演`
  - 继续承接当前的节点掉率分析、推荐、效率信息
- `试剑秘境`
  - 汇总当前可选层数、最高层、钥石库存、秘境进行中状态、两枚宝石与开启入口
- `秘境结算`
  - 展示 `rift_state.recent_results` 中最近的通关 / 失败记录与奖励摘要

这意味着当前的 `P` 已经不再只是“在旧面板上临时多塞几个按钮”，而是正式具备了分析、冲层、回看结算三段式结构。

### 3. 运行时摘要接口已补齐

为了避免 UI 控制器继续堆积运行时拼装逻辑，本轮补了三类轻量摘要接口：

- `ParagonSystem.build_runtime_summary(state)`
- `SeasonSystem.build_runtime_summary(state)`
- `RiftSystem.build_runtime_summary(state)`

其中 `RiftSystem` 本轮额外补齐了：

- `recommended_level`
- `can_start`
- `blocked_reason`

同时，`GameManager` 新增：

- `get_progression_hub_summary()`
- `get_analysis_hub_summary()`

用于给导航按钮和面板顶部摘要统一供数。

### 4. 秘境结算信息已开始持久化回运行态

本轮对 `GameManager.finish_rift_run()` 做了补强：

- 最近一次秘境奖励摘要会回写到 `rift_state.recent_results`
- 奖励文本会进入结算页可读结构

这样 `秘境结算` 页就不再只是显示“通关/失败”状态，而能够读到更完整的奖励反馈。

### 5. 导航与术语已统一

本轮已把产品入口统一到冻结方案：

- `U`：`成长中心`
- `P`：`推演与秘境`

已同步位置包括：

- `MainNavBar`
- 面板标题
- 按钮摘要
- 快捷键提示文案

本轮仍保留：

- `O` 只对应 `江湖见闻录`
- `武学秘录` 只保留在 `B 百炼坊` 内部页签

## 当前边界

虽然 `U / P` 已正式化，但当前仍有明确边界：

- `Paragon / Season` 目前仍只是正式入口与占位摘要，不含真实玩法
- `ResearchPanel / DropStatsPanel` 的内部文件路径仍保留旧名，当前未做重命名清理
- 宝石仍然是轻量运行态，不是独立物品背包系统
- 尚未完成完整人工回归，尤其是三页切换、秘境结算展示与宝石切换的逐项点击验证
