## 已完成

### 1. 宗师修为已进入真实运行态

`ParagonSystem` 本轮已从脚手架扩展为真实状态机，当前状态包含：

- `is_unlocked`
- `level`
- `experience`
- `available_points`
- `allocated`

当前版本的解锁约定基于仓库真实代码，而不是旧文档里的“角色等级”：

- 击破 `第二章首领 ch2_boss` 后正式开启宗师修为
- 开启后，普通推图与试剑秘境都会开始产出宗师修为

当前已接入的可分配项共有七类：

- 攻击百分比
- 生命值
- 防御
- 暴击率
- 暴击伤害
- 香火钱获取
- 宗师修为获取

`成长中心 -> 宗师修为` 页已支持：

- 查看当前修为等级、进度、可用点数
- 选择单项修为并投入 1 点
- 即时把加成同步到真实战斗面板

### 2. 宗师加成已进入真实 Build 链

本轮 `BuildQueryService.get_total_combat_bonuses()` 已开始继续叠加：

- 传承
- 武学秘录
- 宝石
- 宗师修为
- 江湖阅历

也就是说，当前宗师分点已经不只是 UI 文字，而会真实影响：

- 攻击
- 生命
- 防御
- 暴击率
- 暴伤

同时，`MetaProgressionSystem.get_meta_progression_bonuses()` 也开始叠加：

- 宗师修为的香火钱加成
- 宗师修为的修为获取加成
- 江湖阅历的香火钱加成
- 江湖阅历的修为获取加成
- 江湖阅历的掉落品质加成

### 3. 重入江湖已进入真实轮回链

`SeasonSystem` 本轮已从占位态扩展为真实轮回状态机。

当前触发条件：

- 击破第二章首领
- 试剑秘境最高层达到 `Lv.20`

`成长中心 -> 重入江湖` 页已支持二次确认式执行：

- 第一次点击进入确认态
- 第二次点击才真正执行轮回

这样可以避免误触直接清空当前轮进度。

### 4. 当前重入江湖保留与重置规则已落地

本轮按当前仓库最小可玩闭环实现了以下轮回规则。

保留：

- 武学秘录
- 宗师修为
- 江湖见闻录
- 江湖阅历永久加成

重置：

- 装备与背包
- 香火钱 / 祠灰 / 灵核 / 真意残片
- 武学参悟
- 当前章节与节点推进
- 秘境层数与进行中状态
- 宝石拥有与镶嵌状态
- 当前击杀 / 清图计数

当前轮回后会重新回到：

- `chapter_1`
- 第一节点
- 默认自动分解阈值

### 5. 江湖阅历永久加成已进入真实链路

本轮已把 `江湖阅历` 做成真实永久被动，并开始影响多条系统链。

当前已接入的长期收益包括：

- 全属性伤害增益
- 宗师修为获取增益
- 香火钱获取增益
- 掉落品质提升

其中掉落品质提升已开始影响装备生成：

- 当前实现为“低稀有度结果有概率向上提升 1 档”

这让二周目至少在掉落手感和成长速度上，已经和首轮有了可感知差异。

### 6. 成长中心已从“占位”变成可操作页

本轮 `ResearchPanel` 控制器继续沿用原路径，但内容逻辑已更新：

- `武学参悟`：保持原玩法
- `宗师修为`：真实列表 + 详情 + 分点按钮
- `重入江湖`：真实条件 + 保留/重置预览 + 二次确认按钮

同时还补了和运行时进度有关的刷新事件：

- 节点推进
- 试剑秘境开始/结束
- 宗师修为变化
- 重入江湖完成

这样在 `成长中心` 打开状态下，条件变化会立即刷新，而不会停留在旧摘要上。

## 当前边界

这轮虽然已经把 `P3` 接成最小可玩闭环，但仍有明确边界：

- 当前没有独立“角色等级”系统，因此宗师修为解锁采用了“第二章毕业后转长期成长”的仓库兼容方案
- 宗师修为当前只做了单点分配，没有“重置分点”能力
- 江湖阅历当前主要影响伤害、香火钱、修为和掉落品质，还没有补齐更多长期奖励维度
- 轮回后没有单独的“新周目开场演出/引导”
- 尚未完成完整人工回归，尤其是轮回后存档读取、再开秘境、再刷百炼坊的连续流程验证
