# 项目进展记录 v3.25

## 1. 本轮定位

本轮开始把前两轮已经完成的网页稿真正回灌到项目内 UI。

目标不再是继续出网页，而是把：

- `index.html` 的交互结构
- `review.html` 的评审基线

转成 Godot 项目里实际可运行的 HUD、导航栏、模态面板和结果卡样式。

换句话说，本轮的重点是：

- 从“网页稿已经统一”
- 推进到“游戏内相关 UI 也开始对齐这套视觉语言”

## 2. 本轮目标

目标：

- 将项目内相关 UI 调整到更接近交互稿的布局与气质
- 统一 HUD、底部导航栏、主面板、结果卡的视觉规则
- 让项目在完成这轮 UI 改造后仍能正常启动运行

## 3. 本轮完成内容

### 3.1 已新增统一 UI 样式工具

本轮新增：

- `scripts/ui/ui_style.gd`

这份脚本的作用，是把当前项目里原本分散在各个控制器里的 UI 颜色、面板描边、按钮皮肤、列表样式、进度条样式统一收口。

它现在主要提供了几类基础能力：

- 面板样式
- 按钮样式
- 下拉框 / 复选框样式
- 列表样式
- 富文本样式
- 进度条样式
- 标签语义色

这一步很重要，因为如果没有统一样式层，后续每个面板都会继续“局部修对了，但全局还是散的”。

### 3.2 HUD 已开始向交互稿结构靠拢

本轮对 `HUD` 做了首轮回灌，重点集中在：

- 顶栏 `TopBar`
- 左侧 `BattleCard`
- 右侧 `TargetCard + DailyGoalCard`
- 底部 `BattleSafeFrame`
- 高价值掉落 `DropToast`
- 战斗高光横幅

其中最关键的变化是：

- `TargetCard` 不再保持隐藏
- `BattleSafeFrame` 不再保持隐藏
- 底部安全框重新纳入 HUD 布局，装备概览和最近掉落开始回到统一底部区域
- 各卡片开始统一使用同一套深色祠堂式边框和标题色

这意味着游戏内首页终于开始接近网页稿里那套：

- 中间看战斗
- 右侧看追踪与今日机缘
- 底部看装备与掉落

### 3.3 底部导航栏已改成更接近评审稿的等宽结构

本轮调整了：

- `scenes/ui/main_nav_bar.tscn`
- `scripts/ui/main_nav_bar_controller.gd`

改动重点包括：

- 5 个导航按钮改为更接近交互稿的等宽分布
- 导航栏整体背景改成更贴近网页稿的深底分隔条
- 各按钮不再沿用旧版纯文字按钮观感，而是开始统一到同一套按钮皮肤

这让底部导航栏从“工程功能按钮”更接近“正式 HUD 导航条”。

### 3.4 五个主面板和结果卡已统一到新主题

本轮把下列 UI 入口接到了统一样式规则里：

- `背包`
- `悟道`
- `异闻录`
- `机缘推演`
- `GM`
- `闭关所得 / 破局要闻`

这些面板本轮虽然没有大幅改结构，但已经完成了几个关键层面的统一：

- 主标题统一
- 按钮皮肤统一
- 面板边框与背景统一
- 列表、富文本和进度条开始统一
- 结果卡的标题、正文与关闭按钮进入同一套风格

其中还顺手做了几处文案收敛：

- `背包与装备管理` → `背包`
- `悟道与长期成长` → `悟道`
- `异闻录与机缘追踪` → `异闻录`
- `GM 调试面板` → `GM 调试`

### 3.5 项目已调试到可运行

本轮完成代码改造后，已经做了两层验证：

1. Godot 无界面启动验证
- 启动通过
- 退出码 `0`

2. Godot MCP 调试启动
- 项目已成功启动到调试模式
- 调试输出中没有新的解析错误或阻塞型运行错误
- 当前仍只剩 `EventBus` 里 signal 的未显式使用 warning

这意味着这轮 UI 回灌虽然改动面较广，但当前项目仍处于可运行状态。

## 8. 当前已知问题

- 当前 UI 改造仍以“统一气质与层级”为主，和网页稿相比，几何细节还没有完全逐像素对齐
- `BattleSafeFrame` 虽已回到布局中，但其内部内容还可以继续往更强的“安全框”结构收拢
- 目前仍未做人手实机过一遍全部面板

## 9. 推荐下一步

下一轮建议优先从下面两个方向里选一个：

- 继续细化 HUD 和 5 个主面板的几何布局，使其更贴近 `review.html`

如果继续沿当前方向往下做，最值得补的是：

- `BattleSafeFrame` 内部真正的二分栏布局
- 面板标题栏与筛选标签的更精细对齐
- 结果卡的视觉层级再强化
