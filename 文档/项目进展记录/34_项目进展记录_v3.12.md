# 项目进展记录 v3.12

## 1. 本轮定位

本轮重点从纯游戏内功能，延伸到 `交付链路` 与 `外部可访问性`：

- 继续推进项目工程化
- 为 `Web` 版本导出做准备
- 将项目接入 GitHub 私有仓库
- 尝试部署到 GitHub Pages

这一轮的目标不是继续堆功能，而是把当前原型从“本地 Godot 工程”推进到“可版本化、可导出、可分享”的状态。

## 2. 本轮目标

目标：

- 为项目补齐 Web 导出配置
- 完成本地 Web 导出验证
- 将项目上传到新的私有 GitHub 仓库
- 尝试将 Web 构建部署到 GitHub Pages

## 3. 本轮完成内容

### 3.1 补齐 Web 导出预设

项目原先没有 `export_presets.cfg`，因此虽然可以在编辑器里运行，但还没有稳定的 Web 导出入口。

本轮已完成：

- 新增 `export_presets.cfg`
- 增加 `Web` 导出 preset
- 将导出目标固定到 `build/web/index.html`

这样项目已经具备标准化的 Godot Web 导出入口，而不是只能依赖手工点击编辑器。

### 3.2 本地 Web 导出验证通过

在补齐预设后，本轮直接使用本机 Godot 编辑器进行了实际导出验证。

已确认：

- `Web` preset 可正常执行
- `build/web/index.html` 成功生成
- 同目录下配套的 `index.js` 与音频 worklet 文件也已生成

这说明当前项目代码、资源与导出预设之间没有阻塞性问题，至少在本地层面已经具备 Web 构建能力。

### 3.3 接入 GitHub 私有仓库

本轮已创建新的私有仓库：

- 仓库名：`AFK-RPG`
- 仓库地址：`https://github.com/jiuxiaoyijian/AFK-RPG`

随后已完成：

- 本地初始化 `git`
- 添加远程仓库 `origin`
- 首次提交项目内容
- 将 `main` 分支推送到远程私有仓库

同时，为了避免污染本机全局环境，本轮没有修改全局 `git config`，而是在提交命令中临时使用 GitHub noreply 身份完成提交。

### 3.4 补齐 GitHub Pages 自动部署工作流

为了让后续 Web 构建能够通过仓库自动完成，本轮新增了：

- `.github/workflows/deploy-pages.yml`
- `.gitignore`

工作流内容包括：

- 检出仓库
- 配置 Pages
- 安装 Godot 4.6.1 与导出模板
- 执行 `Web` 导出
- 上传 Pages artifact
- 调用 `deploy-pages`

同时还补了自动启用 Pages 的兜底参数，并将 `checkout` 提升到较新的 action 版本。

## 4. 当前阻塞项

### 4.1 GitHub Pages 未能启用

本轮已完成两次尝试：

- 通过 GitHub API 创建 Pages 站点
- 通过 Actions 的 `configure-pages` 自动启用 Pages

实际结果均失败。

确认到的原因不是项目代码问题，而是当前 GitHub 账户/仓库方案限制：

- 当前私有仓库不支持 GitHub Pages
- GitHub API 返回 `422`
- Actions 中 `Setup Pages` 步骤也在创建站点时失败

因此，当前状态是：

- `Web` 导出已经可用
- GitHub 仓库已经接入成功
- GitHub Pages 部署仍被平台权限限制阻塞
