## 已完成

### 1. 三章节点已补正式名称

本轮为当前三章全部主线节点补了 `name` 字段，不再只保留内部 id。

例如：

- 第一章：`溪桥口 / 渡口栈道 / 芦棚哨岗 / 桃瘴深潭`
- 第二章：`荒祠田埂 / 焚香谷口 / 炼灰外场 / 断碑祭坪 / 祭炉地宫`
- 第三章：`铜铃街口 / 乌篷码头 / 锁街暗巷 / 黑旗货栈 / 银钩赌坊`

这样第三章不再只是“配置能跑”，而是开始具备更正式的主线命名层。

### 2. ConfigDB 已补统一节点显示 helper

本轮在 `ConfigDB` 中新增了统一的节点显示接口：

- `get_node_type_display_name()`
- `get_chapter_node_name()`
- `get_chapter_node_short_label()`
- `get_chapter_node_full_label()`

后续节点显示不必各处自己拼接字符串，而是统一走同一套命名口径。

### 3. 主要 UI 和推荐文案已切到正式节点名

本轮已把以下位置的显示从内部节点 ID 改成正式名称：

- HUD 顶部当前节点
- HUD 运行信息中的“当前节点”
- `推演与秘境` 的当前节点、驻守节点、秘境基底节点提示
- `GM` 面板中的节点列表、跳转记录、重开记录
- `LootRuleService` 的推荐刷图标签
- `BuildQueryService` / `GameManager` 的突破预告与节点短标签
- `DailyGoalSystem` 的刷图目标标签
- `LootCodexSystem` 的节点样本、节点详情、近期掉落记录
- 战斗内精英/Boss 高光中的节点说明

这一步的意义不只是“更好看”，而是让第三章内容真正融入现有产品包装层，而不是继续露出配置内部名。
