# 项目进展记录 v3.9

## 1. 本轮定位

本轮继续沿上一轮的计划推进，重点围绕：

- HUD 进一步减压
- 战斗区域可视化边界
- 飞 UI 粒子层级纠偏
- GM 高频调试能力继续补强

用户本轮明确提出：

- 去掉右上角与底部重复的按键说明
- 给战斗区域画一个透明框，用于后续 UI 动态适配的安全区参考
- 飞向 UI 的粒子效果不应高过二级页面

因此本轮的主要工作，是把“战斗画面优先级”进一步抬高，并继续让调试能力更完整。

## 2. 本轮目标

目标：

- 删除重复信息，减少 HUD 冗余
- 给战斗区一个清晰可见的安全边界
- 确保飞 UI 粒子在视觉层级上服从二级页面
- 继续补 GM 的高频测试入口

## 3. 本轮完成内容

### 3.1 删除右上重复按键说明

本轮已经移除了右上角那块重复的快捷键说明。

现在按键提示只保留在：

- 底部主导航提示条

这样可以避免：

- 右上追踪信息区被挤压
- 同一份说明在上下同时出现
- 画面右上角的信息优先级混乱

### 3.2 新增战斗区域透明安全框

本轮新增了一个透明的 `BattleSafeFrame`，用于标识中部战斗区域的推荐安全范围。

它的作用不是做玩法逻辑，而是提供一个明确的视觉边界：

- 让后续 UI 动态适配时更容易知道“哪里不该放面板”
- 帮助后续继续收 HUD 时有一条直观参考线
- 强化中部战斗区的“舞台感”

### 3.3 装备概览与最近掉落进一步下压

为了配合战斗安全区，本轮把底部两块战斗辅助卡继续往下压了一段：

- 左下 `装备概览`
- 右下 `最近掉落`

这样它们会更贴近导航栏上方，而不再更深地侵入中部战斗视野。

### 3.4 飞 UI 粒子层级降到二级页面之下

此前为了保证飞 UI 粒子不被基础 HUD 遮挡，专门加过顶层特效层。

但这样会带来一个问题：

- 粒子可能压过背包、研究、GM 等二级页面

本轮对此进行了修正：

- 收束特效节点不再挂到独立顶层 `CanvasLayer`
- 而是改挂到主 `UILayer` 里的 `CollectEffects`
- 层级放在 HUD 之上、二级页面之下

这样当前的表现规则变成：

- 平时收束特效依然能正常覆盖基础 HUD
- 打开背包、研究、图鉴、统计、GM 等页面时，二级页面会稳定盖住粒子特效

更符合信息层级预期。

### 3.5 GM 面板新增“清空掉落统计”

继续沿上一轮的计划补高频测试能力，本轮给 GM 面板增加了：

- `清空掉落统计`

这样现在 GM 面板已经可以快速处理：

- 资源拉满 / 清零
- 背包清空
- 掉落统计清空
- 多档存读档

对刷样本、重测掉率、重新观察图鉴与统计表现都更方便。
