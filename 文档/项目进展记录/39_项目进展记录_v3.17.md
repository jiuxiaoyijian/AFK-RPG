# 项目进展记录 v3.17

## 1. 本轮定位

本轮继续按排期表推进，正式落地 `P0-3 阶段事件层` 首版。

目标不是新增更多系统页，而是把“Boss 首杀、章节推进、关键成长、目标达成”变成真正会跳出来的事件，而不是只在状态里悄悄发生。

## 2. 本轮目标

目标：

- 做统一的阶段事件结果卡
- 覆盖 Boss 首杀
- 覆盖新章节解锁
- 覆盖研究关键升级
- 覆盖传奇首次获得
- 覆盖追踪目标达成

## 3. 本轮完成内容

### 3.1 新增 StageEventSystem

本轮新增：

- `scripts/autoload/stage_event_system.gd`

它现在负责：

- 判断阶段事件是否成立
- 对同类事件做去重
- 将阶段事件发给 UI 层
- 将相关状态纳入存档

当前首版已经覆盖：

- Boss 首杀
- 新章节解锁
- 研究关键升级
- 传奇首次获得
- 追踪目标达成

### 3.2 统一结果卡接入阶段事件

本轮没有再造新的弹窗体系，而是复用了现有结果卡。

现在这张结果卡除了离线收益，还能直接展示阶段事件：

- 事件标题
- 高光摘要
- 事件内容
- 接下来建议

这让“阶段事件层”第一次真正前台化。

### 3.3 事件支持连续排队展示

本轮还把 UI 总控层补成了“阶段事件队列”模式。

这意味着如果一次流程里连续发生多个事件，例如：

- Boss 首杀
- 新章节解锁
- 同时获得传奇

这些事件现在不会互相覆盖，而会按顺序展示。

### 3.4 研究与传奇事件源已接线

为了让关键成长事件能准确触发，本轮还接了两条更明确的事件源：

- `research_upgraded`
- `legendary_discovered`

其中：

- 研究事件会在关键节点解锁或满级时触发结果卡
- 传奇事件会在首次发现时触发
- 如果首次发现的传奇正好是当前追踪目标，还会追加“追踪目标达成”事件

### 3.5 阶段事件状态已接入存档

本轮已把阶段事件状态写入存档。

这样现在可以避免：

- 已经首杀过的 Boss 再次被当成首杀
- 已解锁过的章节重复弹出解锁事件
- 已展示过的关键研究事件反复出现

这让阶段事件层进入了正式运行状态，而不是临时效果。
