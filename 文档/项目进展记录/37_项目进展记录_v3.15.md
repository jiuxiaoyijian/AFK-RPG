# 项目进展记录 v3.15

## 1. 本轮定位

本轮正式开始执行排期表中的 `P0-1 回流目标层`，目标不是再补一个系统面板，而是把“玩家回来后该做什么”真正做进主界面和离线结算里。

本轮核心方向：

- 新增每日目标系统
- 让 HUD 上直接出现 `今日主目标 + 支线 + 下一步`
- 让离线收益弹窗从“结算面板”升级成“结算 + 推荐下一步”
- 把这一层状态纳入存档与跨天刷新逻辑

## 2. 本轮目标

目标：

- 做完整的首版每日任务层
- 采用 `1 主目标 + 2 支线` 结构
- 采用 `自然日刷新`
- 主界面新增独立目标卡
- 每日目标状态支持存档恢复

## 3. 本轮完成内容

### 3.1 新增 DailyGoalSystem

本轮新增：

- `scripts/autoload/daily_goal_system.gd`

首版职责包括：

- 生成并维护“今日目标”
- 管理 `1 主目标 + 2 支线`
- 生成“回来先做什么”的摘要
- 持久化每日目标状态
- 按自然日自动刷新

当前首版任务池已覆盖三类：

- 推图主目标
- 研究支线
- 刷装支线

推荐逻辑采用当前约定的：

- `推图优先`

也就是说，主目标总是优先围绕章节/节点推进，研究和刷装只作为辅助任务与转向建议存在。

### 3.2 HUD 新增独立目标卡

本轮在现有右侧 `TargetCard` 之外，新加了一块独立的 `DailyGoalCard`。

它现在固定展示：

- 今日主目标
- 主目标进度
- 主目标 CTA
- 两条支线摘要
- 一行“回来先做什么”

这一版没有替换原有追踪目标卡，而是保留：

- 原 `TargetCard` 继续展示当前追踪目标和图鉴摘要
- 新 `DailyGoalCard` 专门承载每日目标层

这样用户一进游戏就能同时看到：

- 当前 build 在追什么
- 今天最值得先做什么

### 3.3 离线收益弹窗升级为“收益 + 下一步”

本轮将离线结算弹窗改为双段结构：

- `本次离线收益`
- `回来先做什么`

具体调整包括：

- 将原先单块长文本改成滚动文本区
- 新增目标建议区
- 固定展示：
  - 下一步建议
  - 今日主目标摘要
  - 最接近完成的支线

这样离线回来时不再只是“看完收益就关掉”，而是能马上接到主线目标。

### 3.4 每日目标接入存档

本轮已把每日目标系统接入现有存档流：

- `SaveManager.save_game()` 会写入每日目标状态
- `SaveManager.load_game()` 会恢复每日目标状态
- 旧存档若没有这部分字段，会自动生成当日目标
- 跨自然日读档时，会自动刷新为新一轮每日目标

这样这套目标层已经不是临时 UI，而是正式进入项目运行状态。

### 3.5 新增事件与 UI 刷新接线

为了让每日目标能随游玩变化推进，本轮还接入了以下事件：

- `battle_finished`
- `resources_changed`
- `research_changed`
- `codex_changed`
- `loot_target_changed`

同时在 `EventBus` 中补了：

- `daily_goals_changed`

用于 HUD 和离线弹窗同步刷新。
