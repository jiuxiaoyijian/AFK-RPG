## 已完成

### 1. 已补轻量 UI Focus Request 机制

本轮在 `GameManager` 增加了轻量 UI 焦点请求：

- `set_ui_focus_request(panel_id, payload)`
- `consume_ui_focus_request(panel_id)`

这套机制只用于面板打开时的临时聚焦：

- 不改存档
- 不改底层物品结构
- 只负责“从 HUD 跳进某个面板后，默认帮玩家选中正确目标”

### 2. 背包与百炼坊已支持消费焦点请求

`InventoryPanel` 现在打开时会读取焦点请求，可自动：

- 聚焦某件最近入包的装备
- 聚焦某个已穿纸娃娃槽位
- 同步筛选、排序与页码

`CubePanel` 打开时也会读取焦点请求，可自动：

- 切到指定工坊页
- 选中某件候选物
- 带入目标槽位 / 精炼词条 / 状态提示

### 3. 掉落高光已进入真正的处理链

本轮 `loot highlight` 数据新增了：

- `item_id`
- `target_slot`

这样 HUD 的掉落高光不再只是“告诉你掉了什么”，而是可以直接把玩家送去正确决策点。

当前点击 `最近掉落` 的行为是：

- 可萃取武学装备：直达 `百炼坊 -> 萃取武学`
- 自动换装产物：直达 `背包`，并聚焦对应已穿槽位
- 普通高价值入包装备：直达 `背包`，并聚焦该物品

### 4. HUD 四张核心卡片都已可点击

当前已接入的入口：

- `当前 Build` -> `推演与秘境`
- `当前目标` -> `成长中心`
- `已穿装备` -> `背包`
- `最近掉落` -> `背包 / 百炼坊`

同时这些卡片也补了指针样式与 tooltip，避免玩家不知道它们是可点入口。

### 5. 文档已补 HUD 焦点路由说明

`文档/15_RPG化系统与交互重构方案.md` 已新增：

- `HUD 焦点路由`
- `HUD 到面板的直达链路`

后续 Figma / prototype 同步时，可以直接按这版交互约定继续。

## 当前边界

本轮虽然已经让 HUD 进入真实处理链，但还有这些边界：

- 焦点请求目前仍是一次性轻量态，不含更复杂的跨面板历史记录
- 还没有做“面板内返回 HUD 后保留选中上下文”
- 还没有做拖拽交互
- 原型页和 Figma 还没同步到这轮 HUD 可点击结构
