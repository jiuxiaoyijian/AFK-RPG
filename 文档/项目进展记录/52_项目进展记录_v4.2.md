# 项目进展记录 v4.2

日期：2026-03-25

---

## 本轮目标

在不重写整套导航结构的前提下，先把 `P2` 的最小可运行切片接进当前工程：

1. `试剑秘境` 真实运行态
2. `宝石` 拥有/镶嵌/战斗加成
3. `华戒` 掉落与传承 `+2 件数`
4. 现有 `P` 面板临时升级为 `推演与秘境`

---

## 已完成

### 1. 试剑秘境已进入真实战斗链路

`RiftSystem` 本轮已从脚手架扩展为真实状态机，当前状态包含：

- `current_level`
- `highest_level`
- `owned_keys`
- `recent_results`
- `active_run`

已实现能力：

- 按层数读取 `rift_scaling.json`
- 自动挑选可用 `试剑令`
- 开启秘境时写入
  - `base_node_id`
  - `time_limit`
  - `enemy_hp_multiplier`
  - `enemy_damage_multiplier`
  - `reward_multiplier`
- 通关/失败后落入 `recent_results`
- 首领节点通关后补发 `试剑令`
- 通关高层时追加 `宝石 / 华戒` 特殊奖励

`CombatSystem` 已开始接入这些状态：

- 战斗基底节点改为读取 `GameManager.get_active_combat_node_id()`
- 秘境怪物会吃到真实 `HP / 伤害` 倍率
- 秘境战斗已加入倒计时失败
- 秘境通关后不推进主线节点，只回到当前驻守节点
- 普通首领通关后会补发 `试剑令`

### 2. 宝石已具备最小闭环

`GemSystem` 已新增：

- `sanitize_state()`
- `equip_gem()`
- `grant_rift_reward()`
- `build_runtime_state()`
- `build_combat_bonuses()`

当前宝石状态仍保持轻量：

- `owned_gems`
- `equipped_gems`

本轮规则：

- 宝石奖励会优先贴合当前道统
- 第一次获得宝石会自动镶入首个空饰品槽
- 同一宝石不能同时镶入两个饰品槽
- 战斗加成已开始真实叠加进 `BuildQueryService.get_total_combat_bonuses()`

### 3. 华戒已进入传承件数计算

本轮新增了最小版 `华戒` 奖励物：

- 掉落形态为饰品
- 物品详情会显示映照的传承名
- `SetSystem.get_equipped_set_counts()` 已支持 `hua_ring_set_id -> +2 件`

这意味着当前已经可以在运行时验证：

- 正常传承件数
- `华戒` 补件后的 `2/4/6` 件加成变化

### 4. 现有 `P` 面板已临时升级为“推演与秘境”

为了避免本轮把功能接线和导航重写绑死，当前继续复用现有 `drop_stats` 面板 id。

本轮已完成：

- 面板标题改为 `推演与秘境`
- 顶部摘要增加秘境层数、钥匙、进行中状态
- 面板内新增
  - 秘境层数加减
  - 开启秘境按钮
  - 两个饰品宝石切换按钮
- `MainNavBar` 中 `P` 按钮文案已改为 `推演 / 秘境`

也就是说，现在线路已经变成：

- `P` 打开临时秘境入口
- `B` 打开百炼坊
- `O` 继续保留给 `江湖见闻录`

本轮仍然只是 `P2` 的最小可运行切片，还没有进入更完整的玩法与导航整理：

- `P` 面板目前是临时复用，不是最终的 `推演 / 秘境` 中心设计

- 秘境仍复用现有章节节点作为战斗基底
- `Paragon / Season` 仍只停留在默认存档状态
