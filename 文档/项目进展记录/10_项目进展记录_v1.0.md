# 项目进展记录 v1.0

## 1. 本轮定位

本轮承接 `v0.9` 的研究树版本，按既定顺序继续推进：

- 先补第二章与更高层掉落
- 再补精英专属词条与 Boss 专属传奇池
- 同步维护版本化项目进展记录

本文件是新增文档，不覆盖旧记录。

## 2. 本轮目标

目标：

- 让原型从“单章节循环”升级到“至少两章连续推进”
- 让高阶掉落不只是数值更大，而是来源更明确
- 建立 `normal / elite / boss` 三层更清晰的词条与传奇来源差异
- 让打完第一章后可以自然进入第二章

## 3. 本轮完成内容

### 3.1 第二章内容扩展

已新增 `chapter_2`：

- 章节名：`焦灼矿场`
- 节点数量：4 个推进节点 + 1 个 Boss 节点
- 新敌人：普通怪 3 个、精英 2 个、Boss 1 个
- 新掉落表：`drop_normal_ch2`、`drop_elite_ch2`、`drop_boss_ch2`

章节推进逻辑也已扩展：

- 通关 `chapter_1` Boss 后，不再回到第一章起点
- 现在会自动切到第二章首节点
- 切章时同步更新稳定节点，避免失败后退回上一章 Boss

### 3.2 掉落层级继续上探

第二章掉落的变化：

- 普通节点开始稳定产出更高 `item_level` 的底材
- 精英节点进入“高评分词条 + 传奇胚子 + 材料提升”区间
- Boss 节点开始进入 `epic / legendary / ancient` 掉落层

当前掉落层级大致分布：

- `chapter_1 normal`：普通词条、基础底材
- `chapter_1 elite`：成型装、核心材料
- `chapter_1 boss`：Boss 专属传奇池的早期入口
- `chapter_2 normal`：高级底材、稀有词条、更多 `core`
- `chapter_2 elite`：高评分精英词条、传奇胚子、少量 `legend_shard`
- `chapter_2 boss`：Boss 专属传奇池、远古胚子、碎片爆发

### 3.3 精英专属词条

本轮新增了一组仅在 `elite / boss` 来源中出现的词条：

- `破阵`：通用高强度攻击提升
- `暴卷`：旋风专属高阶范围强化
- `猎血`：流血专属处决线强化
- `过载弧`：连锁闪电专属高阶增伤

实现方式：

- 词条表新增 `source_profile_types`
- `LootSystem` 在生成词条时按掉落来源过滤
- 同一件装备本轮还增加了“尽量不重复词条”的处理

### 3.4 Boss 专属传奇池

本轮把 Boss 传奇池做成了真正的来源池，而不只是“Boss 更容易出传奇”：

- Boss 掉落表新增 `required_legendary_source_type`
- `LootSystem` 会在生成传奇特效时按该来源限定候选池
- 因此 Boss 掉落现在可以稳定区分于普通 / 精英掉落

本轮新增 Boss 专属传奇：

- `王座风暴`：旋风向
- `猩红敕令`：流血向
- `雷王御座`：连锁闪电向
- `暴君壳层`：通用向

这让当前 3 套 build 都有了更明确的 Boss 目标装方向。

### 3.5 高阶底材补充

为了避免第二章只是重复第一章的旧底材，本轮新增了一批更高等级基础装备：

- `锯齿斩刀`
- `风暴发射器`
- `矿场战面`
- `耐热重甲`
- `静电握套`

这些底材主要服务第二章的 `item_level` 区间。

## 4. 关键实现点

逻辑侧重点：

- `GameManager.complete_node()`：支持章节间切换

- `HUD`：当前显示会带上章节名、节点与节点类型，方便观察推进状态

## 7. 当前项目状态

截至 `v1.0`，项目已经具备：

- 两章连续推进
- 普通 / 精英 / Boss 三层更清晰的掉落来源差异
- 高价值材料的研究消耗回路
- 高阶底材、精英专属词条、Boss 专属传奇池
- 更接近刷装挂机游戏的中期结构

## 8. 当前已知问题

- 第二章已经接入推进与掉落，但场景视觉仍未做章节差异化
- 敌人行为还主要复用现有基础 AI，尚未做真正的章节专属技能机制
- 传奇与高阶词条虽然已按来源分层，但还没有“图鉴 / 目标追踪 / 定向 farming 提示”
- 系统边界依旧偏原型化，后续继续拆分会更稳

## 9. 推荐下一步

按当前节奏，下一轮建议优先做：

- 把研究树继续拆成更明确的 `战斗 / 挂机 / 经济` 三分支
- 增加更明显的章节视觉差异与节点表现
- 开始把 `GameManager / LootSystem / CombatSystem` 往更标准的系统边界拆分
