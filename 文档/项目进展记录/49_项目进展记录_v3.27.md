# 项目进展记录 v3.27

## 1. 本轮定位

本轮不再只修首页，而是直接处理“所有二级界面”的结构回灌。

核心目标是把：

- 背包
- 悟道
- 异闻录
- 机缘推演
- GM
- 闭关所得 / 破局要闻

这些二级界面从“已经换了主题色，但结构还偏工程原型”推进到“布局也开始更接近交互稿”。

## 2. 本轮目标

目标：

- 统一所有二级界面的标题栏、主体分区、操作区与关闭区
- 让各面板结构更贴近 `Figma_交互设计简介.md` 中的分栏与弹窗草图
- 在这轮重排后继续保证项目可运行

## 3. 本轮完成内容

### 3.1 背包面板已按“左列表 + 右详情 + 右下操作区”重排

本轮对 `InventoryPanel` 做了结构重排。

当前背包面板更接近交互稿中的：

- 左侧物品列表
- 右侧装备详情
- 底部操作区

具体包括：

- 列表区与详情区重新分栏
- 自动分解阈值按钮回到操作区
- 装备 / 锁定 / 分解三个按钮重新排列
- 顶部摘要行保留在标题栏下方

这让背包不再只是“左边 ItemList，右边一大块文字”，而是开始有正式面板的分区秩序。

### 3.2 悟道与异闻录面板已统一成“顶部筛选 + 左列表 + 右详情 + 底部操作”

本轮对 `ResearchPanel` 和 `CodexPanel` 做了同步重排。

当前这两张面板都已统一到：

- 顶部标题栏
- 标题栏下的筛选区
- 左侧条目列表
- 右侧详情区
- 右下操作区

这样它们的使用逻辑与视觉逻辑都更一致了：

- 悟道：看列表 → 看详情 → 点悟道
- 异闻录：看条目 → 看详情 → 设为机缘追踪

也更贴近网页稿的模块化结构。

### 3.3 机缘推演面板已从“左窄右宽”继续收拢成更像评审稿的纵向阅读结构

`DropStatsPanel` 本轮改动较大。

当前结构更接近：

- 顶部概览
- 快捷定位 / 当前节点 / 推荐机缘
- 中部节点列表
- 下方样本概览
- 再下方三条效率条
- 最底部最近样本

虽然内部数据仍沿用现有脚本逻辑，但阅读路径已经更像一张真正的“推演面板”，而不是纯工程统计页。

### 3.4 GM 面板也已接入统一模态结构

本轮没有忽略 `GMPanel`。

它现在也进入了和其他二级界面一致的模式：

- 居中主面板
- 顶部标题栏
- 中部四个功能分区
- 底部存档 / 清理操作 / 最近操作反馈

这样 GM 面板虽然仍是调试用途，但不再在视觉上完全跳出整套 UI 体系。

### 3.5 闭关所得 / 破局要闻弹窗已补出更明确的内容分区

本轮还对 `OfflineReportPopup` 做了结构增强：

- 新增收益内容区底板
- 新增目标建议区底板
- 关闭按钮改为更居中的确认区

这让 `闭关所得 / 破局要闻` 两种模式更接近文档里那种“结果卡 + 内容块”的展示方式。
