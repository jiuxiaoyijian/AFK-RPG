# 项目进展记录 v3.4

## 1. 本轮定位

本轮承接前几轮已经完成的：

- 角色 / 怪物 / Boss 占位立绘
- 章节背景与流派图标
- 掉落高光卡、品质边框和结果卡 UI

继续把 `刷宝感` 从“界面上看起来像在掉东西”，推进到“战斗里真的有东西掉在地上、被捡起来、飞回 UI”。

本轮重点不是改数值，而是补齐掉落演出的完整生命周期：

- 掉落
- 落地
- 等待拾取
- 主动拾取
- 超时自动拾取
- 飞向 UI 收束消失

## 2. 本轮目标

目标：

- 让怪物死亡后的资源与装备掉落具备实体感
- 让掉落不再只存在于 HUD 文本里，而是在地面短暂停留
- 让拾取过程具备“收菜”和“刷到宝”的动作反馈
- 保持原有数值结算链路不被打乱

## 3. 本轮完成内容

### 3.1 掉落物改为先抛物线落地

本轮新增了独立的世界掉落演出节点 `LootDropVisual`，所有掉落演出不再直接原地闪现，而会：

- 从怪物死亡点抛出
- 走一段短抛物线
- 落在地面附近

不同掉落类型还带有不同参数：

- 普通资源掉落：弧线较低、扩散较小
- 装备掉落：弧线更高、落点更散
- 高品质掉落：落地后保留光柱

这样视觉上已经更接近刷宝游戏里“怪死后地上散出东西”的节奏。

### 3.2 主角靠近可主动拾取

掉落物落地后会进入 `grounded` 状态。

在这个状态下，如果主角进入一定范围内，会触发主动拾取：

- 当前使用固定拾取半径
- 不需要玩家点击
- 靠近后立刻开始收束动画

这让角色移动路径和地面掉落之间产生了更真实的互动关系。

### 3.3 3 秒无人拾取触发被动拾取

为避免挂机过程中掉落物长期堆积，本轮同时加入了自动拾取兜底：

- 掉落落地后开始计时
- 3 秒内若主角没有靠近
- 则自动触发被动拾取

这一点很重要，因为当前项目仍是挂机自动战斗导向，必须保证：

- 有地面掉落演出
- 但不会因为地面物件长期滞留影响节奏

### 3.4 拾取后飞向对应 UI 入口

本轮实现了拾取后的“飞 UI”收束逻辑：

- 货币类掉落飞向顶部资源栏
- 装备类掉落飞向底部 `背包` 按钮

为了做到这一点，本轮补了两个 UI 目标接口：

- `HUD` 提供资源栏目标坐标
- `MainNavBar` 提供背包按钮目标坐标

这样掉落物在拾取时不只是消失，而是有一个明确的“回到玩家资产入口”的方向感。

### 3.5 拾取效果改为品质色粒子收束

拾取触发后，掉落物不会直接淡出，而会先收束为一簇对应品质颜色的粒子，再飞向 UI 入口：

- 普通资源：偏金色 / 橙色 / 蓝色
- 装备：按品质使用不同颜色
- 传奇 / 远古：保留更强的高亮色和光柱

收束过程包括：

- 地面物件本体隐藏
- 生成品质色小粒子
- 粒子向中心聚合
- 整体飞向 UI 目标
- 到达后消失

这一版已经具备了比较明确的“掉到地上 -> 被吸走 -> 进入资产入口”的闭环感。

### 3.6 掉落演出与真实奖励结算解耦

本轮为了支持“落地等待 3 秒再自动拾取”，对掉落演出链路做了一个关键调整：

- 真实奖励仍然按原有逻辑结算
- 掉落演出对象独立存在于 `LootContainer`
- 新节点开始时不再立刻清空旧的演出对象

这样就避免了：

- 节点结算后马上切下一个节点
- 结果上一波掉落刚出现就被清空

现在旧掉落演出可以短暂停留，并在合适时机自己完成拾取流程。

## 4. 本轮关键实现点

本轮新增和修改的关键点主要集中在 `掉落演出节点 + 战斗层拾取调度 + UI 目标接口`：

- `scenes/effects/loot_drop_visual.tscn`
- `scripts/effects/loot_drop_visual.gd`
- `scripts/systems/combat_system.gd`
- `scripts/systems/loot_system.gd`
- `scripts/entities/enemy_actor.gd`
- `scripts/ui/hud_controller.gd`
- `scripts/ui/main_nav_bar_controller.gd`

其中核心职责分工为：

- `EnemyActor`：死亡时把位置和敌人类型传出来
- `LootSystem`：继续负责真实掉落结算，同时额外生成“掉落演出数据”
- `CombatSystem`：负责在世界里生成演出对象，并调度主动/被动拾取
- `LootDropVisual`：负责抛物线、落地等待、粒子收束、飞 UI 消失

- 拾取过程还没有加入音效
- 传奇掉落虽然有光柱，但缺少更强的全屏级提示
- 当前还没有“点击地面查看掉落详情”这类交互层
