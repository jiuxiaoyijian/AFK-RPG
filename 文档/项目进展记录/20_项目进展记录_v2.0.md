# 项目进展记录 v2.0

## 1. 本轮定位

本轮不是扩新系统，而是回头修一个会影响自动战斗连续性的基础体验问题：

- 无怪物时主角持续向右移动
- 主角可能移动出屏幕
- 导致自动战斗观感和流程都被打断

## 2. 本轮目标

目标：

- 修复无目标时主角仍持续推进的问题
- 防止主角离开可视战斗区域
- 保证战斗空窗期不会打断自动战斗节奏

## 3. 本轮完成内容

### 3.1 无怪物时主角不再继续前冲

问题原因是：

- `CombatSystem` 在没有存活敌人时，仍把 `player.should_move` 维持为 `true`

这会让主角在以下阶段继续向右推进：

- 波次清空后等待下一波
- 节点完成后的结算等待
- 节点失败后的回退等待
- 某些无敌人阶段

本轮已统一改为在这些空窗状态中：

- 清空 `player.target`
- 设置 `player.should_move = false`

这样主角会在无战斗目标时停下，而不是继续跑出战场。

### 3.2 主角出生后默认先停住

此前 `PlayerActor.reset_state()` 会把 `should_move` 设为 `true`，这意味着即使刚进入节点、怪还没刷出来，主角也会先向前冲。

本轮已改为：

- 角色重置后默认静止

然后由 `CombatSystem` 在真正进入追敌阶段时再决定是否移动。

这让战斗状态控制重新回到战斗系统，而不是角色自己默认前进。

### 3.3 增加屏幕边界保护

为了避免未来其他状态再次把主角推离屏幕，本轮还在 `PlayerActor` 中增加了屏幕边界限制：

- 每帧把主角 `x` 坐标限制在屏幕可视区域内

这相当于给战斗流程加了一层保险，即使后续有遗漏状态，也不容易再出现角色跑出屏幕的问题。

## 4. 本轮关键实现点

逻辑侧重点：

- `CombatSystem._update_player_targeting()`：无敌人时停下
- `CombatSystem._check_wave_progress()`：波次等待/结算等待时停下
- `CombatSystem._on_player_died()`：失败等待时停下
- `PlayerActor.reset_state()`：出生后默认静止
- `PlayerActor._physics_process()`：增加屏幕边界夹取

## 7. 当前项目状态

截至 `v2.0`，自动战斗在基础体验上更稳定了：

- 无怪时角色不会继续乱跑
- 战斗空窗期不会把主角送出屏幕
- 自动战斗节奏更连续

这类修复虽然不属于大功能扩展，但对挂机游戏的“持续观看体验”和“稳定循环感”很关键。

## 8. 当前已知问题

- 当前只是限制角色不出屏，还没有做更自然的“待机站位 / 回到默认站位”表现
- 战斗中的位移仍然比较功能性，缺少更明显的前压感和收招感

## 9. 推荐下一步

按当前节奏，下一轮建议优先做：

- 给主角补一个更自然的待机站位逻辑，而不是单纯停住
- 或给战斗状态增加更明显的动画/视觉反馈
- 或继续推进掉落统计面板的视觉化表现
