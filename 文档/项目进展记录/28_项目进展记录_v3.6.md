# 项目进展记录 v3.6

## 1. 本轮定位

本轮主要围绕两个目标继续推进：

- 继续强化 `地面掉落 -> 收束 -> 飞向 UI` 的视觉存在感
- 增强项目的 `调试效率` 与 `移动端可用性`

用户本轮提出的关键需求包括：

- 飞向 UI 的粒子效果继续放大，并延长持续时间
- 飞向 UI 的粒子层级调到最高，避免被其他 UI 遮挡
- 增加 GM 面板，支持资源、道具、图鉴、天赋树等快速调试
- 增加移动端横屏适配，确保手机上保持横屏体验

## 2. 本轮目标

目标：

- 让飞 UI 收束不再只是“隐约有点粒子”，而是明显可感知的收菜反馈
- 让收束特效真正成为最上层演出，不会被任何 HUD 或弹窗压住
- 让测试时不必频繁改存档或改代码，直接通过 GM 面板完成常见调试
- 为后续移动端试玩打基础，先锁定横屏体验

## 3. 本轮完成内容

### 3.1 飞向 UI 的粒子继续放大并延长

本轮再次强化了拾取后的收束特效：

- 粒子数量增加
- 单个粒子尺寸显著放大
- 初始扩散半径显著增大
- 粒子收束时间延长
- 飞向 UI 的整体飞行时间延长

这一版的观感已经从“地面东西轻轻一闪就没了”，变成更明确的：

- 地面物件消失
- 大颗粒品质色能量向中心聚合
- 聚合后整体飞向资源栏或背包

### 3.2 收束特效改到最高层 UI 特效层

之前即使提高 `z_index`，由于掉落物仍在世界层，飞 UI 的收束演出依旧可能被 `CanvasLayer` 内的 UI 压住。

本轮专门新增了顶层特效层：

- `TopEffectLayer`
- `CollectEffects`

当掉落进入收束阶段时，会被重新挂载到顶层特效层中，再执行飞 UI 动画。

这样可以确保：

- 收束粒子
- 飞行中的能量团

始终显示在现有 UI 面板、HUD、导航条之上。

### 3.3 新增 GM 调试面板

本轮新增了正式的 `GMPanel`，并接入了当前 UI 管理体系。

打开方式：

- 主导航新增 `GM` 按钮
- 键盘快捷键 `G`

当前已经支持的调试能力包括：

- 指定资源加数值
- 指定资源直接设置为某数值
- 向背包生成指定底材、指定稀有度、指定等级的测试装备
- 可选强制附带传奇特效
- 跳转到任意节点
- 重开当前节点
- 一键解锁全部天赋树
- 一键解锁全部图鉴
- 一键保存 / 读档

这意味着后续很多迭代工作都不需要再通过改 JSON、改存档或跑多轮流程才能验证。

### 3.4 增加移动端横屏适配基础配置

本轮先完成了移动端横屏的基础项目设置：

- 增加 `display/window/handheld/orientation=0`
- 增加 `display/window/stretch/aspect="keep"`

其中：

- `orientation=0` 对应默认横屏
- `aspect="keep"` 用于尽量保持当前横向布局比例

这一轮主要是先把项目级约束立起来，保证后续移动端试玩时不会默认竖屏打开。

### 3.5 HUD 与导航提示同步更新

为了让测试流程更顺手，本轮把提示信息也同步改了：

- 主 HUD 提示里加入 `G`
- 主导航提示里加入 `G 打开 GM`

这样在试玩和联调时，不需要额外记忆面板入口。

## 7. 当前项目状态

截至 `v3.6`，项目已经从：

- 只有基础飞 UI 动画
- 缺少统一调试入口
- 没有移动端方向约束

推进到：

- 飞 UI 粒子明显放大并延长
- 收束特效提升到最上层
- 具备可直接操作资源 / 装备 / 节点 / 图鉴 / 天赋树的 GM 面板
- 项目级移动端横屏配置已就位

## 8. 当前已知问题

- 飞 UI 粒子虽然已大幅增强，但还没有拖尾、残影或光带
- GM 面板目前以“够用”为目标，视觉样式还偏调试工具
- 移动端适配目前还是项目设置级别，尚未做触控按钮与安全区专项优化
- 暂未验证高刘海 / 异形屏设备上的边缘占位

## 9. 推荐下一步

按当前节奏，下一轮建议优先做：

- 给飞 UI 收束增加拖尾或品质色光带
- 为 GM 面板补更多常用项，例如跳章节、清空背包、多档测试快照
- 开始做移动端触控按钮与安全区适配
