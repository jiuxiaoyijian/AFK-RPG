# 项目进展记录 v3.8

## 1. 本轮定位

本轮继续围绕 `调试效率` 与 `主战斗画面可读性` 做收束。

用户本轮提出的重点包括：

- 继续推进
- 存档位置增加到 3 个
- GM 面板增加全部系统资源重置功能
- 右侧功能按键说明不要再遮挡目标追踪区域
- 所有 UI 字号缩小一档，并尽量给中心战斗区域留出视觉中心

因此这一轮的核心不在于新增战斗系统，而在于：

- 让测试更方便
- 让 HUD 更收敛
- 让战斗画面的“视觉中心”更明确

## 2. 本轮目标

目标：

- 把单档存档扩成 3 档，方便多进度并行测试
- 让 GM 面板具备更完整的“回到初始状态”能力
- 把右侧提示从目标追踪信息里剥离出去
- 让 HUD、底部导航、弹窗整体字号缩小一档
- 收瘦四周 UI，让中部战斗区更像舞台中心

## 3. 本轮完成内容

### 3.1 单档存档扩展为 3 档

本轮将原本单一存档文件扩展为了 3 个独立存档位。

当前规则为：

- `F5 / F8` 仍默认作用于 `档位 1`
- GM 面板中可手动选择 `1~3` 档执行保存和读档

这样现在可以用来做：

- 正常游玩进度
- 某章节 / 某节点专项测试
- 高资源 / 高成长调试档

三条线并行，不再互相覆盖。

### 3.2 GM 面板增加“全部系统资源重置”

本轮为 GM 面板新增了 `资源清零` 能力，用于一键把核心系统资源重置回 0。

当前 GM 面板和资源相关的常用按钮已经包括：

- 增加资源
- 设置资源
- 资源拉满
- 资源清零

这样无论你是要验证：

- 缺资源时的交互
- 高资源时的购买 / 升级
- 极限边界值下的表现

都比之前更快。

### 3.3 GM 面板的存读档入口支持选择档位

为了配合三档存档，本轮 GM 面板新增了：

- 存档位下拉框
- 针对所选档位执行保存 / 读档
- 档位说明文字

这让测试时可以明确知道当前是在操作哪一个档位，而不是只能依赖默认热键档位。

### 3.4 右侧功能说明上移，避免遮挡目标追踪

你截图里指出的问题，本轮已经处理：

- 原本右侧功能说明在 `目标追踪` 卡片内部
- 现在它被拆出来，单独移到了目标追踪卡上方

这样就避免了：

- 目标描述
- 图鉴统计
- 快捷键说明

三块内容在同一块区域内互相挤压和遮挡。

### 3.5 HUD 与主导航整体缩小一档

本轮对 HUD 和主导航做了统一收瘦：

- 标签字号整体缩小一档
- 底部导航按钮字号缩小一档
- 弹出式面板字号也统一缩小一档

同时配合布局调整：

- 顶部资源条变矮
- 左侧战斗信息卡缩小
- 右侧目标追踪卡缩小
- 底部导航栏变矮
- 左下装备概览和右下最近掉落都进一步贴边收紧

最终目标是让：

- 信息仍然可读
- 中间战斗区域不再被四周 UI 压得过满

### 3.6 中心战斗区留出更明确的视觉中心

本轮并不是只缩字号，而是把四周框体一起收窄。

这样带来的效果是：

- 画面中部更像真正的战斗舞台
- 角色、怪物、掉落与飞 UI 动效更容易成为视觉焦点
- HUD 从“压住画面”转为“围绕画面服务”

## 7. 当前项目状态

截至 `v3.8`，项目已经从：

- 单档存档
- GM 只能做增量调试
- 右侧提示与追踪信息挤在一起
- HUD 对战斗中心挤压偏强

- 右侧快捷提示独立上移
- HUD / 主导航 / 弹窗统一缩小一档
- 中心战斗区域拥有更明确的视觉中心
