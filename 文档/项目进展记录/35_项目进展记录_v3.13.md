## 1. 本轮定位

本轮继续沿上一轮的工程化交付方向推进，但目标已经从“打通链路”升级为“真正对外可访问”：

这一轮的核心意义，是让项目第一次具备“公开展示 + 在线访问”的对外形态。

目标：

- 将 `AFK-RPG` 从私有仓库改为公开仓库

## 3. 本轮完成内容

### 3.1 仓库已切换为公开

本轮已将仓库：

- `https://github.com/jiuxiaoyijian/AFK-RPG`

从 `private` 切换为 `public`。

这一步直接解除上一轮里 GitHub Pages 的平台限制，使得 Pages 站点能够被正常创建与访问。

### 3.2 新增公开展示用 README

项目此前虽然已有完整文档沉淀，但仓库根目录还缺少一份对外可读的首页说明。

本轮新增 `README.md`，内容已整理为公开仓库视角，覆盖：

- 项目定位
- 当前已实现系统
- 当前键位
- 本地运行方式
- Web 导出方式
- 仓库结构
- 文档入口
- GitHub Pages 入口

这样首次访问仓库的人，不需要先翻内部文档也能快速理解项目是什么、做到哪一步、怎么运行。

### 3.3 修复 GitHub Actions 的 Web 导出环境

仓库公开后，Pages 已经可以创建，但工作流第一次真正执行导出时又暴露了新的问题：

- CI 使用的是 `Godot mono`
- Godot 4 的 `.NET/Mono` 版本无法导出 Web

本轮已修复：

- 在 `deploy-pages.yml` 中显式设置 `use-dotnet: false`

这样 GitHub Actions 会使用非 `.NET` 的 Godot 编辑器，从而顺利执行 `Web` preset 导出。

### 3.4 GitHub Pages 已正式上线

在修复 CI 导出环境后，最新一次工作流已成功通过。

确认结果：

- `Deploy Web Build` 工作流成功
- GitHub Pages 站点已启用
- 站点地址返回 `HTTP 200`

当前可访问地址为：

- `https://jiuxiaoyijian.github.io/AFK-RPG/`

同时，本轮还将仓库主页链接同步指向了这个 Pages 地址，方便从 GitHub 仓库首页直接跳转试玩。
