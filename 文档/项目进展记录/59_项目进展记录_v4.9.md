在上一轮已经完成程序化正式资源替换和批量生成脚本后，进一步接入 OpenAI 官方生图接口，把当前工程真正会运行到的高频资源替换成 GPT 实际生成版本。

本轮聚焦：

1. 接入 OpenAI 官方生图批处理脚本
2. 先验证第三章背景可成功生成、裁切和替换
3. 把当前已接线的 43 个运行资源全部替换为 GPT 生图版本
4. 再次验证 Godot 启动与资源引用稳定性

---

## 已完成

### 1. 已新增 OpenAI 官方生图批处理脚本

本轮新增：

- `tools/generate_openai_art_assets.py`

脚本能力包括：

- 复用现有 `generate_art_assets.py` 的资源清单
- 按 `wired / unwired / all` 范围筛选资源
- 自动调用 OpenAI Images API
- 按资源类型自动选择生成尺寸
- 自动保持透明背景或不透明背景
- 自动中心裁切并缩放到工程真实尺寸
- 自动输出生成日志

这意味着后续继续补美术时，不必手动一张张生成和改名，已经具备可重复执行的批处理链路。

### 2. 已完成最小验证并切到全量运行资源替换

本轮先用 `bg_tiesuo_zhen_v1a` 做了单资源验证，确认：

- OpenAI 接口可用
- 背景风格方向正确
- 输出 PNG 可被本地裁切成 `1280x720`
- 替换后尺寸校验通过

在这一步通过后，继续把当前工程已接线的全部 43 个运行资源切换到 GPT 生图版本。

### 3. 当前工程已接线的 43 个运行资源已完成 GPT 替换

本轮已由 OpenAI 实际生成并替换的运行资源包括：

- 第一章、第二章、第三章全部战斗背景
- 三位 Boss 头像
- 主角 `idle / move / combat`
- 通用普通敌人 / 精英敌人头像
- 已接线的主要系统图标、掉落图标、资源图标、流派图标
- 装备槽图标与导航图标
- 品质框与结果卡底板

这一步之后，游戏当前真正会显示的高频资源，已经不再主要依赖本地程序化占位图，而是切到 GPT 生图版本。

### 4. 已保留原始出图与生成日志

本轮新增了两类可追溯文件：

- 原始大图：`assets/generated/_openai_raw/`
- 生成日志：`文档/美术资源_openai生成日志_v1.jsonl`

日志中记录了：

- `logical_id`
- `output_path`
- 使用的 `model / quality`
- 请求尺寸与目标尺寸
- 背景模式
- 原始大图路径
- 实际提交到接口的 prompt

这样后续要复刻、微调或替换单张资源时，不需要重新猜测参数。

## 验证结果

### 资源验证

当前工程已接线的 43 个资源已由脚本自动完成尺寸校验。

关键类型包括：

- 背景 `1280x720`
- 头像 `768x768`
- 图标与框体 `256x256`
- 结果卡 `1024x512`

### Godot 启动验证

本轮再次通过 Godot MCP 调试启动验证。

结果：

- 无 `missing resource`
- 无新增 parse error
- 无新增 runtime error

当前剩余 warning 主要是：

- 既有 `EventBus` signal warning
- 既有 `BuildQueryService / LootRuleService` 常量名与全局类重名 warning

这些 warning 不是本轮美术替换引入的资源断链问题。
