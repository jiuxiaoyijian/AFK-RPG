# 项目进展记录 v5.9

日期：2026-03-29

---

## 本轮目标

修正横版三层背景存在的两个根问题：

1. 三层背景虽然有 `far / mid / near` 文件，但本质上仍是单张图的颜色/透明度后处理，不是真正的语义分层
2. 背景虽然在运行时做了双图循环，但资源本身没有循环意识，首尾接缝仍然会突兀

---

## 本轮完成

### 1. 正式三层背景脚本重做

已重写：

- `tools/generate_parallax_background_layers.py`

当前新脚本不再做简单明暗分层，而是改成：

- 单张母版输入
- 横向扩展成超宽 `master`
- 对 `master` 做 `loop-safe`
- 基于 `profile` 规则生成：
  - `__far`
  - `__mid`
  - `__near`

第一版 profile 已新增：

- `chapter1_town_road`

### 2. 超宽背景输出规范落地

当前脚本默认输出：

- `3072x720`

并固定生成：

- `bg_chapter1_travel_road.png`
- `bg_chapter1_travel_road_loopsafe.png`
- `bg_chapter1_travel_road_loopsafe__far.png`
- `bg_chapter1_travel_road_loopsafe__mid.png`
- `bg_chapter1_travel_road_loopsafe__near.png`

### 3. CombatSystem 开始消费正式第一章背景

已调整：

- `scripts/systems/combat_system.gd`

当前第一章背景路径已经切换为：

- `res://assets/generated/afk_rpg_formal/backgrounds/bg_chapter1_travel_road_loopsafe.png`

第一章节点 `ch1_n1 / ch1_n2 / ch1_n3 / ch1_boss` 也统一切到该背景基底。

### 4. 循环参数改为更贴近真实纹理宽度

`CombatSystem` 当前已不再用“屏宽与贴图宽度取最大值”的粗略 spacing。

当前改为：

- 优先使用实际贴图宽度 * 缩放值
- 若配置显式写了 `loop_spacing`，则尊重显式配置

同时已新增：

- `x_anchor_bias`

用于后续微调超宽背景在屏幕中的横向锚点。

## 当前判断

本轮完成后，横版三层背景的正式方案已经从：

- “单图 + 明暗拆层 + 双图复制”

开始升级为：

- “单图母版 + 超宽扩展 + loopsafe + profile 拆层 + 双图循环”

这仍然不是最终极方案，但已经足以开始验证：

- 画风一致性
- 层次职责
- 首尾衔接自然度
- 第一章实际战斗体验

## 2026-03-29 补充更新

- 第一章母版已升级为更严提示词约束的 `chapter1_travel_road_master_v3`：

- 第一章运行时背景已切到：
  - `res://assets/generated/afk_rpg_formal/backgrounds/bg_chapter1_travel_road_v2_loopsafe.png`
- 当前量化结果：
  - 旧版 `near` 的 `strong alpha ratio` 约为 `0.022`
  - 新版 `v2 near` 的 `strong alpha ratio` 提升到约 `0.175`
  - 说明近景层已经更接近“真实前景物件”，而不是整张图的淡薄重影
