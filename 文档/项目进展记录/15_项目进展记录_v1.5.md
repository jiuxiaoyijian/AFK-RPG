# 项目进展记录 v1.5

## 1. 本轮定位

本轮承接 `v1.4` 的图鉴与目标追踪系统，继续把这套功能从“可用”推进到“更接近正式功能”：

- 扩展图鉴分类浏览
- 升级刷图推荐逻辑
- 继续维护版本化项目进展记录

## 2. 本轮目标

目标：

- 把图鉴从“只有传奇列表”扩成更完整的 `底材 / 词条 / 传奇` 浏览
- 把推荐刷图从“最早可掉落”升级为“更值得刷”的推荐
- 让玩家能更清楚地理解掉落系统和 build 追求

## 3. 本轮完成内容

### 3.1 图鉴升级为分类浏览

本轮后，图鉴不再只有传奇条目，而是支持三大类浏览：

- `底材`
- `词条`
- `传奇`

图鉴面板新增分类筛选按钮：

- `全部`
- `底材`
- `词条`
- `传奇`

当前在 `全部` 模式下，会把三类条目混合展示；切到具体分类后，则只看该类数据。

### 3.2 各类条目详情已可查看

目前三类条目都已经能显示自己的详情文本：

- 底材：部位、基础等级、底材方向、推荐刷图
- 词条：词条类型、适配流派、作用属性、数值范围、推荐刷图
- 传奇：适配流派、描述、掉落来源、追踪状态、推荐刷图

这让图鉴开始从“收集记录”变成“可查询数据库”。

### 3.3 推荐刷图逻辑升级为评分推荐

本轮不再只按“最早可掉落节点”推荐，而是改成了带评分的推荐：

评分会综合考虑：

- 掉装轮次
- 装备掉落概率
- 最大词条数
- 最高装等区间
- 是否保底传奇
- 是否偏好当前 build
- 当前章节是否已解锁/已推进到
- 节点类型（普通 / 精英 / Boss）
- 条目与当前流派的适配程度

这使得推荐更接近“刷哪更划算”，而不是“哪里理论上会掉”。

### 3.4 推荐逻辑已覆盖三类图鉴条目

现在推荐不只用于传奇：

- 底材也能推荐适合刷的章节节点
- 词条也能推荐更匹配的来源节点
- 传奇则继续保留目标追踪与推荐刷图

这意味着图鉴、掉落层级和 build 方向已经开始形成更完整的说明闭环。

### 3.5 追踪功能仍保持简洁

本轮没有让系统变复杂到难用，仍然保留：

- 只有传奇可设为追踪目标
- 底材和词条主要用于查询与推荐
- 当前追踪目标仍然可以通过 `T` 快速轮换

这种设计让“查询系统”和“追踪系统”分工更清楚。

## 4. 本轮关键实现点

逻辑侧重点：

- `LootCodexSystem.get_codex_entries(category)`：支持分类取条目
- `LootCodexSystem.get_codex_detail_text(entry_id, category)`：按分类输出详情
- `LootCodexSystem._get_best_recommendation_entry()`：统一做评分推荐
- `CodexPanel`：支持分类按钮、分类列表与按类型展示详情

## 7. 当前项目状态

截至 `v1.5`，图鉴系统已经从“最小可用”升级到：

- 有分类浏览
- 有目标追踪
- 有 build 相关推荐
- 有更像正式系统的查询能力

这让玩家不仅知道“追什么传奇”，也开始能看懂：

- 有哪些底材
- 有哪些词条
- 哪些节点更适合自己当前 build

## 8. 当前已知问题

- 当前推荐仍是规则评分，不是基于真实掉落统计或模拟期望值
- 图鉴条目还没有图标、颜色、稀有度视觉区分
- 当前追踪仍只支持单一传奇目标，没有多目标愿望单

## 9. 推荐下一步

按当前节奏，下一轮建议优先做：

- 做掉落推荐的“期望值可视化”或更真实的效率提示
- 或继续拆 `GameManager`，把节点推进和背包装备管理分离
- 或开始补章节视觉差异化，让当前玩法成长在画面上也更明显
