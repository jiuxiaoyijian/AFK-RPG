## 本轮完成

### 1. 运行时正式切到 5 层 `Parallax2D`

已调整：

- `scenes/main/game_root.tscn`
- `scripts/systems/combat_system.gd`

当前 `WorldLayer` 背景结构已经替换为：

- `SkyLayer`
- `FarLayer`
- `MidLayer`
- `NearBackLayer`
- `NearFrontLayer`

每层均使用 `Parallax2D` 驱动，不再继续沿用旧的三张 `Sprite2D` 背景层和手写 spacing 逻辑。

背景推进逻辑已经统一为：

- 主角向右推进
- 转波等待
- 结算持续推进

共享同一条 `forward_scroll_distance`，再由每层自己的：

- `scroll_speed_multipliers`
- `parallax_distances`
- `anchor_bias`

进行消费。

- `scripts/autoload/config_db.gd`

中补了正式查询接口：

- `get_parallax_scene(scene_id)`
- `get_parallax_scene_key(chapter_id, node_id)`

当前章节映射状态：

- 第一章：`chapter1_town_road_v1`

- 第三章：`legacy_validation_scroll_v1`

### 3. 第一章正式母版重新生成

已新增正式母版生成脚本：

- `tools/generate_formal_scene_master.py`

并重新生成了第一章母版：

- `assets/generated/formal_replacement_samples/backgrounds/chapter1_master_concept_v1.png`

这次母版提示词已显式约束：

- 同一地点、同一透视
- 中央横版战斗走廊干净
- 左右边缘具备可拆前景物件
- 禁止双影、重复前景、切边大地标

### 4. 拆层脚本升级为“内容感知 + 主层归属 + 去重修补”

已重写：

- `tools/generate_parallax_background_layers.py`

当前正式背景生产链为：

1. 单张母版输入
2. 生成 `3072x720` 的超宽 `master`
3. 做 `loop-safe`
4. 计算内容图：
   - `luma`
   - `saturation`
   - `detail`
   - `object`
   - `sky_bias`
5. 生成 5 张候选掩膜：
   - `sky`
   - `far`
   - `mid`
   - `near_back`
   - `near_front`
6. 使用“主层归属”而不是简单叠影：
   - `near_front` 只保留顶部挂件型物件
   - `near_back` 只保留底边和侧边近景器物
   - `mid` 在近景抽离处用模糊填补，尽量避免重影

当前第一章正式输出固定为：

- `chapter1_town_road_master_v1.png`
- `chapter1_town_road_master_loopsafe_v1.png`
- `chapter1_town_road__sky_v1.png`
- `chapter1_town_road__far_v1.png`
- `chapter1_town_road__mid_v1.png`
- `chapter1_town_road__near_back_v1.png`
- `chapter1_town_road__near_front_v1.png`

全部尺寸统一为：

- `3072x720`

## 当前判断

这一轮之后，背景系统已经真正切到新的正式方向：

- 运行时：`5 层 Parallax2D`
- 资源生产：`母版 -> loopsafe -> 内容感知 5 层拆分`
- 配置消费：`scene_key -> parallax_scene_defs.json`

3. 背景消费和章节映射已配置化，不再继续在 `combat_system.gd` 里堆路径常量
