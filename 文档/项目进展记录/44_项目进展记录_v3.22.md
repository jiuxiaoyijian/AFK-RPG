# 项目进展记录 v3.22

## 1. 本轮定位

本轮继续完善已经落地的 `P0-1 回流目标层` 与 `P0-2 Build 目标层`。

目标不是再加新面板，而是补齐当前版本里仍然偏弱的一层：

- 系统虽然会告诉玩家“缺什么”
- 但还不够明确说明“为什么现在要转向”
- 也还没有把“卡关时先回刷、先悟道、还是先筹材料”做成稳定建议

因此本轮重点是：

- 强化每日目标的转向能力
- 强化 build 建议里的卡关诊断
- 让 HUD 和闭关结算里都能更直接看见这层判断

## 2. 本轮目标

目标：

- 给 `GameManager` 增加更细的卡关诊断信息
- 让 `DailyGoalSystem` 在明显卡关时自动切换主目标方向
- 让刷装支线和悟道支线更贴合当前缺口
- 把诊断结果同步到 HUD 与闭关结算

## 3. 本轮完成内容

### 3.1 Build 建议已补入“卡关诊断”层

本轮扩展了 `GameManager.get_build_advice_data()`。

现在除了已有的：

- 当前缺口
- 下一件目标
- 推荐刷图

还会额外输出：

- 当前是否属于 `进度卡关`
- 当前卡住的是哪个节点
- 当前缺口的严重度
- 当前更适合 `继续推进 / 先回刷 / 先悟道 / 先筹悟道材料`
- 当前节点突破后的后续预告

这意味着 build 建议第一次不只是“告诉你该补什么”，而是开始回答：

- 你是不是已经卡住了
- 为什么该暂时转向
- 转向后再回来能解决什么

### 3.2 DailyGoalSystem 开始根据卡关状态改主目标

本轮没有增加新的目标卡，而是增强了 `DailyGoalSystem` 的主目标生成。

现在在以下情况里，主目标会从单纯“继续推图”切换成更贴近当前阻塞的方向：

- 明显卡关时，切到 `先补当前卡点`
- 可立刻悟道时，切到 `先做一次悟道`
- 悟道材料不足时，切到 `先筹悟道材料`

这让“今日机缘”从一个固定结构的任务池，变成了会根据当前 build 状态主动转向的目标层。

### 3.3 支线目标开始更明确表达“补哪种缺口”

本轮还增强了两条支线的文本：

- 悟道支线现在会更明确地围绕当前缺口生成说明
- 刷装支线标题从泛化的“刷装推进道统”调整为更明确的 `补伤害缺口 / 补生存缺口 / 补功能缺口`

这让用户即使只看 DailyGoalCard，也更容易理解：

- 当前为什么刷
- 当前刷装是在补哪一层

### 3.4 HUD 与闭关结算已同步显示卡点判断

本轮把这层诊断同步接到了两个高频入口：

- `HUD TargetCard`
- `OfflineReportPopup`

现在：

- HUD 右侧推荐信息在卡关时会优先显示 `卡点判断`
- 闭关结算“回来先做什么”里增加了 `卡点判断`
- 如果当前节点后面存在明确解锁结果，也会显示 `突破后续`

这样玩家回来之后，不仅知道“补什么”，也更容易理解“为什么现在不该硬顶当前节点”。

## 7. 当前项目状态

截至 `v3.22`，项目已经从：

- 知道当前缺什么
- 但转向建议还偏概括

推进到：

- 能识别当前是否已经卡在推进节点
- 能区分该继续推、该回刷、该悟道还是该先筹材料
- 能把这层判断体现在每日目标、HUD 和闭关结算里

这意味着 `P0-1` 与 `P0-2` 虽然早已完成首版，但现在又补上了一层更实用的“卡关转向能力”。

## 8. 当前已知问题

- 当前卡关诊断仍然是规则式首版，不包含真实失败次数和战斗耗时统计
- 还没有把“连续多次失败”或“长时间未推进”作为显式判断输入
- 悟道建议仍偏通用，没有完全按流派分支细分
- DailyGoalCard 仍然是轻量展示，还没有更细的说明性 UI

## 9. 推荐下一步

下一轮建议优先继续两个方向中的一个：

- 继续增强 `P0-3 / P0-4`，补更多中间阶段事件和战斗高光
- 或继续深挖当前诊断层，把 `失败次数 / 节点耗时 / 期望推进效率` 真正接进规则

如果继续沿当前方向往下做，最值得补的是：

- 连续卡关计数
- 更真实的节点推进效率判断
- 研究树与 build 建议的更深联动
