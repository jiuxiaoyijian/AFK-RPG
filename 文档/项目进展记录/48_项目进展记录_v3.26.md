# 项目进展记录 v3.26

## 1. 本轮定位

本轮继续承接 `v3.25` 的游戏内 UI 回灌，但重点不再是“统一主题”，而是做两件更细的收口工作：

- 把几处仍明显偏工程化的布局继续往交互稿靠拢
- 把本轮引入的新 warning 清回到只剩旧的 `EventBus` warning

因此这轮更像一次：

- `UI 几何细化 + 调试收口`

## 2. 本轮目标

目标：

- 继续收紧 HUD 与主面板的几何布局
- 让底部导航栏、BattleSafeFrame 和几个关键面板更贴近评审稿
- 把运行时 warning 收回到稳定基线

## 3. 本轮完成内容

### 3.1 HUD 与底部区域又向评审稿靠近了一步

本轮继续调整了 `HUD` 相关场景与布局逻辑。

重点变化：

- `BattleSafeFrame` 改为场景内默认可见
- `TargetCard` 改为场景内默认可见
- `EquipCard / LootCard` 的几何尺寸与位置继续向底部安全框收拢
- `BattleSafeFrame` 的高度进一步贴近网页稿结构

这使当前首页不只是“有这些块”，而是开始更像：

- 中间战斗
- 右侧双卡
- 底部统一安全框

### 3.2 底部导航栏与几个主面板继续做了几何整理

本轮还继续调整了：

- `main_nav_bar.tscn`
- `inventory_panel.tscn`
- `codex_panel.tscn`
- `drop_stats_panel.tscn`

主要方向是：

- 让底部导航 5 个按钮更接近等宽结构
- 把背包、异闻录、机缘推演里几处明显不够顺的区块位置收整
- 减少“标题、分区、正文、操作区”之间的工程式错位感

这轮还不是逐像素对齐，但已经在“结构秩序”上更接近网页稿。

### 3.3 样式脚本引用 warning 已清掉

上一轮引入了一个新的脚本 warning：

- `UIStyle` 既是全局类名
- 又在各脚本里被当作同名常量 preload

本轮已经统一改成：

- `UI_STYLE`

因此当前运行 warning 已重新收敛到旧基线：

- 仅剩 `EventBus` 的 signal 未显式使用 warning

### 3.4 项目再次验证为可运行

本轮完成后又做了两轮验证：

1. Godot 无界面启动
- 退出码 `0`

2. Godot MCP 调试启动
- 项目成功启动
- 调试输出没有新的解析错误或阻塞型运行问题
- 只剩 `EventBus` signal warning
