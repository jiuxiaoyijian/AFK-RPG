# 项目进展记录 v3.2

## 1. 本轮定位

本轮承接 `v3.1` 的统计可视化，继续处理实际游玩中的两个明显 UI 问题：

- 主界面下方的 `装备概览` 与 `最近掉落` 卡片压到了战斗区域
- `背包 / 研究 / 图鉴 / 统计` 等二级页面虽然已经能独立弹出，但内部层级仍然不够清晰

本轮目标不是继续堆文字，而是把主界面与二级页面都做成更稳定的视觉分区。

## 2. 本轮目标

目标：

- 让主战斗区中央重新保持可观看状态
- 让二级页面打开后，玩家能快速识别摘要、列表、详情、操作的位置
- 继续维护版本化项目进展文档

## 3. 本轮完成内容

### 3.1 主 HUD 下半区重新排布

本轮重新调整了主 HUD 中 `装备概览` 与 `最近掉落` 两张卡的位置和尺寸：

- 两张卡整体上移
- 卡片宽高做了收缩
- 掉落卡进一步向右侧收拢
- 内部槽位与文本区域同步压缩

这样战斗区的中下部重新空出来，不再被两个摘要卡直接盖住。

### 3.2 二级页面增加全局 dimmer 和导航收束

为了让玩家打开二级页时更聚焦，本轮新增了全局 UI 遮罩和导航收束逻辑：

- 打开常规二级面板时，主界面后方显示半透明暗幕
- 打开二级面板时，底部主导航栏自动隐藏
- 离线收益弹层仍保持自己的独立遮罩逻辑

这样二级页不再只是“浮在主界面上”，而是更接近真正的模态层体验。

### 3.3 背包面板改成列表 + 详情 + 操作三段式

`背包` 面板本轮增加了明确分区：

- 顶部保留标题与自动分解阈值控制
- 左侧为 `物品列表`
- 右上为 `装备详情`
- 右下为 `快捷操作`

并将 `装备 / 锁定 / 分解` 按钮重新排布，减少原先“信息与操作混在一起”的拥挤感。

### 3.4 研究、图鉴、统计统一为分区骨架

本轮把 `研究 / 图鉴 / 统计` 三个二级页面统一成相近的骨架结构：

- 顶部 `标题栏`
- 上方 `摘要区`
- 左侧 `筛选或列表区`
- 右侧 `详情区`
- 底部或下半部 `操作区 / 效率区 / 最近样本区`

具体表现为：

- `研究`：分出 `研究分类 / 研究节点 / 节点详情 / 成长操作`
- `图鉴`：分出 `条目分类 / 图鉴条目 / 条目详情 / 追踪操作`
- `统计`：分出 `快捷定位 / 节点列表 / 样本概览 / 效率对比 / 最近样本`

这让四个二级页即使内容不同，也具备了一致的视觉阅读顺序。

## 4. 本轮关键实现点

本轮主要是场景结构和 UI 管理层面的整理：

- `scenes/ui/hud.tscn`：调整主 HUD 摘要卡的布局
- `scenes/main/game_root.tscn`：加入全局 `UIDimmer`

- `scenes/ui/research_panel.tscn`：加入摘要/筛选/列表/详情/操作分区
- `scenes/ui/codex_panel.tscn`：加入摘要/筛选/列表/详情/操作分区
- `scenes/ui/drop_stats_panel.tscn`：加入快捷定位/列表/详情/效率/最近样本分区

## 7. 当前项目状态

截至 `v3.2`，项目的 UI 已经从“主界面和弹层堆在一起”进一步推进到：

- 主战斗区重新留白
- 二级页具备明确模态层
- 二级页内部开始形成一致的内容骨架

这意味着当前界面虽然还没有正式美术资产，但已经不再只是“功能能用”的调试面板，而是开始具备挂机游戏该有的层级秩序。

## 8. 当前已知问题

- 二级页虽然已经分区，但还缺少更强的配色层次、图标和视觉重点
- 统计页的 `最近样本` 仍然是文本记录，尚未做成更明显的记录卡或时间线
- 背包和图鉴条目目前仍以 `ItemList` 为主，交互颗粒度偏粗

## 9. 推荐下一步

按当前节奏，下一轮建议优先做：

- 给二级页补充轻量级图标、状态色和更清晰的标题层级
- 把 `最近样本 / 最近掉落 / 离线收益` 进一步做成结果卡片化表达
- 继续增强主战斗区的动效和伪图标，让首页更像“可观看的挂机战斗界面”
