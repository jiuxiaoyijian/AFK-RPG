# 项目进展记录 v5.1

日期：2026-03-28

---

## 本轮目标

把项目从“列表式挂机工具”继续推进到更像 ARPG / RPG 的装备驱动体验，重点落在：

1. 背包页升级成纸娃娃 + 格子库存
2. 百炼坊升级成工坊式三栏结构
3. HUD 的 Build / 目标 / 掉落主链进一步收口
4. 在不改存档结构的前提下，先把 UI 改造建立在新的 screen state / ViewModel 层之上

---

## 已完成

### 1. 新增 RPG 化 ViewModel / Presentation 层

本轮新增了四个轻量服务：

- `ItemPresentationService`
- `InventoryViewModelService`
- `EquipmentViewModelService`
- `CubeViewModelService`

职责分别覆盖：

- 物品短名、标签、槽位名、对比文案
- 格子背包状态映射
- 纸娃娃 9 槽摘要
- 百炼坊候选物、配方材料与结果预览

这一层的目的，是把 `GameManager` 从“直接供面板拼文案”转成“提供 screen state 聚合接口”。

### 2. GameManager 已补 RPG 化聚合接口

本轮 `GameManager` 已新增：

- `get_inventory_screen_state()`
- `get_equipment_screen_state()`
- `get_cube_screen_state(page_id)`
- `get_item_card_title()`
- `get_item_card_subtitle()`
- `unequip_slot(slot_id)`

其中背包与百炼坊面板已开始直接吃这些聚合态，而不是再直接操作原始数组。

### 3. 背包已重构成纸娃娃 + 格子库存

`InventoryPanel` 已完成首轮 RPG 化：

- 左侧：角色纸娃娃 9 槽
- 中间：10x4 格子背包
- 右侧：详情、对比、标签
- 底部：筛选、排序、装备、卸下、锁定、分解、自动分解阈值

当前格子只是展示层：

- 不做真实摆放
- 每件物品固定 1x1
- 不改存档中的 `inventory: Array`

### 4. 百炼坊已重构成工坊式三栏

`CubePanel` 现在不再依赖 `ItemList` 作为主交互，而是改成：

- 左：工坊分支 + 候选目标
- 中：目标详情 + 必要选项
- 右：材料、结果预览、执行反馈

五个百炼配方和 `武学秘录` 子页都仍然接在当前真实运行链上，没有回退为纯展示稿。

### 5. HUD 已开始向 RPG 主循环驾驶舱收口

本轮 HUD 先做了第一步术语与信息结构校正：

- `TargetCard` 标题改为 `当前 Build`
- `DailyGoalCard` 标题改为 `当前目标`
- `EquipCard` 标题改为 `已穿装备 · 9槽`

同时 `TargetCard` 已开始混合吃 `BuildAdvice + EquipmentScreenState`，不再只是一块孤立的掉落追踪文本。

### 6. 已新增结构文档

本轮已新增：

- `文档/15_RPG化系统与交互重构方案.md`

文档中冻结了：

- 格子库存模型
- 纸娃娃结构
- 百炼坊三栏结构
- RPG 化 ViewModel 拆分方式
- 当前边界与验收标准

本轮通过 Godot MCP 再次启动工程验证。

## 当前边界

这轮已经把“背包像角色装备页、百炼坊像工坊”这件事做出了第一版，但仍有边界：

- 还没做真实拖拽
- 还没做多尺寸物品占位
- 还没做仓库分页
- HUD 仍是“在现有框架上收口”，还不是完全重排后的终版
- Figma / prototype 还没有完全追平这轮新的纸娃娃 + 格子库存结构

---
