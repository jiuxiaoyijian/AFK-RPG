# 项目进展记录 v1.6

## 1. 本轮定位

本轮承接 `v1.5` 的图鉴分类浏览，继续把“推荐刷图”从规则评分推进到更接近真实 farming 参考的层级：

- 增加期望值可视化
- 让推荐结果不只是“推荐去哪”，而是“推荐值不值、多久能见到一次”
- 继续维护版本化项目进展记录

## 2. 本轮目标

目标：

- 给图鉴推荐补期望收益信息
- 让传奇 / 底材 / 词条三类条目都能看到更具体的 farming 估算
- 提升当前追踪目标的可执行性

## 3. 本轮完成内容

### 3.1 推荐系统新增期望值估算

本轮在 `LootCodexSystem` 中新增了期望值估算逻辑，推荐结果现在会额外给出：

- 单次结算期望值
- 平均约多少次结算能见到一次
- 按节点时限粗估大约多少分钟

这样图鉴已经不只是“说明系统”，而开始具备“辅助决策”的作用。

### 3.2 期望值覆盖三类条目

目前三类图鉴条目都支持期望值估算：

- `底材`
- `词条`
- `传奇`

不同条目采用的估算方式不同：

- 底材：按可选底材池和当前 build 偏好加权估算
- 词条：按词条池、词条权重、预期词条数和来源限制估算
- 传奇：按传奇来源池、传奇权重、装等稀有度门槛和保底规则估算

这使得推荐结果更接近“真实期望”，不再只是简单规则排序。

### 3.3 推荐刷图说明更具体

本轮后，推荐文本会同时提供两层信息：

1. 推荐原因
2. 期望收益

示例维度包括：

- 当前可刷 / 后续章节
- 掉装轮次
- 装备掉落概率
- 综合评分
- 掉落方向
- 单次期望
- 约多少次结算见一次
- 约多少分钟见一次

这让“为什么推荐这里”比之前更透明。

### 3.4 追踪目标摘要也接入了期望值

HUD 中当前追踪目标的摘要，不再只有：

- 目标名
- 推荐刷图

现在还会补上：

- 约多少次结算见一次

这让玩家在不打开图鉴面板时，也能对当前 farming 目标有更直观的时间预期。

## 4. 本轮关键实现点

逻辑侧重点：

- `LootCodexSystem._get_expectation_metrics()`：统一期望值入口
- `LootCodexSystem._get_base_distribution()`：估算底材出现概率
- `LootCodexSystem._estimate_affix_probability_per_item()`：估算词条出现概率
- `LootCodexSystem._estimate_legendary_probability_per_item()`：估算传奇出现概率
- `LootCodexSystem._build_expectation_text()`：把期望值转成可读文本

## 7. 当前项目状态

截至 `v1.6`，图鉴与推荐系统已经从：

- 只是分类浏览

进一步升级到：

- 分类浏览
- 目标追踪
- build 相关推荐
- 期望值可视化

现在玩家能更直接地看到：

- 某条传奇/词条/底材值不值得追
- 刷哪个节点更划算
- 大概要刷多久才能见到一次

## 8. 当前已知问题

- 当前期望值仍然是基于规则和静态权重的估算，不是基于真实掉落统计采样
- 对“同一件装备内多词条无放回抽取”的处理仍是近似估算
- 节点用 `time_limit` 粗估分钟数，和真实平均通关时长还不完全一致

- 或继续拆 `GameManager`，把节点推进与背包装备管理分离
- 或开始补章节视觉差异化，让当前系统成长在画面上也能被感知
