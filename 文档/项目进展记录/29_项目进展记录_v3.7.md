# 项目进展记录 v3.7

## 1. 本轮定位

本轮继续围绕 `UI 可读性` 和 `调试效率` 两个方向做收束。

用户本轮提出的重点非常明确：

- 继续推进项目
- 所有弹出式 UI 的底板改为不透明，避免半透明叠在战斗画面上导致信息重叠看不清

因此本轮的工作重点放在：

- 统一弹窗底板样式
- 继续补强 GM 面板常用调试能力

## 2. 本轮目标

目标：

- 让所有二级弹窗与离线结算卡在阅读时不再受战斗背景干扰
- 保持同一套弹窗底板风格，避免后续又出现某些弹窗透明、某些弹窗不透明
- 继续降低日常调试成本

## 3. 本轮完成内容

### 3.1 所有弹出式 UI 底板统一改为不透明

本轮没有只对单个场景手改，而是直接在 UI 管理层新增了一套统一的弹窗底板样式覆盖。

当前已统一覆盖的弹出式 UI 包括：

- 背包
- 研究
- 图鉴
- 掉落统计
- GM 面板
- 离线收益弹窗

样式策略为：

- 主面板使用更深的实底色
- 头部条使用略亮的实底色
- 子分区使用独立的实底色
- 所有面板边框统一补齐

这样做的好处是：

- 不再透出战斗背景和地面掉落
- 文本阅读明显更清楚
- 后续新增弹窗也更容易接入统一风格

### 3.2 离线收益卡面改为实底

除了普通 `Panel` 底板外，离线收益弹窗原本还叠了一层半透明 `CardArt`，这也是阅读时会感觉“底板仍然发灰透底”的原因之一。

本轮将离线收益卡面也改为了不透明版本：

- 常规离线收益卡面改为实底
- Boss 风格离线收益卡面也改为实底

这样离线结算页在视觉上会更加稳，更像正式结算卡而不是覆盖在战斗画面上的浮层。

### 3.3 GM 面板继续补两项高频调试能力

在上一轮已经具备：

- 加资源
- 设资源
- 生成装备
- 跳节点
- 解锁天赋树
- 解锁图鉴
- 保存 / 读档

的基础上，本轮继续补了两个高频能力：

- `资源拉满`
- `清空背包`

这两个操作对测试非常有用：

- 想直接验证高阶成长、购买、升级链路时，可以一键资源拉满
- 想重新验证掉落、装备生成与背包排序时，可以快速清空背包

## 7. 当前项目状态

截至 `v3.7`，项目的弹出式 UI 已经从：

- 半透明底板
- 容易与战斗背景、角色、地面掉落叠在一起

推进到：

- 主弹窗、子区块、头部条统一实底
- 离线收益卡面也为实底
- 信息阅读显著更清楚

同时 GM 面板的调试链路也进一步补强，日常验证效率更高。

## 8. 当前已知问题

- 弹窗底板已经不透明，但按钮和列表仍然是默认主题风格，整体美术一致性还不够强
- GM 面板仍偏向开发调试工具感，未做正式游戏内工具风格包装
- 移动端弹窗虽然已有横屏基础设置，但还没有针对触控点击区域专门放大

## 9. 推荐下一步

按当前节奏，下一轮建议优先做：

- 继续统一所有弹窗的按钮、列表、滚动条风格
- 给 GM 面板补 `清空掉落统计 / 跳章节 / 多测试快照`
- 开始做移动端触控按钮与安全区适配
