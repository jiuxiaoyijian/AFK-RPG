# 项目进展记录 v3.24

## 1. 本轮定位

本轮继续沿着 `P1-4 Web/展示优化层` 往前推进，但目标不再是补交互，而是把上一轮已经完成的网页原型再整理成一套更适合评审、汇报和快速过稿的静态展示页。

上一轮的 `index.html` 更像“可点可切的交互稿”：

- 适合自己对照结构和做前端/游戏内映射
- 但并不一定适合在评审会上直接展示

因此本轮重点是：

- 把核心布局、关键面板、结果卡与题材语言压缩成几张静态评审板
- 让策划、美术、程序都能更快对焦“这套 UI 到底要长什么样”

## 2. 本轮目标

目标：

- 新增一份适合评审的静态展示页
- 让“主界面 / 模态面板 / 结果卡 / 术语基线”在同一文件内清楚呈现
- 与现有交互原型形成分工：
  - `index.html` 继续承担交互原型
  - `review.html` 负责评审展示

## 3. 本轮完成内容

### 3.1 已新增独立评审展示页

本轮新增：

- `文档/prototype/review.html`

这份文件不是把原来的交互原型简单截图，而是重新按“评审视角”组织了一套静态展示内容。

当前页面结构分成三块：

- 主界面评审板
- 模态面板评审板
- 结果卡与术语评审板

这样在开会时，不需要来回切面板和演示交互，也能快速对照：

- 首页信息结构是否成立
- 二级面板是否统一
- 结果卡是否具备传播性
- 题材语言是否已经成体系

### 3.2 主界面评审板已压缩成一张完整构图

主界面部分当前集中展示了：

- `TopBar`
- `BattleCard`
- 中央战斗区
- `机缘追踪`
- `今日机缘`
- `BattleSafeFrame`
- `MainNavBar`

并在板下补了三类评审说明：

- 视线流向
- 题材气质
- 当前应重点讨论的问题

这让它更适合作为“第一张评审图”，快速判断首页是否已经把挂机、build、目标感三层信息讲清楚。

### 3.3 模态面板评审板已拆成背包 / 悟道 / 异闻录 / 推演

本轮没有继续做新的交互切换，而是把当前最关键的四类二级面板压成静态状态展示：

- 背包
- 悟道
- 异闻录
- 机缘推演

其中：

- 背包重点强调 `列表 / 品质 / 详情 / 操作按钮`
- 悟道重点强调 `当前可悟道什么`
- 异闻录与机缘推演则分别承担 `收藏追踪` 和 `刷图推荐`

这样后续无论是继续做网页稿细化，还是往 Godot 里回灌，都更容易按模块拆解。

### 3.4 结果卡与术语评审板已补齐

本轮还把两个高频结果卡做成了静态评审版本：

- `闭关所得`
- `破局要闻`

同时在同一页底部补入了四个评审提示卡，专门用于讨论：

- 术语基线是否统一
- 配色判断是否已经立住
- 评审时应该先看什么
- 后续如何回接到游戏内资源清单

这比单纯展示结果卡本身更适合团队开会时直接使用。

## 7. 当前项目状态

截至 `v3.24`，`文档/prototype/` 目录已经形成两类用途明确的网页稿：

- `index.html`：交互原型
- `review.html`：静态评审展示页

这意味着项目现在在“网页展示层”已经不再只有单一文件，而是形成了：

- 自己推敲结构时用的原型稿
- 给团队过稿时用的评审稿

两者可以并行推进，不必再互相挤压。

## 8. 当前已知问题

- 当前评审页仍是 HTML 静态稿，尚未导出为图片、长图或 PDF
- 视觉层已经适合评审，但还未标注尺寸、间距和组件规范
- 还未把“评审通过后的组件清单”拆成真正的 UI 开发任务单

## 9. 推荐下一步

下一轮建议优先从下面两个方向里选一个：

- 继续增强 `review.html`，补尺寸标注、组件态和评审批注层
- 开始把 `review.html` 与 `index.html` 共同拆解成真实的 HUD / Panel / Popup 开发清单

如果继续沿当前方向往下做，最值得补的是：

- 组件尺寸标注
- 配色 / 状态 / 品质色图例
- 输出一版更适合发给外部协作者的长图或 PDF 样式页
