# 项目进展记录 v1.1

## 1. 本轮定位

本轮承接 `v1.0`，继续按照既定优先级推进研究树：

- 把研究树从“已有战斗 / 挂机节点”扩成更明确的 `战斗 / 挂机 / 经济` 三分支
- 让经济分支真正影响资源获取与回收，而不只是文案层概念
- 继续维护版本化项目进展记录

本文件是新增记录，不覆盖旧版本文档。

## 2. 本轮目标

目标：

- 补齐研究树中的经济向成长线
- 把资源加成收口到统一结算入口，保证在线、离线、首通、分解逻辑一致
- 让研究面板能更清晰地区分三条分支

## 3. 本轮完成内容

### 3.1 研究树扩展为三分支

本轮新增了经济分支研究节点：

- `贸易合同`
- `回收工艺`
- `核心提炼`

当前研究树正式形成三条方向：

- `战斗`：攻击、核心伤害、核心冷却
- `挂机`：离线效率、离线时长上限
- `经济`：金币收益、铁屑收益、核心收益、碎片收益、分解回收率

这意味着研究树已经不只是“战斗强化树”，而是开始承担中期整体养成规划。

### 3.2 经济分支接入统一资源结算

本轮把资源收益加成收口到了 `GameManager`：

- `grant_rewards()` 现在会先按研究加成修正奖励数值，再真正入账
- 新增 `get_adjusted_reward_count()`
- 新增 `get_adjusted_salvage_scrap()`

已接入经济分支影响的来源包括：

- 节点首通奖励
- 普通战斗掉落材料
- 离线材料结算
- 手动分解
- 自动分解

这样做的结果是：

- 以后继续加经济向研究或被动时，不需要到处重复改逻辑
- 在线和离线收益不会因为分散计算而不一致

### 3.3 研究面板分支筛选

研究面板现在新增了：

- `全部`
- `战斗`
- `挂机`
- `经济`

四个筛选按钮。

同时增加了概览区，显示：

- 当前筛选分支
- 三条分支当前总等级 / 最大总等级
- 当前经济向收益加成汇总

这让研究树从“单列表浏览”提升成了更像正式系统面板的状态。

## 4. 本轮关键实现点

逻辑侧重点：

- `GameManager.RESEARCH_META_KEYS`：扩充经济向元加成键
- `GameManager.grant_rewards()`：统一资源收益修正入口
- `GameManager.get_adjusted_salvage_scrap()`：统一分解收益修正入口
- `LootSystem` 与 `OfflineSystem`：显示文本改为基于实际结算数值
- `ResearchPanel`：支持按分支筛选并显示研究概览

## 7. 当前项目状态

截至 `v1.1`，项目已经具备：

- 两章连续推进
- 普通 / 精英 / Boss 掉落来源分层
- 高价值材料与研究树消耗回路
- `战斗 / 挂机 / 经济` 三分支研究树
- 统一的资源收益修正入口
- 更清晰的研究面板筛选与概览

## 8. 当前已知问题

- 研究树现在已经功能化，但仍是列表式原型，还不是可视化连线树
- 资源加成已统一，但装备掉落概率、定向 farming 保障机制还没有经济分支联动
- 系统边界依旧偏原型化，`GameManager` 仍承担了较多职责

## 9. 推荐下一步

按当前顺序，下一轮建议优先做：

- 开始把 `GameManager / LootSystem / CombatSystem` 往更标准的系统边界拆分
- 或者补一轮更明显的章节视觉差异与场景表现
- 或者继续做“图鉴 / 目标追踪 / 定向 farming 提示”，让掉落分层更可感知
