### 2. Wave 1：状态解耦与查询层

新增：

- `scripts/utils/build_query_service.gd`
- `scripts/utils/loot_rule_service.gd`

已完成的接线：

- `GameManager.get_total_combat_bonuses()` 改为委托 `BuildQueryService`
- `GameManager.get_build_advice_data()` 改为委托 `BuildQueryService`
- `LootCodexSystem` 的推荐刷图 / 理论期望 / 传奇门槛概率等逻辑改为委托 `LootRuleService`
- `SaveManager` 新增 `_emit_loaded_state()`，收口读档后广播链
- `EventBus` 预补了后续暗黑like 重构需要的统一事件：
  - `set_bonus_changed`
  - `cube_operation_completed`
  - `martial_codex_changed`
  - `rift_run_started`
  - `rift_run_finished`
  - `gem_upgraded`
  - `paragon_changed`
  - `season_reborn`

### 3. 自动处理规则先行修正

为避免当前原型继续污染后续套装 / 百炼坊体验，已先收紧高价值装备自动处理规则：

- `epic` 及以上装备不再进入自动穿戴流
- 带传奇特效的装备不再进入自动穿戴流
- 高价值装备强制进入玩家手动处理流

这一步是为了给后续 `传承 / 百炼坊 / 武学秘录` 接入留出正确的物品处理前提。

## 风险与决策

### 已确认决策

- 继续保留武侠包装，不回退到西幻
- 系统结构按暗黑like 走
- 可破坏旧存档兼容，但本轮尚未改存档结构，所以 `CURRENT_SAVE_VERSION` 暂未递增

### 已知风险

- 当前运行中的 UI 仍然是旧导航结构，和新版原型存在阶段性不一致
- `BuildQueryService` 现已成为多处 UI 的共同依赖，后续继续重构时要避免再次把文案和状态拼装塞回 `GameManager`
- `LootRuleService` 现阶段只完成推荐/理论层统一，尚未与 `EquipmentGeneratorSystem` 共用概率核心
