# 项目进展记录 v5.4

日期：2026-03-28

---

## 本轮目标

继续推进 `RPG 化系统与交互重构` 的组件层，把统一物品卡片从“只有文本与边框色”补到更接近正式 RPG 物品格子的视觉形态。

重点目标：

- 给 `ItemCardButton` 增加图标区
- 给 `ItemCardButton` 增加品质框贴图
- 给 `ItemCardButton` 增加角标
- 让纸娃娃、格子背包、百炼坊候选物继续吃同一套组件能力

---

## 已完成

### 1. 统一卡片组件已补图标区

`ItemCardButton` 现在不再只是：

- 标题
- 副标题
- 标签行

而是新增了独立的：

- 图标容器
- 图标贴图加载

当前卡片会优先读取传入的 `icon_path`，若未显式传入，则按 `slot_id` 自动回退到装备槽位图标映射。

这意味着：

- 纸娃娃槽位
- 背包格子中的装备
- 百炼坊候选物

开始具备统一的“看起来像同一类物品卡”的识别感。

### 2. 统一卡片组件已补品质框贴图

卡片组件新增了品质框贴图层。

当前会按 `rarity` 自动映射：

- `common / uncommon`
- `rare`
- `epic / set`
- `legendary`
- `ancient`

到现有资源中的对应框体贴图。

这样当前组件不再只是“按钮底色变一下”，而是已经开始接近真正 RPG 装备卡格的视觉语言。

### 3. 统一卡片组件已补角标

卡片右上角现在会自动显示单个高优先级角标。

当前优先级为：

1. `传承`
2. `武学`
3. `高价值`
4. `已精炼`
5. `已锁定`

这样玩家在不展开详情时，也能更快看懂：

- 这件装备是不是传承件
- 有没有武学
- 是否属于掉落决策流里的高价值目标

### 4. 背包 / 纸娃娃 / 百炼坊已统一接入新字段

当前这些入口都已经开始向 `ItemCardButton` 传递：

- `slot_id`
- `rarity`

因此同一套组件能力已经进入真实运行链，而不是停留在“组件可用，但还没真正吃到业务数据”的阶段。

覆盖范围包括：

- `InventoryPanel` 左侧 9 槽纸娃娃
- `InventoryPanel` 中间 10x4 格子背包
- `CubePanel` 左侧候选物列表

## 验证结果

本轮再次通过 Godot MCP 启动验证。

结果：

- 无新增 parse error
- 无新增 runtime error
- 无 `missing resource`

当前剩余 warning 仍然主要是既有的：

- `EventBus` signal warning
- `BuildQueryService / LootRuleService` 等常量重名 warning

本轮新增组件代码中的 `RuntimeTextureLoader` 常量重名 warning 已顺手清理。

## 当前边界

当前统一卡片组件已经更像正式 RPG 物品格，但仍有这些边界：

- HUD 的装备概览与掉落高光还没有完全切到同一组件
- 还没有拖拽换装
- 还没有更复杂的多层品质装饰
- 还没有把物品专属美术图标进一步细化到“每件装备不同图”
