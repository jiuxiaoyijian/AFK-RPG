# AFK-RPG

一个基于 Godot 4.6 制作的横版挂机 ARPG 原型，目标是拆解并重构《英雄没有闪》这类产品的核心体验，包括自动战斗、刷图掉落、装备构筑、离线收益、图鉴追踪与研究成长等系统。

## 项目定位

当前仓库处于 `原型验证 + 持续迭代` 阶段，重点在于：

- 验证挂机刷图的主循环是否成立
- 验证战斗反馈、掉落表现与收束动画的体验
- 验证装备、词条、传奇目标追踪与研究成长的中期驱动力
- 持续收敛 UI 层级与 Web 可交付能力

项目当前更接近一个可玩的系统原型，而不是最终完成品。

## 当前已实现

- 自动战斗循环：主角与敌人自动交战、刷图、结算、继续推进
- 核心成长：核心技能、研究树、资源积累、离线收益
- 装备系统：底材、词条、传奇词缀、掉落生成与装备概览
- 图鉴系统：图鉴发现、传奇追踪、推荐刷图节点
- 掉落统计：节点掉落样本、效率对比、最近样本记录
- 掉落演出：抛物线掉落、主动/被动拾取、飞向 UI 的收束动画
- 调试能力：GM 面板、三存档位、资源重置、图鉴/研究解锁
- UI 收敛：战斗安全框、弹窗不透明底板、可滚动详情区
- Web 导出：已补齐 `Web` 导出预设并通过本地导出验证

## 当前键位

- `1 / 2 / 3`：切换核心技能流派
- `R`：重开当前流程
- `T`：切换追踪目标
- `I`：打开背包
- `U`：打开研究
- `O`：打开图鉴
- `P`：打开掉落统计
- `G`：打开 GM 面板
- `F5`：快速存档（档位 1）
- `F8`：快速读档（档位 1）
- `Esc`：关闭当前弹窗

## 本地运行

### 方式一：用 Godot 编辑器打开

1. 使用 `Godot 4.6.x`
2. 打开项目目录
3. 运行主场景 `res://scenes/main/game_root.tscn`

### 方式二：命令行导出 Web

仓库已经包含 `export_presets.cfg`，可直接使用 Web preset 导出：

```powershell
& "D:\SteamLibrary\steamapps\common\Godot Engine\godot.windows.opt.tools.64.exe" `
  --headless `
  --path "D:\GodotProject\桌面挂机" `
  --export-release "Web" "build/web/index.html"
```

导出结果默认位于：

- `build/web/index.html`
