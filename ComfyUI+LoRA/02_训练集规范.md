# 训练集规范

## 1. 文档目的

本文档用于约束 `桌面挂机` 项目的 LoRA 训练集整理方式。

如果后续由你自己、外包美术、或其它 AI 一起生产训练素材，统一遵守本规范，能显著减少：

- 风格混乱
- 标签混乱
- 训练目标不清
- 多套 LoRA 相互污染

## 2. 当前训练集总原则

### 2.1 一套 LoRA 只学一类核心目标

不要把下面内容混在一套训练里：

- 主角
- 怪物/Boss
- 背景
- UI 图标
- 结果卡

正确做法是分目录、分 LoRA、分配置。

### 2.2 原图与处理图分开

- 原始图放 `01_dataset`
- 处理后图放 `02_processed`

不要直接在原图上反复覆盖。

### 2.3 caption 要可维护

要求：

- 统一语言
- 统一前缀
- 统一触发词
- 不同人写的标注风格不能完全不同

## 3. 当前推荐训练集拆分

## 3.1 style

路径：

- `01_dataset/style`

学习目标：

- 灾后黑铁废土
- 工业遗迹
- 熔核科技残骸
- 废土工业色板与氛围

建议素材：

- 场景气氛图
- 工业设施
- 熔岩矿场
- 金属遗迹
- 废墟高墙
- 炉心能量感参考

不要混入：

- 明确的主角特写
- 大量 UI 图标
- 卡通风角色

## 3.2 hero

路径：

- `01_dataset/hero`

学习目标：

- 主角外观
- 主角装备轮廓
- 主角兵装感

建议素材：

- 半身/全身设定图
- 战斗姿态图
- 武器与背载炉心装置图
- 不同流派下的主角视觉变化

不要混入：

- 大量复杂背景
- 不同风格的人物
- 普通怪或 Boss

## 3.3 enemies

路径：

- `01_dataset/enemies`

学习目标：

- 敌人阵营风格
- 精英与 Boss 识别

建议素材：

- 异化兽群
- 守卫残军
- 熔灾信徒
- 铁爪巨兽
- 熔核监工

不要混入：

- 主角
- UI 图标
- 与项目题材不符的现代军武敌人

## 3.4 items_icons

路径：

- `01_dataset/items_icons`

学习目标：

- 技能图标
- 装备图标
- 资源图标
- 品质边框
- 结果卡元素

建议素材：

- 单体图标
- 透明底图标
- 品质边框
- 工业终端风格图形

不要混入：

- 大型场景
- 人物立绘
- 混乱背景

## 3.5 backgrounds

路径：

- `01_dataset/backgrounds`

学习目标：

- 章节背景风格
- 横版场景构图

建议素材：

- 废墟边境背景
- 焦灼矿场背景
- 工业遗迹远景
- 熔炉设备中景
- 地面与前景装饰参考

不要混入：

- 大型角色主体
- 图标
- UI 面板

## 4. 命名规范

建议文件名结构：

`类别_主题_序号`

示例：

- `style_ruinsborder_001.png`
- `hero_reclaimer_idle_001.png`
- `enemy_boss_ironbeast_001.png`
- `icon_core_whirlwind_001.png`
- `bg_scorchedmine_001.png`

对应 caption 文件：

- `style_ruinsborder_001.txt`
- `hero_reclaimer_idle_001.txt`

## 5. caption 规范

## 5.1 统一结构

建议 caption 采用：

`全局风格前缀, 子类触发词, 主体描述, 关键视觉特征, 可选补充`

示例：

`wasteland industrial fantasy, style_lora, ruined border outpost, dark steel debris, rust red glow, semi realistic 2d game art`

`wasteland industrial fantasy, hero_lora, frontier reclaimer, patched armor, furnace core backpack, semi realistic 2d character design`

## 5.2 全局前缀建议

所有训练集都建议包含同一组全局题材前缀：

- `wasteland industrial fantasy`
- `furnace core technology`
- `semi realistic 2d game art`

这样能让各套 LoRA 共享同一世界基调。

## 5.3 子类触发词建议

- `style_lora`
- `hero_lora`
- `enemy_lora`
- `items_icons_lora`
- `background_lora`

## 5.4 caption 注意事项

- 不要每张图写完全不同的句法
- 不要把与图无关的故事文本塞进去
- 不要混中英双语
- 不要让同一对象出现多个互相矛盾的名字

## 6. 数据质量要求

### 6.1 优先要的图

- 主体清晰
- 轮廓明确
- 风格统一
- 分辨率足够
- 题材贴近项目

### 6.2 不建议纳入训练的图

- 压缩严重
- 水印明显
- 风格完全不一致
- 构图过乱
- 主体过小
- 与项目题材偏差太大

## 7. 数量建议

以下是第一阶段建议规模，不求一步到位。

### style

- 建议起步：`40-80` 张

### hero

- 建议起步：`25-50` 张

### enemies

- 建议起步：`40-80` 张

### items_icons

- 建议起步：`60-120` 张

### backgrounds

- 建议起步：`30-60` 张

说明：

- 第一轮训练更重要的是“流程跑通”和“风格是否有效”
- 不一定一开始就追求大数据集

## 8. 与当前项目文档的映射

训练集应优先服务以下文档需求：

- `09_P0留存版开发路线图.md`
  重点支持观赏性、结果卡、目标感相关资源

- `10_题材与世界观包装草案.md`
  重点支持废土工业、熔核科技、章节敌人阵营统一

- `11_首批美术资源清单与外包需求表.md`
  重点支持主角、Boss、章节背景、图标、结果卡模板

## 9. 第一批最建议先收集的训练集

如果现在就开始准备素材，优先建议：

1. `style`
2. `items_icons`
3. `backgrounds`

原因：

- 最快形成可用的项目风格资产
- 最快服务当前 UI 与截图质量
- 风险比角色和 Boss LoRA 更低

## 10. 当前结论

训练集整理阶段最重要的不是“多”，而是：

- 分类明确
- 风格统一
- caption 可维护
- 能服务当前项目最急缺的资源

只要这四点先做好，后续无论是继续本地训练，还是交给其它工具链，都会顺很多。
